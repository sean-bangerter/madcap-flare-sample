Client Override Options

Secure Release Printing

The Client Override Options are used to change client functionality on a single device. Instead of making a change in the Admin Console, which impacts all clients, these options make changes on a per-device basis. Client Overrides are set in the Windows registry on the target device. To open these registry settings, just open the Registry Editor in Windows (Run > Regedit.exe).

Epic Print Server Mode

Epic Print Server Mode is set on the clients running epic print servers. It improves the efficiency of the client in the epic printing environment in the following ways:

  • Adds new printers or removes printers or printers that have had their deployment removed when no one is logged in to the server. Normally, printers are not removed until some logs in.
  • Disables unneeded host name to IP address resolution for printers.
  • Allows the installation of epic-licensed printers.

When using Printer Installer to manage printers in both the enterprise and EPIC environment, you will need to manually turn on EPIC Print Server Mode on the EPIC print servers. To turn on EPS mode manually, create the following registry entry on each EPS:

  1. Browse to HKEY_LOCAL_MACHINE\SOFTWARE\PrinterLogic\PrinterInstaller\Overrides.
  2. Create a new string value named epic.
  3. Set the value to 1.
  4. Reboot the workstation or shutdown and restart the client.

Hidden Client Icon Per Workstation

To manually hide the client icon per workstation instead of turning it off for everybody, create the following registry entry on each computer you wish to hide it on:

  1. Browse to HKEY_LOCAL_MACHINE\SOFTWARE\PrinterLogic\PrinterInstaller\Overrides.
  2. Create a new string value named client_show_icon.
  3. Set the value to 0.
  4. Reboot the workstation or shutdown and restart the client.

Client Check-in Interval Per Workstation

This client override option is used to manually specify a check-in interval for the client on a workstation or server. The benefit of setting the interval is being able to set different intervals for any reason, for example, an epic server may need a shorter interval than a workstation.

To manually set the automatic client check-in per workstation, instead of turning it on globally for everybody, create the following registry entry on each computer:

  1. Browse to HKEY_LOCAL_MACHINE\SOFTWARE\PrinterLogic\PrinterInstaller\Overrides.
  2. Create a new string value named client_check_periodically.
  3. Set the value equal to the number of minutes that should expire before each check in (0 means never check in).
  4. Reboot the workstation or shutdown and restart the client.

Force Delete Printers from Deployment Assignments

If, for any reason, we cannot validate a Active Directory user, computer, or group membership(s), then the default functionality is to not remove deployed printers from workstations. This functionality is set up to prevent printers from being unintentionally removed due to temporary issues with Active Directory.

To allow the client to delete printers that have been installed via deployment assignments even when Active Directory information cannot be determined, create the following registry entry on the Printer Installer Server (OR if this is a VDI session, then on the individual VDI workstation):

  1. Browse to HKEY_LOCAL_MACHINE\SOFTWARE\PrinterLogic\PrinterInstaller\Overrides.
  2. Create a new string value named AlwaysDelete.
  3. Set the value to 1 to enable the override.
  4. Reboot the workstation or shutdown and restart the client.

Move Cache Manager Storage to New Location

By default, the Cache Manager stores cache files to c:\program files (x86)\printer properties pro\printer installer client\cache. This override enables you to change the cache storage location, for example, to a different hard drive.

To manually set where the Printer Installer Cache Manager stores files, create the following registry entry on the Printer Installer Server:

  1. Browse to HKEY_LOCAL_MACHINE\SOFTWARE\PrinterLogic\PrinterInstaller\Overrides.
  2. Create a new string value named cache_manager_location.
  3. Set the value as the path to the new location.
  4. Reboot the workstation or shutdown and restart the client.

Store Self-installed Printers in Roaming Profiles

This override will allow TCP/IP printers that were installed via the Portal to be stored in users' roaming profiles, so the next time the user logs into a View Client or XenDesktop session the printers will be automatically reinstalled. This is normal behavior in a XenApp or Terminal Services session, but that can also be disabled with this key.

  1. Browse to HKEY_LOCAL_MACHINE\SOFTWARE\PrinterLogic\PrinterInstaller\Overrides.
  2. Create a new string value named RESTORE_USERINSTALLED_PRINTERS.
  3. Set the value to -1 to force OFF the behavior of TS mode that remembers printers from previous sessions, and cleans up reg keys that normally store them as users log in.
  4. Set the value to 0 to keep everything behaving as normal.
  5. Set the value to 1 to force ON the behavior of TS mode that remembers printers from previous sessions and restores them.
  6. Reboot the workstation or shutdown and restart the client.

The registry key where the user's printers are stored is: HKCU\Printers\Settings\TS.

Start Up Delay

This override is used to delay the client from performing any action until a specific period of time has passed from when the workstation or server was started. The Min value is set to the minimum amount of time to wait before the client is engaged. The Max value is set to the maximum amount of time to wait for the network to be ready before the client is engaged.

NOTE: This override is generally used for Windows workstations or VDI environments.

  1. Browse to HKEY_LOCAL_MACHINE\SOFTWARE\PrinterLogic\PrinterInstaller\Overrides.
  2. Create a new string value named MinStartupDelay.
  3. Set the value equal to the number of seconds that should expire before the Printer Installer client starts its process.
  4. Create a new string value named MaxStartupDelay.
  5. Set the value equal to the maximum number of seconds that the client will wait for the network to be available.
  6. Reboot the workstation or shutdown and restart the client.

Instance Delay

This override is used to delay the client from performing any action until a specific period of time has passed from when the client was started. The Min value is set to the minimum amount of time to wait before the client is engaged. The Max value is set to the maximum amount of time to wait for the network to be ready before the client is engaged.

NOTE: This override is generally used for terminal servers or Citrix/XenApp.

  1. Browse to HKEY_LOCAL_MACHINE\SOFTWARE\PrinterLogic\PrinterInstaller\Overrides.
  2. Create a new string value named MinNewInstanceDelay.
  3. Set the value equal to the number of seconds that should expire before the Printer Installer client starts its process.
  4. Create a new string value named MaxNewInstanceDelay.
  5. Set the value equal to the maximum number of seconds that the client will wait for the network to be available.
  6. Reboot the workstation or shutdown and restart the client.

Force Client to Ignore XenDesktop/ViewClient Registry Keys

This override will allow you to force the client to ignore the registry keys from both Citrix XenDesktop and VMware ViewClients and act as if it was running on a normal desktop.

  1. Browse to HKEY_LOCAL_MACHINE\SOFTWARE\PrinterLogic\PrinterInstaller\Overrides.
  2. Create a new string value named DISABLE_ENDPOINT_DETECTION.
  3. Set the value to 1 to enable the override.
  4. Reboot the workstation or shutdown and restart the client.

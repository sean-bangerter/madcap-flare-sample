Deploy a Printer

Secure Release Printing

Printer Deployment in Printer Installer is the process of installing and updating a print queue and driver that is installed on one or more workstations. However, before printers can be deployed to workstations, you must upload the driver to a workstation through the Admin Console.

Driver deployment without Printer Installer could be an arduous process of manually updating drivers through GPO, and worse, piecemeal if you have direct IP printers.

With Printer Installer, printer deployments take the print queue and driver you have uploaded to the Admin Console and easily installs them on one or more workstations. Any updates you make to the print queue or driver will be updated on the workstation the next time the client installed on the workstation checks in with the server. Any time you upload a new version of the driver to the Admin Console, then the next time the client checks in with the server, the driver on the workstation will be updated.

NOTE: Bear in mind the difference between printer deployment and driver management. A common mistake that some end-users run into is that they upload the drivers and print queues in the Admin Console to be installed on the workstations, and they think that the drivers are updated automatically. This is not true. You can upload a new version of the driver to the Admin Console, and when the client (installed on the workstation) checks-in with the server, the updated driver will be installed on the workstation.

When you select a printer object in the tree view, you can then click the Drivers/Profiles tab to see which drivers and profiles are applied to the printer object. As such, you can then click the Deploy tab to see users, groups, computers, IP address ranges, MAC address and other which are updated when you upgrade the drivers and/or profiles in the Drivers/Profiles tab. When the end-user logs into the Self-service Portal or the Release Portal, then the upgraded drivers and/or profiles are automatically applied to their workstation. Let's put this into context.

John, for example, is the Printer Installer administrator for his company. He logs into the Admin Console and clicks the KM Color printer object in the tree view (the KM printer object is associated with the Konica Minolta color printer in their office). He then clicks the Drivers/Profiles tab, which shows a 64-bit Windows 10 driver that was applied to the KM Color printer object about a month ago. John has an driver that he wants to use to replace the existing driver. He goes through the process of upgrading the driver. When he completes the upgrade process, John clicks the Deploy tab, which shows a group of eight users who will receive the upgrade. John then clicks Save. At this point, the next time each user in that group logs into the Self-service Portal or the Release Portal, then the updated driver from the KM Color printer object will automatically be updated on their workstation. The next time the end-user sends a print job to the KM color printer, their workstation will use the updated driver.

Can I sync my printer deployment?

The Deploy tab enables you to create printer deployments, the purpose of which is to install drivers and profiles on the workstations of the end-users selected in the deployment. Because you can automatically deploy the drivers to the workstation, the Admin Console also provides an option to remove the deployed drivers on a workstation if a deployment is removed from the Deploy tab. This option is called Sync Deployment. However, there are times that you want to remove a deployment from the Deploy tab, and you have the Sync Deployment setting enabled, which means that when you remove the deployment, the drivers associated with the printer object that uses the deployment will be removed from the workstations of the end-users associated with the deployment.

For example, the tree view has a printer object named Printer A, which has Driver B and Profile B assigned to it (in the Drivers/Profiles tab). Additionally, a group consisting of end-users John Smith, Joe Jackson, and Jeff Greene have been added to a deployment called Deployment D (in the Deploy tab). As a result, John Smith and Joe Jackson automatically have Driver B and Profile B installed on their computer, but Jeff already has that driver and profile added to his workstation. Moreover, the Sync Deployment setting is enabled in the Admin Console, so if Deployment D is ever deleted, then the driver will be removed from workstations belonging to the three end-users.

The Admin Console provides an additional option to retain the driver on end-user workstations if that driver existed on the workstation prior to the deployment being created. This option is called Keep Existing Printers. The example above states that two end-users did not have the driver installed when the deployment was created, but one end-user did. If you enable both the Sync Delete option and the Keep Existing Printers option, then when the deployment is deleted, the drivers will be removed from the workstations of John Smith and Joe Jackson (who had the drivers installed when the deployment was created), and Jeff Greene's workstation will retain the driver (because it was installed on his workstation before the deployment was created. The following steps describe the process for keeping existing printers on workstations when a deployment is deleted.

  1. In the Admin Console, click Tools > Settings > Client.
  2. In the TCP/IP Windows Shared Printer Settings section, click Sync Deployment.
  1. Click Keep Existing Printers.
  1. Click Save.

How do I deploy printers as shared?
  1. In the Admin Console tree view, select the printer object you would like to deploy.
  2. Click the Deploy tab.
  3. Click Add and then select Active Directory > User, Computer, or Group.
  1. In the Enter the object names to select field, type a partial or full name of the computer to which you would like to deploy the printer.
  1. Click Check Names.
  2. Select the computer.
  1. Click OK.
  2. Click OK again.
  3. When the computer appears in the deployment list, click Share.
  1. Click Save.
  2. Click Tools > Settings > Client.
  3. In the TCP/IP Printer settings section, click Create connections to shared TCP/IP printers.
  1. Click Save.

NOTE: When you have finished the setup, a recommended best practice is to refresh the Printer Installer client on the server.

What is the printer deployment priority?

Items higher in the following list take priority over items lower in the list.

  • User
  • Computer
  • Group (user membership)
  • Indirect user group (group is member from assigned group)
  • Container user
  • Container computer
  • Child-container user
  • Child-container computer
  • IP address range (smaller range)
  • IP address range (larger range)
How do I set a workstation default printer from Printer Installer?
  1. In the Admin Console tree view, select a printer object.
  2. Click the Deploy tab.
  1. Find the object to set as the default.
  2. In the Set Default field, select Once or Always.
  1. Click Save.
How do I set an IP range to deploy printers automatically?

An IP address range can be used to deploy to workstations that have IP addresses that fall within the range. Select an IP address range object in the Admin Console tree view and then click the Deploy tab. Any objects you assign to the Deploy tab can be installed on the workstations that have IP addresses falling within the given range.

For example, let's assume you have created an IP address range object in the Admin Console tree view. The name of this object is IP Range 1, and it is currently selected in your tree view. Furthermore, two employees, Mary and Shelley, have workstations with IP addresses that fall within the range set in IP Range 1. Finally, Printer 1 is a printer object that has been added to the Deploy tab for IP Range 1. With this setup, Printer 1 can be installed on both Mary's workstation and on Shelley's workstation because their workstations have IP addresses within the range set for IP Range 1.

Can I automatically remove deployed printers?

Because Printer Installer enables end-users to install printers on their workstations, the Admin Console provides an option to remove a printer from the workstations on which it has been installed if you remove the printer object corresponding to that printer from the Admin Console. For example, the Admin Console tree view has a printer object named Printer A, which also appears in the Self-service Portal. As a result, forty end-users have installed the printer from the Self-service Portal, so the drivers and profiles assigned to Printer A are installed on the end-users' workstations.

Printer Installer has an Sync Delete option that enables you to automatically remove the printer from end-users' workstations if you delete the printer object from the tree view. The value of this feature is that you do not have to remove the printer drivers and profiles individually from each workstation. Furthermore, you can deselect this feature, which leaves the printer drivers and profiles on the workstations even if you remove the printer object from the tree view. Using the example above, if you have Sync Delete enabled and you delete the Printer A object from the tree view, then the drivers and profiles associated with the Printer A object will automatically be removed from the 40 workstations on which they are installed. The following steps describe the process for enabling Sync Delete to automatically remove printer drivers and profiles if the printer object is deleted from the tree view. The Admin Console also provides an option to remove printer drivers and profiles from workstations that have been assigned via a deployment set in the Deploy tab.

  1. In the Admin Console, click Tools > Settings > Client.
  2. In the TCP/IP Windows Shared Printer Settings section, click Sync Delete to select it.
  1. Click Save.

Option 9: Deploy to an Advanced Group

For detailed information on this feature, see Advanced Groups.

Step 1: Set up your Advanced Group

  1. In the Admin Console, click Tools > Advanced Groups.
  2. When the Advanced Groups page appears, click Add.
  1. Type a name for the group and a description.
  2. Click Save.

Printer Installer will display the new group with the settings ready to set up the first rule.

  1. In the Operation drop-down, select the type of operation for the rule.

The operation determines whether the value will be included or excluded from the printer deployment. For example, select Contained In and then select an IP address range as the value to have the printer deployed to workstations in that range. Similarly, select Excluding with the same IP address range as the value, and the printer will be deployed to workstations with IP addresses outside that IP address range.

  1. In the Value field, select the deployment type that you are including in or excluding from the rule.

Using the example in the previous step, this field is where you would enter the IP address range after selecting Contained In or Excluding as the Operation. Depending on the value you select, a variety of options becomes available.

Any of the Active Directory options opens the Active Directory Object window that enables you to select an Active Directory user, group, and so on. Similarly, if you select IP address range, you will be able to select one of the IP address ranges from the ones that have been set up in the tree view. Selecting Hostname or MAC Address enables you to enter the specific values for either of those options.

What is the Terminal Service Session option?

The purposes of the Terminal Services Session option is to identify end-users who are working from a virtual desktop. As such, if the rule contains the Terminal Services Session option, the only end-users working from virtual machines will apply. See below.

The reason for this option is that come printers might not work correctly in a virtual environment, so if you want to deploy a printer to everyone but those using a virtual machine, you can.

  1. When the selected value appears in the Value field, click Save.
  2. (Optional) If you have another value that you want to apply to the Advanced Group, click the Plus Sign icon to add a new rule.

Add as many rules to your Advanced Group as you need. Just remember, the more rules you add, the more narrow the printer deployment will be. For example, a rule with an IP address range and an Active Directory group will deploy printers to a relatively narrow list of workstations, but if you add a third rule that uses Terminal Services Session, then the deployment will be even more narrow (perhaps too narrow) as it applies only to workstations that are connected over a virtual machine.

  1. Click Save after adding each new rule to the Advanced Group.

Step 2: Apply the Advanced Group to a Deployment

  1. Click Return to tree view if you are not already in tree view mode.
  2. Select a printer object in the Admin Console tree view.
  3. Click the Deploy tab.
  1. Click Add > Advanced Group.

All available advanced groups will appear as options.

  1. Select an advanced group.
  2. Click Add.
  3. Click Save.

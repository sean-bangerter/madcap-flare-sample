Step 1: Set up an LDAP Domain

  1. In the Admin Console, click Tools > Settings > General.
  1. Navigate to the LDAP Authentication Settings section.
  2. Click Add.
  1. When the LDAP settings appear, fill in the required fields.

For core features such as driver management, printer deployment, Role-based Access Control, and reporting, fill in the Domain Name and Primary LDAP server fields. The NETBIOS domain name and Base DN fields should fill in automatically. For both advanced modules (pull printing and mobile printing), fill in the entire top section down to the LDAP port field. Also fill in the Domain Alias, Bind User, and Bind Password fields. Make sure the LDAP Email Attributes are separated by commas. If you think you might need to use the SSO fields, please contact your PrinterLogic Customer Support Manager.

  1. When you have finished entering the LDAP settings, click Test Settings to ensure the settings are correct.
  2. Click Apply.
  3. (Optional) In the Default domain when logging in field, select the LDAP domain you have created.
  4. Click Enable Advanced LDAP administrative authentication.

When you click this option, the LDAP Group/User option becomes available in the Users section of the Admin Console as shown below.

  1. Click Save.

Step 2: Set up a Virtual Pull Print Queue

  1. In the Admin Console tree view, select the folder that will hold the printer object.
  2. Click New > TCP/IP Printer.
  1. In the Printer Name field, type a name for your virtual pull print queue.
  2. Click Make Pull Printer.
  1. If you have an existing driver you would like to use, click the drop-down box for the operating system you are supporting and then select an existing driver from the Driver/Profile Repository.
  2. If you do not have any drivers uploaded that you would like to use, click Upload New Driver under the bit-type and operating system you want to support.
  3. Navigate to the .inf file you would like to upload.
  4. Click Add.

Step 3: Configure the Pull Print Queue to be used for Mobile Printing

  1. In the Admin Console, click Tools > Settings > Printing.
  2. Scroll to the Mobile Printing section.
  3. Click None Selected.
  1. Select a printer object that has been configured as a virtual pull print queue.

Virtual pull print queue printer objects have Make Pull Printer selected in the Printing tab as shown below.

  1. Click Select.
  2. Click Save.

Step 4: Set a Printer to Release Mobile Print Jobs

Default Settings

  1. In the Admin Console tree view, select a printer object that you want to use to release mobile print jobs.

The printer object you select cannot be configured as a virtual pull print queue, which means the Make Pull Printer check box must not be selected.

  1. In the Mobile Printing section, click Use default Mobile Printing setting.
  1. Click Save.
  2. Click Tools > Settings > Printing.
  3. In the Mobile Printing section, click Enable Mobile Printer.
  1. Click Allow this printer to be configured as a user's Mobile Printer....
  2. Click Save.

Printer-specific Settings

  1. In the Admin Console tree view, select a printer object that you want to use to release mobile print jobs.

The printer object you select cannot be configured as a virtual pull print queue, which means the Make Pull Printer check box must not be selected.

  1. In the Mobile Printing section, click Use default Mobile Printing setting.
  1. Click Save.
  2. Click Tools > Settings > Printing.
  3. In the Mobile Printing section, click Enable Mobile Printer.
  1. Click Allow this printer to be configured as a user's Mobile Printer....
  2. Click Save.

Step 5: Enable and Configure Google Cloud Print

  1. In the Printing tab, select Enable Google Cloud Printing as shown below.
  1. In the Google Cloud Print Account section, type the admin email address and the admin password of the Google Cloud Print account to which the documents will be published for printing.

  1. (Optional) If you use a proxy server to connect to Google Cloud Print, then click Use Proxy Server and set your proxy server settings.

  1. Open a new browser window.

  2. Log in to your Google Cloud Account using the credentials you entered in the Admin Email and Admin Password fields above.

  3. Click Save.

At this point, DO NOT click Register Printer because you must follow the steps described in STEP 6: Log into your Google Cloud Print Account, before you can register the printer with Google Cloud Print.

Step 6: Log in to your Google Cloud Print Account and Register the Printer with Google Cloud Print

  1. Open a new browser window.
  2. Log in to your Google Cloud Print account using the credentials you entered in the Admin Email and Admin Password fields.

NOTE: You MUST perform this step before you can register the pull printer with Google Cloud Print.

  1. In the Admin Console, under Tools > Settings > Printing, click Register Printer.

After you click Register Printer, you will see the following message.

  1. Click OK.

Make sure you have any popup blockers disabled so the activation can continue.

  1. The browser in which you have Google Cloud Print open will appear as shown below.
  1. Click Finish Printer Registration.
  2. Click Manage your printers.

The Google Cloud Printer you registered will appear as a line item in Google Cloud Print as shown in the example below.

Take notice of the Owned by PrinterLogic text following the name of the printer.

  1. Click the printer name to select it.
  2. Click Share.
  3. When the Google Cloud Printer sharing settings appear, click Close.

Step 7: Send a Print Job from the Device to the Pull Print Queue

On the Android or Chromebook device, submit the print job to the Google Cloud Print account you have set up. When the print job appears in the pull print queue, then you can release the job on any pull print-enabled printer.

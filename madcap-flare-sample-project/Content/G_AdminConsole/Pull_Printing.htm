Pull Printing

Pull Printing enables you to submit a print job to a virtual queue that "holds" the print job on your workstation until you choose to release it on a pull print-enabled printer.

How long should it take for a secure print or pull print job to print?

Time to print depends on a number of factors, including size of the job and the type of printer, which might control how long it takes to start printing. However, after you click Release, expect the print job to be sent to the printer within 15 seconds. Mobile printing can be a little different since all mobile print jobs are sent, sequentially, by a central computer, which can be slower.

Without Pull Printing, print jobs release on the printer in order of arrival in the print queue. As a result, printing a document with personal, financial, or confidential information can be a problem if you are not immediately ready to pick it up at the printer. Pull printing solves this problem by releasing only when you are ready to release the print job from the Release Portal or the Control Panel Application.

Pull Printing requires two separate printer objects in the tree view. The first printer object will be used as the virtual pull print queueis the queue that holds a print job on an end-user's workstation until the end-user decides to release it or until the print job is purged from the queue. A Virtual Pull Print Queue has the "Make Pull Printer" button selected in the Printing tab., and the second printer object will represent the release printerThe actual printing device that outputs your print job..

For example, let's say you are the CTO of your organization, and you are preparing for an executive offsite by printing your internal product notes, which are not only essential to the company's business plan but also highly confidential. Print to the virtual pull print queue, and those notes will sit in queue on your computer until you are ready to release them. When you are standing at the release printer (there can be more than one), use one of the mechanisms (Release Portal or Control Panel Application) to release the print job.

NOTE: The printer you use to release the document must be on the same network used to send the document to the print queue.

Step 1: Set up an LDAP Domain in the Admin Console

  1. In the Admin Console, click Tools > Settings > General.
  1. Navigate to the LDAP Authentication Settings section.
  2. Click Add.
  1. When the LDAP settings appear, fill in the required fields.

For core features such as driver management, printer deployment, Role-based Access Control, and reporting, fill in the Domain Name and Primary LDAP server fields. The NETBIOS domain name and Base DN fields should fill in automatically. For both advanced modules (pull printing and mobile printing), fill in the entire top section down to the LDAP port field. Also fill in the Domain Alias, Bind User, and Bind Password fields. Make sure the LDAP Email Attributes are separated by commas. If you think you might need to use the SSO fields, please contact your PrinterLogic Customer Support Manager.

  1. When you have finished entering the LDAP settings, click Test Settings to ensure the settings are correct.
  2. Click Apply.
  3. (Optional) In the Default domain when logging in field, select the LDAP domain you have created.
  4. Click Enable Advanced LDAP administrative authentication.

When you click this option, the LDAP Group/User option becomes available in the Users section of the Admin Console as shown below.

  1. Click Save.

Step 2: Enable Workstation Pull Printing/Secure Release

  1. In the Admin Console click Tools > Settings > Printing.
  2. In the Pull Printing section, click Enable Workstation Pull Printing/Secure Printing.
  1. Click Save.

Step 3: Make Sure your Printer Object is Using the Default Settings

  1. Click Return to tree view.
  1. In the Admin Console tree view, select a printer object.
  2. Click the Printing tab.
  3. In the Pull Printing section, click Use default Pull Print setting.
  1. Click Save.

Step 4: Create the Virtual Pull Print Queue

NOTE: It is highly recommended that you do not select an existing printer object to be used as a pull print queue because printer objects in the tree view are frequently already tied to existing printers. Instead, create a new printer object and configure it as a pull printer.

  1. In the Admin Console tree view, select a folder to house the new printer object.
  1. Click New > TCP/IP Printer.
  2. Type the name of the printer object.
  3. Click Make Pull Printer.
  1. Click Upload New Driver for the operating system type being used.
  2. Click Browse.
  3. Select a universal printer driver.
  1. Click Open to select the driver.
  2. Click Next.
  1. Click Upload.
  2. Click Add.

When a printer object has been configured as a pull print queue, the Printing tab will appear as shown below.

Step 5: Configure a Printer Object to Release Pull Print Jobs

  1. In the Admin Console, click Tools > Settings > Printing.
  2. In the Pull Printing section, click Allow Pull Print jobs to be released to this printer.
  1. Click Save.

Step 6: Deploy the Pull Print Queue to a Workstation

There are two ways to deploy the pull print queue to a workstation. Either the end-user can install it manually from the Self-service Portal, or the administrator can deploy it from the Deploy tab.

Step 7: Release a Print Job from a Workstation

Click Print (in the application you are using) and select the pull print queue from the list of available printers. The print job will be sent to the queue. Open the Release Portal, click the print job, and select Release. Select the pull print-enabled printer and the print job will be sent to the designated printer.

Install the Control Panel Application

Secure Release Printing

The Control Panel Application is a feature that can be installed on a printer from Printer Installer. The purpose of the Control Panel Application is to enable end-users to release pull printing and release printing jobs directly from the printer's digital console. Specifically, you enable the Control Panel Application on a printer object in the tree view, and then you install the application on the printer.

The image below shows the PrinterLogic Release Portal as it appears following a successful login to the Control Panel Application

This interface is used for all printer manufacturers with the exception of HP, which has its own interface as shown below.

The print jobs that appear in the application are only the ones sent to queue by the person who logged in to the application. The print jobs in queue can be updated by clicking the Refresh button. Notice the options to release a single print job or all jobs in queue. Similarly, you can delete a single print job or delete all documents in queue.

Before you install the Control Panel Application, you must first enable SNMP Status Monitoring and then Enable LDAP Authentication. If you do not have LDAP enabled, you will receive an error.

The Control Panel Application uses two login options in the printer's digital display. The image below shows both of these options.

Login Option #1 is available by default. With this option, the end-user logs in to the Control Panel Application using their Active Directory credentials. The Control Panel Application provides a built-in keyboard for data entry. Login Option #2 is for badge swipe access to the application. This access type does not appear in the interface unless you have enabled badge swipe in the Admin Console.

If you install the Control Panel Application without the badge reader option enabled, you cannot just enable badge authorization on the printer object and automatically expect the Badge Authorization to appear on the application interface. Instead, you must uninstall the application from the printer, enable badge authorization on the printer object, and then re-install the Control Panel Application on the printer object.

Does the Control Panel Application support languages other than English?

The Control Panel Application will display text in the following languages:

  • French
  • Spanish
  • Portuguese
  • Japanese
  • Chinese
  • German
  • Dutch
  • Thai
  • Italian
  • Turkish

Card Reader Support: HP embedded readers only

Step1: Configure DNS Server Settings in the Printer Configuration Web Page

The printer's configuration web page varies by manufacturer. Just log in to the printer configuration web page and, at a minimum, add the IP address to the Primary DNS Server field and to the Secondary DNS Server field, if applicable. Save your changes and close the browser tab.

Step 2: Set up a Printer Object as a Release Printer

  1. In the Admin Console tree view, select the root folder or the parent folder you would like to hold the new printer object.
  2. Click New > TCP/IP Printer.
  1. When the Add TCP/IP Printer window appears, type the name of the printer.
  2. Enter the IP address or the hostname of the printer.
  1. Click the Upload New Driver button that corresponds to the driver type you want to upload.
  2. Click Browse.
  3. Navigate to the .INF file you want to upload and select it.
  1. Click Open.
  2. Click Next.
  3. Click Upload.
  1. When the driver has been uploaded, it will appear in the corresponding field.
  1. Click Add.

Step 3: Set up a Printer Object as a Virtual Pull Printer

  1. In the Admin Console tree view, select the folder that will hold the printer object.
  2. Click New > TCP/IP Printer.
  1. In the Printer Name field, type a name for your virtual pull print queue.
  2. Click Make Pull Printer.
  1. If you have an existing driver you would like to use, click the drop-down box for the operating system you are supporting and then select an existing driver from the Driver/Profile Repository.
  2. If you do not have any drivers uploaded that you would like to use, click Upload New Driver under the bit-type and operating system you want to support.
  3. Navigate to the .inf file you would like to upload.
  4. Click Add.

Step 4: Configure SNMP Settings

Option 1: Use Default Settings

  1. Log in to the Admin Console.
  2. In the tree view, select a printer object associated to the printer that will be sending alerts.

The printer object you select cannot already be set up to be a pull print queue. What this means is that the printer object you select must have the Make Pull Printer check box in the Printing tab deselected as shown below.

If you do not yet have any printer objects set up, select a folder in the tree view and then click New > TCP/IP Printer to create a new printer object. For more information, see Add a New Printer Object to the Tree View.

  1. Click the Alerts tab.
  2. Click Use Global.
  1. Click Save.
  2. Click the Status tab.
  3. Click Use Global.
  1. Click Save.

Option 2: Use Printer-specific Settings

  1. Log in to the Admin Console.
  2. In the tree view, select a printer object associated to the printer that will be sending alerts.

The printer object you select cannot already be set up to be a pull print queue. What this means is that the printer object you select must have the Make Pull Printer check box in the Printing tab deselected as shown below.

If you do not yet have any printer objects set up, select a folder in the tree view and then click New > TCP/IP Printer to create a new printer object. For more information, see Add a New Printer Object to the Tree View.

  1. Click the Alerts tab.
  2. Click Use Global.
  1. Click Save.
  2. Click the Status tab.
  3. Click Use Global.
  1. Click Save.

Step 5: Set Up An LDAP Domain

  1. In the Admin Console, click Tools > Settings > General.
  1. Navigate to the LDAP Authentication Settings section.
  2. Click Add.
  1. When the LDAP settings appear, fill in the required fields.

For core features such as driver management, printer deployment, Role-based Access Control, and reporting, fill in the Domain Name and Primary LDAP server fields. The NETBIOS domain name and Base DN fields should fill in automatically. For both advanced modules (pull printing and mobile printing), fill in the entire top section down to the LDAP port field. Also fill in the Domain Alias, Bind User, and Bind Password fields. Make sure the LDAP Email Attributes are separated by commas. If you think you might need to use the SSO fields, please contact your PrinterLogic Customer Support Manager.

  1. When you have finished entering the LDAP settings, click Test Settings to ensure the settings are correct.
  2. Click Apply.
  3. (Optional) In the Default domain when logging in field, select the LDAP domain you have created.
  4. Click Enable Advanced LDAP administrative authentication.

When you click this option, the LDAP Group/User option becomes available in the Users section of the Admin Console as shown below.

  1. Click Save.

Step 6: Enable Pull Printing

Option 1: Use Default Settings

  1. Click Return to tree view.
  1. In the Admin Console tree view, select a printer object.
  2. Click the Printing tab.
  3. In the Pull Printing section, click Use default Pull Print setting.
  1. Click Save.

Option 1: Use Printer-specific Settings

  1. Click Return to tree view.
  1. In the Admin Console tree view, select a printer object.
  2. Click the Printing tab.
  3. In the Pull Printing section, click Use Printer-specific Pull Print setting.
  1. Click Allow Pull Print jobs to be released to this printer.
  2. Click Save.

Step 7: Enable Secure Release Printing

Option 1: Use Default Settings

  1. In the Admin Console tree view, select a printer object.
  2. Click the Printing tab.
  3. In the Pull Printing section, click Use default Secure Release settings.
  1. Click Save.

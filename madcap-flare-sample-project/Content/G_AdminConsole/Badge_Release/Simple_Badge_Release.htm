Simple Badge Release

Secure Release Printing

With the Control Panel Application and a digital display on the printer, an end-user can easily log in to the application and release both pull printing and secure printing jobs. Yet, even without a digital display and an application, the end-user still has an option for releasing both pull printing and secure printing jobs. This option is called Simple Badge Release.

Simple Badge Release combines a small device (connected to the network) with a USB-connected badge scanner to enable an end-user to scan a badge and release a print job to IP-based printers only. The image to the left shows a supported Simple Badge Release device. Printer Installer currently supports RFIDeas model 241 (shown here) and Elatec TCPConv 2. While the device is required to be connected to the network, it does not need to be connected directly to or even near the printer.

The badge scanner will be connected to the printer, and (depending on the Simple Badge Release settings configured in the Admin Console) when the end-user swipes their badge, the printer will release either the last print job that end-user sent, or all print jobs in queue from that end-user. The image to the left shows the Simple Badge Release settings in Printer Installer.

With these settings, you not only enable Simple Badge Release, but you also select which device you are using and the IP address to connect to your network to the device. However, when you click Install Simple Badge Release Device, you are actually installing the Simple Badge Release settings on the device rather than "installing" the device.

Does Simple Badge Release work with all types of printers?

No. Simple Badge Release works with IP-based printers only.

What do I do if the print jobs are not being released or badges are not scanning correctly?

RFIDeas

Every printer in Printer Installer has its own printer ID. Within Printer Installer, these ID’s can be found by browsing to <PrinterInstallerServerURL>/tools/tests/tests.php and selecting “Check ID’s of Printers.”

The RFIDeas controller has a web portal that can be accessed by entering the IP Address of the controller box in a web browser.

To verify the correct configuration, select the server tab of the RFIDeas web portal. If you look in the Data Server Str: field you will see a section of the string that says “printer_id=…”

If the printer id is -1 then the controller is currently configured for badge registration. Any other value will be the printer id of the printer that the job is being released to and indicates that the controller is configured to release print jobs.

Elatec

When an Elatec device is programmed, its settings are stored in the database. Since the device connects at a socket level, you will be unable to tell it what information to send (such as a printer ID). So, store that information in the database and when an Eletec device connects to your socket server with badge information, then you can look up the device in the table and check from there whether the badge information is for release or for badge registration.

Can the Simple Badge Release Controller be used to register badges and release print jobs?

The Simple Badge Release Controller can be used only to register badges or release print jobs. It cannot perform both functions at the same time. If you want to switch functions, you will need to reinstall the controller.

What Active Directory pre-setup checks do I need to make for Simple Badge Release?
  1. Create a service account with read access to Active Directory. The service account does not need to be limited to the Printer Installer application. Some organizations use existing service accounts.
  2. Have the Active Directory username and password ready at the time of configuration.

NOTE: The Active Directory username and password are required only if badge information is stored in Active Directory.

How do I set up Simple Badge Release if I have multiple websites on my server?

The recommended best practice when installing Printer Installer is that you do not have other websites on the server where Printer Installer is set up. However, if your environment is set up such that you have multiple websites on the server, then follow the steps below to get Simple Badge Release to function properly.

  1. On the server where Printer Installer is set up, log in to the Administrative Tools.

If you do not know how to access the Administrative Tools, refer to the Microsoft documentation.

  1. Double-click the Internet Information Services (IIS) Manager to open it.
  1. In the left navigation bar, click the server name.
  1. Click Sites.
  2. Select the site you want to use as the Printer Installer website.
  1. Click Bindings.
  1. In the Site Bindings windows, select the desired protocol and then click Edit.
  1. In the IP address field, select the IP address of the website.
  1. Make sure the Host name field is left empty.
  2. Click OK.
  3. Click Close.
  4. Close the IIS Manager.
How do I use an Active Directory attribute to authenticate end-users to release print jobs?

In the Admin Console, click Tools > Settings > General. In the Control Panel Application section, enter the Active Directory attribute you are using for your stored badge IDs in Active Directory. The image to the below shows the field in which you add the attribute.

Step 1: Set up an LDAP Domain

  1. In the Admin Console, click Tools > Settings > General.
  1. Navigate to the LDAP Authentication Settings section.
  2. Click Add.
  1. When the LDAP settings appear, fill in the required fields.

For core features such as driver management, printer deployment, Role-based Access Control, and reporting, fill in the Domain Name and Primary LDAP server fields. The NETBIOS domain name and Base DN fields should fill in automatically. For both advanced modules (pull printing and mobile printing), fill in the entire top section down to the LDAP port field. Also fill in the Domain Alias, Bind User, and Bind Password fields. Make sure the LDAP Email Attributes are separated by commas. If you think you might need to use the SSO fields, please contact your PrinterLogic Customer Support Manager.

  1. When you have finished entering the LDAP settings, click Test Settings to ensure the settings are correct.
  2. Click Apply.
  3. (Optional) In the Default domain when logging in field, select the LDAP domain you have created.
  4. Click Enable Advanced LDAP administrative authentication.

When you click this option, the LDAP Group/User option becomes available in the Users section of the Admin Console as shown below.

  1. Click Save.

  1. In the Admin Console click Tools > Settings > Printing.
  2. In the Pull Printing section, click Enable Workstation Pull Printing/Secure Printing.
  1. Click Save.

Step 3: Set up the Virtual Pull Print Queue

  1. In the Admin Console tree view, select the folder that will hold the printer object.
  2. Click New > TCP/IP Printer.
  1. In the Printer Name field, type a name for your virtual pull print queue.
  2. Click Make Pull Printer.
  1. If you have an existing driver you would like to use, click the drop-down box for the operating system you are supporting and then select an existing driver from the Driver/Profile Repository.
  2. If you do not have any drivers uploaded that you would like to use, click Upload New Driver under the bit-type and operating system you want to support.
  3. Navigate to the .inf file you would like to upload.
  4. Click Add.

Step 4: Set up the Pull Print/Secure Release Print-enabled Printer

The steps shown here enable you to create the printer object associated to the printer that will release pull print jobs and secure release print jobs. This step is important because the end-user will print jobs on a specific printer (secure release) or on any pull print-enabled printer. However, please bear in mind that you do not have to configure the printer object for both pull printing and secure printing to be able to release a print job via Simple Badge Release. Rather, you can configure pull printing on the printer object but not secure release (or vice-versa) and then release the print job on the printer using Simple Badge Release.

NOTE: If you already have a printer object that you want to use as a pull print-enabled printer, select that printer object in the Admin Console tree view and then move to Step 5.

4A. Create the Printer Object

  1. In the Admin Console, click Return to tree view if you are not already in tree view mode.
  2. In the tree view, select a folder in which to store the printer object being used as the pull print-enabled printer.
  3. Click New > TCP/IP Printer.
  4. In the Printer Name field, type the name of the new printer object.
  5. In the IP Address or Hostname field, type the IP address or hostname of the printer that will be releasing pull print jobs.
  6. In the field corresponding to the operating system, click Upload New Driver and then follow the steps to upload the driver to the printer object.

4B. Configure the Printer Object for Pull Printing

Option 1: Use Default Settings

  1. Click Return to tree view.
  1. In the Admin Console tree view, select a printer object.
  2. Click the Printing tab.
  3. In the Pull Printing section, click Use default Pull Print setting.
  1. Click Save.

Option 2: Use Printer-specific Settings

  1. Click Return to tree view.
  1. In the Admin Console tree view, select a printer object.
  2. Click the Printing tab.
  3. In the Pull Printing section, click Use Printer-specific Pull Print setting.
  1. Click Allow Pull Print jobs to be released to this printer.
  2. Click Save.

Step 6: Set up the RFIDeas Controller

  1. Connect the controller to the network by inserting an Ethernet cable into the network port and then connecting the device to an active network connection.
  2. Connect the controller to a printer by inserting an Ethernet cable into the controller device port.
  3. Plug the card reader into the USB port.
  4. Plug in the controller.

When power is applied, all LEDs on the device will appear red. The LEDs next to the power cord and USB port will turn green when the unit is operational.

Step 7: Configure the Controller to Set a Static IP Address Using the Web Interface

For RFIDeas 241 to communicate with the printers and Printer Installer server on the network, the IP settings must be assigned to uniquely identify the specific device. The recommended process is to set a static IP address for your controller.

  1. Get the current IP address of the controller.
    1. Make sure the printer is connected to the controller's Ethernet port.
    2. Turn the controller on.
    3. Wait for the system LED to turn green.
    4. Use a small screwdriver, paperclip, or similar tool to press the recessed button on the side of the controller.
    5. The attached printer will print a job showing the current IP address of the controller along with other parameters.
  2. Set up the static address.
    1. Open a web browser.
    2. In the Address bar, type the IP address of the controller as the URL (for example, http://192.168.1.2).
    3. Select the IP page (if needed)
    4. Set the addressing mode to Static.
    5. Set the IP address to the desired value.
    6. Set up the subnet mask, gateway, and DNS addresses.
    7. Set the location to any readable text (as needed)
    8. Click Update.
    9. Click the Reboot button.

Step 8: Configure the Device to Manage Badges or to Authorize Print Jobs

8A. Register and Manage Badges

  1. In the Admin Console, click Tools > Badge Management.
  2. Click Register New.
  1. In the IP Address field, type the IP address of the Simple Badge Reader Device.
  1. Click Configure RFIDeas Ethernet 241.

The Printer Installer server will send the appropriate final configuration data to the controller, which will then reboot.

  1. Click Done.

Any badges registered through the controller will be stored in the Printer Installer database but not in Active Directory. When you scan a badge, the badge information will appear in the window as shown below.

At this point, you can enter the Active Directory details for the badge user in the Badge AD User Name field and then click Register the badge to check the Active Directory information entered against information in your Active Directory.

The format to use to enter badge information in the Badge ID Username field is domain name\username as shown below.

Additionally, a entry will be made in the Printer Installer server. Every time the end user scans their badge, Printer Installer checks its directory rather than checking Active Directory.

8B. (Optional) Install the Simple Badge Release Device

This step is necessary in two instances:

  • if you want to program the controller to release print jobs.
  • if the controller is currently programmed to register badges, but you want to reprogram it to release print jobs.
  1. In the Admin Console, click Return to tree view if you are not already in tree view mode.
  2. In the tree view, select the printer object you set up in Step 4A (the pull print/secure print-enabled printer).
  3. Click the Printing tab.
  4. In the Pull Printing: Simple Badge Release Device section, click Enable the use of the simple badge release device with this printer.
  1. Select Elatec.
  1. In the IP Address field, enter the controller's IP address.
  1. Enter the username and password of the simple badge release device.

It is recommended you just use the default username and password.

  1. In the TCP Client Configurations settings, the Device Port and Printer Port fields are read-only.
  2. Set the Disconnect timeout

A Disconnect timeout is provided to service card readers or Keypads with no predefined "End" character. In this case the converter will close the connection 10 seconds after a character is received. The timer would reset on each character int he data stream and only 10 seconds after the last character would it actually close the connection. The converter is default configured to disconnect 10 seconds after the last character but the user can increase or decrease this time to fit their environment.

  1. In the Which Print Jobs to release when a user's badge is swiped field, select the desired option.
  1. Click Configure Elatec TCPConv2.

At this point, the controller will reboot.

Step 9: Conclusion

After you have Simple Badge Release properly set up, you should be able to swipe the badge, and depending on the way in which you have Simple Badge Release configured, the printer will print either all print jobs or the last sent print job associated to the card owner.

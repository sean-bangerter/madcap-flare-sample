Email Printing

Email to Print enables an end-user to forward an email (from a smartphone, tablet, or laptop) to a printer that has been designated specifically for printing from email. "Email Print" is used as an umbrella term to identify all three email printing types (Email to Print, Direct Email to Print, and Direct Email Guest to Print) as described below.

Email Printing forwards an email to a specific email address using Active Directory to validate the end-user. The forwarded email is sent to a universal print queue that enables the user to release the print job the same way as if they were releasing a pull print job. The end-user would open the Release Portal or the Embedded Control Panel application (if it is installed on the printer) to release the print job. Email to Print is the only option that works in this way. The other two options do not wait for the end-user to release the print job. Rather, the printer immediately releases the print job.

Direct Email Printing forwards an email to a specific email address, and Active Directory determines if the end-user's email address is authorized to release to the printer. If Active Directory approves the end-user's email address, then the print job is immediately released to the printer.

Direct Email Guest Printing is an option offered if Direct Email to Print is enabled but Active Directory does not authorize the end-user to release the print job to the designated printer. In this case, the end-user is considered a "guest" and can release a print job to the printer, but they are limited to a specific number of pages that can be released per day or to black and white printing only.

NOTE: Direct Email to Print and Direct Email Guest to Print both require LDAP settings to connect to Active Directory. For more information, see Lightweight Directory Access Protocol.

The settings for the Email to Print options are available in the Admin Console. Email printing can be set up in Tools > Settings > Printing, and must be enabled for all types of Email to Print. In other words, if you want to use Email to Print, Direct Email to Print, or Direct Email Guest to Print, you must enable Email to Print.

Direct Email to Print and Direct Email Guest to Print provides both default settings or printer-specific settings in the Printing tab. Moreover, if you select the default settings to be used, then you configure those settings in Tools > Settings > Printing under Mobile Printing: Direct Email Printing. If you are using Google to manage your email boxes, contact your PrinterLogic representative for instructions and information.

If you are setting up Direct Email Printing or Direct Email Guest Printing, you will need to configure email routing rules. Each direct email printer will have its own email address assigned to it. When a user sends an email to that email address, the email needs to be routed to the primary email printing inbox, which can be done in a variety of ways, but the details of each implementation will vary depending on the email server platform being used. The following items are three possible options for setting up routing to a primary email printing mailbox.

What formats are supported for email attachments sent to a printer?

The formats supported for email attachments depends on whether or not you have LibreOffice installed on your s erver.

NOTE: LibreOffice is not required per se, but if you do not install it, the file formats allowed will be significantly limited.

Without LibreOffice Installed

.pdf, .txt, .html, .htm, .jpg, .gif, .png, .bmp

With LibreOffice Installed

Document: .pdf, .doc, .dot, .docx, .dotx, .odt

Spreadsheet: .xls, sxlsx, .xlt, .xltx, .xlsm, .ods

Presentation: .ppt, .pptx, .pot, .potx, .odp

Text: .txt, .rtf

HTML: .html, .htm

Images: .jpg, .jpeg, .gif, .png, .tiff, .tif, .bmp

What are the mailbox requirements for setting up a mailbox for Email to Print?

The following requirements are necessary for setting up a mailbox for Email to Print.

  • The new email account cannot be a shared mailbox.
  • IMAP must be enabled for the mailbox.
  • To configure Email to Print, you will need to know the mailbox username, password, and IMAP/SMTP connection information. (POP3 is currently not supported)
  • If using Direct Email Printing, email routing rules on the mail server must be configured to route mail sent to the direct email addresses of the printers to this mailbox.

Step 1: Prerequisites

These are configuration steps not completed in the Admin Console.

  1. Set up a new email account.
  1. IMAP must be enabled for the mailbox.
  2. To configure the feature, you must know the mailbox username, password, and IMAP/SMTP connection information.
  1. Install LibreOffice on the Printer Installer server.
  2. Store each user's email address as an attribute of the user's Active Directory user object.

Step 2: Create a Virtual Pull Print Queue in the Admin Console

  1. In the Admin Console tree view, select the folder that will hold the printer object.
  2. Click New > TCP/IP Printer.
  1. In the Printer Name field, type a name for your virtual pull print queue.
  2. Click Make Pull Printer.
  1. If you have an existing driver you would like to use, click the drop-down box for the operating system you are supporting and then select an existing driver from the Driver/Profile Repository.
  2. If you do not have any drivers uploaded that you would like to use, click Upload New Driver under the bit-type and operating system you want to support.
  3. Navigate to the .inf file you would like to upload.
  4. Click Add.

Step 3: Configure the Virtual Pull Print Queue to Process Mobile Print Jobs

  1. In the Admin Console, click Tools > Settings > Printing.
  2. Scroll to the Mobile Printing section.
  3. Click None Selected.
  1. Select a printer object that has been configured as a virtual pull print queue.

Virtual pull print queue printer objects have Make Pull Printer selected in the Printing tab as shown below.

  1. Click Select.
  2. Click Save.

Step 4: Set up your LDAP Domain

  1. In the Admin Console, click Tools > Settings > General.
  1. Navigate to the LDAP Authentication Settings section.
  2. Click Add.
  1. When the LDAP settings appear, fill in the required fields.

For core features such as driver management, printer deployment, Role-based Access Control, and reporting, fill in the Domain Name and Primary LDAP server fields. The NETBIOS domain name and Base DN fields should fill in automatically. For both advanced modules (pull printing and mobile printing), fill in the entire top section down to the LDAP port field. Also fill in the Domain Alias, Bind User, and Bind Password fields. Make sure the LDAP Email Attributes are separated by commas. If you think you might need to use the SSO fields, please contact your PrinterLogic Customer Support Manager.

  1. When you have finished entering the LDAP settings, click Test Settings to ensure the settings are correct.
  2. Click Apply.
  3. (Optional) In the Default domain when logging in field, select the LDAP domain you have created.
  4. Click Enable Advanced LDAP administrative authentication.

When you click this option, the LDAP Group/User option becomes available in the Users section of the Admin Console as shown below.

  1. Click Save.

Step 5: Configure Email Processing (IMAP and SMTP) Settings

  1. In the Admin Console, click Tools > Settings > Printing.
  2. Scroll to the Mobile Printing: Email Processing section.
  3. Fill in the fields of the Incoming Mail (IMAP) Settings section and the Outgoing Mail (SMTP) section as shown below.

For more information, see Printing Settings Cheat Sheet.

  1. Click Office-type File Attachment Printing if you want to enable end-users to print attachments.
  2. Type the path required to access the LibreOffice executable.
  3. Click Save.
  4. Click Test Executable if you want Printer Installer to validate whether your LibreOffice executable is functioning correctly.

Step 6: Enable Email Printing

  1. In the Admin Console, click Tools > Settings > Printing.
  2. In the Mobile Printing: Email Printing section, click Enable Email Printing.
  3. In the Pull Printing Address field, type the email address the end-users will enter to send the email to the print queue.
  1. Click Save.

Step 7: Release an Email Print Job to the Pull Print Queue

After you have set up Email Printing, send a print job to the email address you designated for Email Printing. This is the email address you set up in the Prerequisites, and the Email Address you entered in Tools > Settings > Printing under the Mobile Printing: Email Printing section. The print job will be sent to the pull print queue. You can then release the print job on any printer that has both pull printing and mobile printing enabled. You can then release the print job online through the Release Portal, or at the printer through Simple Badge Release, a Release Station, or the Embedded Control Panel Application.

Secure Release Printing

Secure Release Printing enables you to submit a print job to a virtual queue that "holds" the print job on your workstation until you release it to a printer. If this sounds like Pull Printing, it is with one major exception. Pull Printing jobs can be released on any pull print-enabled printer, but Secure Release Printing jobs can be released on one specific printer only.

How long should it take for a secure print or pull print job to print?

Time to print depends on a number of factors, including size of the job and the type of printer, which might control how long it takes to start printing. However, after you click Release, expect the print job to be sent to the printer within 15 seconds. Mobile printing can be a little different since all mobile print jobs are sent, sequentially, by a central computer, which can be slower.

For example, your Human Resources Manager regularly prints personal employee information. She requests a personal printer in her office even though a shared printer sits right outside her office. While the request seems like a good one, you would rather not set the precedent of buying personal printers. The other option is to set the HR Manager up with Secure Release Printing.

When she sends a print job to the pull print queue, the job will remain on her workstation until she releases the print job to the printer outside her office using one of the release mechanisms such as the Release Portal or the Control Panel Application. So, as you can see, the HR Manager gets the privacy she needs and you do not need to buy a printer.

Step 1: Set up an LDAP Domain in the Admin Console

  1. In the Admin Console, click Tools > Settings > General.
  1. Navigate to the LDAP Authentication Settings section.
  2. Click Add.
  1. When the LDAP settings appear, fill in the required fields.

For core features such as driver management, printer deployment, Role-based Access Control, and reporting, fill in the Domain Name and Primary LDAP server fields. The NETBIOS domain name and Base DN fields should fill in automatically. For both advanced modules (pull printing and mobile printing), fill in the entire top section down to the LDAP port field. Also fill in the Domain Alias, Bind User, and Bind Password fields. Make sure the LDAP Email Attributes are separated by commas. If you think you might need to use the SSO fields, please contact your PrinterLogic Customer Support Manager.

  1. When you have finished entering the LDAP settings, click Test Settings to ensure the settings are correct.
  2. Click Apply.
  3. (Optional) In the Default domain when logging in field, select the LDAP domain you have created.
  4. Click Enable Advanced LDAP administrative authentication.

When you click this option, the LDAP Group/User option becomes available in the Users section of the Admin Console as shown below.

  1. Click Save.

Step 2: Enable Workstation Pull Printing/Secure Release Printing

  1. In the Admin Console, click Tools > Settings > Printing.
  2. In the Pull Printing section, click Enable Workstation Pull Printing/Secure Printing.
  1. Click Save.

Step 3: Configure Default Secure Release Settings

  1. In the Admin Console, click Tools > Settings > Printing.
  2. In the Default per-printer Secure Release settings field, select Yes (Always) or Yes (Prompt).

On the end-user's workstation, the Yes (Always) option automatically releases the print job to the designated printer, whereas the Yes (Prompt) option displays the following window after the end-user prints a document but before the printer releases the document.

  1. (Optional) Click Allow End-users to Override.

If you select Allow End-Users to Override in the Admin Console, then the end-user can click the Add Printer icon in the Windows taskbar, and in the Secure Print Settings window select whether specific printers can release secure print jobs or not, as shown in the following image.

  1. Click Save.
  2. Click Return to tree view.

Step 4: Configure a Printer Object to use Default Secure Release Settings

  1. In the Admin Console tree view, select a printer object.
  2. Click the Printing tab.
  3. In the Pull Printing section, click Use default Secure Release settings.
  1. Click Save.

Step 5: Release a Secure Release Print Job

Secure print jobs initiated from a workstation will remain on the workstation until the end-user releases the print jobs from the Release Portal or through the Control Panel Application.

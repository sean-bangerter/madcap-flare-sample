Printing in a Virtual Environment

Printing in Virtual Environments means printing in Citrix, VMWare, or MS Terminal Services. For Citrix, Printer Installer supports XenApp and XenDesktop. For VMWare, Printer Installer supports VDI. For Microsoft Terminal Services, Printer Installer supports VDI, remote desktops, and remote apps.

There are a few ways to set up printing in a virtual environment. These options include:

  • For XenApp and Microsoft Terminal Services, Printer Installer is set up in two ways. Install the client on the end-point device and then redirect the printer into Citrix. The other option is to install the client on the XenApp server and configure the client for RDS mode.
  • For XenDesktop, VMWare, and Microsoft VDI, just install the client on your golden (parent) image.

Redirected Printers

What are redirected printers?

Redirected printers, also known as auto-created printers, are created in a Citrix session based on the printers that are installed on the end-user’s computer.

Why redirect my printers into Citrix?

When a user prints to a redirected printer in their Citrix session, the Citrix Universal Print Driver will send the unrendered print job over the compressed ICA channel to the client/endpoint where the print job is spooled and sent to the printer. This is less intensive on the network and requires less processing for the Citrix server.

  1. Install the client on the end-point/client machine. For more information, see Install Printer Installer Client.
  2. After the client is installed on the end-point machine, create a Citrix policy to redirect the printers into Citrix.

NOTE: The policy can redirect all printers into the session or the default printer only.

Session Printers

What are session printers?

Session printers are printers that get installed directly on the XenApp server. All users logged into this server will share the printers. All print jobs are spooled and rendered from the server.

When the client is configured in RDS mode, it will allow users to only see printers they have installed from the Self Service Portal or that have been deployed to them. When a user installs a new printer, the permissions are adjusted for that user who now can see that printer in their session.

  1. Install the Printer Installer Client on the XenApp server.
  2. Configure the client for RDS mode
  1. Install the client on the RDS server.
  2. Activate the client's Remote Desktop Services mode by creating a new String Value named TerminalServices with a value of 1 into the following registry key:

On a 64-bit RDS Server: HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node\PPP\PrinterInstaller

On a 32-bit RDS Server: HKEY_LOCAL_MACHINE\SOFTWARE\PPP\PrinterInstaller

  1. Open Task Manager.
  2. Right click PrinterInstallerClientLauncher.exe and select End Process Tree.
  3. Open the Start menu and run Printer Installer > Administration > Start Client.
  1. Install printers in XenApp
  1. Publish the Printer Installer Client (Published Applications).
  2. Configure Deployments in the Printer Installer Admin Console (Published Applications/Desktops).
  3. Navigate to the Self Service Portal and install printers (Published Desktops).

Citrix Universal Print Driver

When selecting the Citrix Universal Printer this tells Printer Installer to use the Citrix Print Driver when installing the printer. This driver should already be on the Citrix server.

Why redirect my printers into Microsoft Terminal Services?

When redirecting your printer into Microsoft Terminal Services, the job is printed from the client/endpoint machine. The Microsoft RDS Universal Print Driver will send the unrendered print job over the RDP channel to the client/endpoint where the print job is spooled and sent to the printer. This is less intensive on the network and less processing for the Terminal server.

  2. After the client is installed on the end-point machine, create a Microsoft Server policy to redirect the printers.

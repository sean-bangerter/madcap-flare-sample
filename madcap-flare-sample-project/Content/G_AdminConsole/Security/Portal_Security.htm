Portal Security

Secure Release Printing

The Portal Security settings in the Admin Console enable you to show or hide folders or printer objects in the Self-service Portal tree view (and map) based on who is logged into the portal. To use a simple example, let's say you purchase a new printer for the specific purpose of printing corporate checks. To prevent unwanted printing on the new printer, you want only three specific employees (George, Mary, and Shelley) to be able to see the printer in the Self-service Portal and then download drivers for the new printer.

The Portal Security settings are found in the Security tab menu, as shown in the image to the left, and enable you to assign those users (George, Mary, and Shelley) to the folder in which the printer object resides or to the printer object itself. Take note that there are several ways in which you can configure the Portal Security settings so that George, Mary, and Shelley can download the printer drivers for the new printer.

You can use Active Directory objects such as users, groups, computer, etc. to authenticate a user. For example, let's say George, Mary, and Shelley all have Active Directory accounts within your environment. In the Portal Security settings section, click Add and select the three users from Active Directory. If you do not want to use Active Directory objects, you can simply create a Printer Installer object such as an IP address range, which can be used to authenticate a workstation's IP address to determine if the printer driver can be downloaded on that workstation. If, for example, George's workstation has an IP address that falls within the range, then George will be able to open the Self-service Portal and install the drivers for the new printer.

Include Inherited Security

Of particular importance in the Portal Security tab is the Include Inherited Security option as shown in the image to the right. This option enables you to automatically take the permissions set on the parent folder and automatically apply them to a subfolder or a printer object stored within the parent folder. By default, new folders and printer objects have Include inherited Security selected automatically.

NOTE: Inherited Security settings not properly configured can have a tremendous impact on Printer Installer, so make sure you understand how they work before implementing them.

To put this in perspective, take a closer look at the image to the right. For this example, let's say that your organization has three locations with printers in Northern California. As such, your Admin Console tree view shows a parent folder named California, which contains a subfolder named North, which contains three subfolders named, respectively, Palo Alto, Sacramento, and San Francisco for the cities in which the printers are located. Furthermore, the San Francisco folder has one printer object named Printer A. The structure for such a hierarchy can be viewed in the tree view of the sample image.

If you set an object on the root level of the tree, any folders or printer objects that reside on the root level and have Include Inherited Security selected will automatically assume the objects set at the root level. For example, let's say that in the Admin Console, you select the root level of the tree view (default is My Company), and in the Portal Security section of the Security tab, you add Group 1, which consists of User 1, User 2, and User 3. Because you added the group at the root level, any folders directly under My Company that have Include Inherited Security selection will also have Group 1 added to it. So, if you add Folder A and Folder B under My Company, and Folder A has Include Inherited Security selected but Folder B does not. This configuration means that Folder A will assume Group 1, which has been set at its parent level. Folder B will not assume the Group 1 object, which means that the Active Directory or Printer Installer object(s) will need to be set on Folder B itself. The objects set at the root level will continue to cascade into child folders and printer objects until you deselect Include Inherited Security in a folder.

Furthermore, Portal Security contains the Assignment Type column, which identifies whether an object is set at the selected level or is inherited from a parent level. If the Assignment Type column displays Explicit then the object was created explicitly for the selected folder or printer object in the Admin Console tree view. If the Assignment Type column displays Inherited then the object was created on the parent folder, or the parent folder also inherited the object from its parent folder.

For example, you select the San Francisco folder in the Admin Console tree view, and then you click Security > Portal Security to see three objects - a user, a group, and a computer (all Active Directory objects), where the group object displays Explicit in the Assignment Type field, which means that whoever created the group did so by clicking Add in Portal Security and creating the object for that folder. If the group object displays Inherited then the object was created on the selected folder's parent folder, and Include Inherited Security is selected for the San Francisco folder.

NOTE: Deselecting Include Inherited Security at a higher level in the tree view will automatically remove the inherited objects in items further down the tree view. As such, please be careful 1) when using Include Inherited Security, and 2) when adding Everyone as an Active Directory object to Portal Security. For example, if you added Everyone at the My Company root level, then all folders and printer objects that have Include Inherited Security selected will also assume Everyone, which will give everyone access to everything, which you most likely do not want.

What is the difference between Hide from Portal and Portal Security?

The Hide from Portal setting in the General tab (as shown below) is used to hide a printer from all users in the Self-service Portal.

This feature is generally used for testing printers in development prior to pushing them to a production environment.

The Security tab contains the Portal Security settings, which are used to give Self-service Portal access for a specific printer object to certain users, computers, etc. as shown below.

The Portal Security settings enable you to configure Printer Installer to show or hide a folder or printer object in the tree view to a specific group of users or workstations. Hide from Portal tends to be used temporarily while Portal Security is more for long-term use.

What does Include Inherited Security do?

When you set portal security permissions on a folder or printer object in the Admin Console tree view, you can click the Include Inherited Security checkbox to use the parent folder's security permissions. For example, if you have a printer object named HP 700 Color that is housed within the Floor 34 folder, and the folder has permissions for a group, then those same permissions are applied to the HP 700 Color printer, which means any end-users added to the group will have access to the HP 700 Color printer in the Self-service Portal.

Similarly, if you deselect Include Inherited Security, then the group permissions applied to the Floor 34 folder are no longer applied to the HP 700 Color printer. However, this also means that you need to directly assign permissions to the printer itself. Otherwise, the printer will not be available in the Self-service Portal to end-users. In other words, if you apply the same group that was assigned to the Floor 34 folder, then the users in that group will be able to see the printer in the Self-service Portal.

Keep in mind that if the parent folder has Everyone set as the security level, then all items within the folder will also have Everyone set to their security levels as well. As such, if you want to set different security levels to different items in the folder, then you will not want to have Everyone set as the level, so you will want to deselect Include Inherited Security.

Additionally, an object name in the list of Portal Security items indicates whether an object name is inheriting the permissions of the parent or not. The Association Type column shows either Inherited or Explicit. The reason for this column is that a folder or printer object can be assigned both inherited permissions and other permissions that apply specifically to the printer object or folder. For example, if the parent folder is associated to a group with 50 users, and you want all the members of that group to have access to a printer object in that folder. However, you have three users that you do not want to have access to the group, but you do want them to have access to the printer object. Rather than creating a separate group with all 53 users, you can provide inherited permissions from the parent folder, and then assign the other three users to the printer object.

  1. In the Admin Console, select either a folder or a printer object from the tree view.
  2. Click Security > Portal Security.
  1. In the tree view, select the folder or printer object to which you would like to provide access for the end-user.
  1. Click Add.
  2. Select the object that you want to apply to the selected folder or printer object.
  1. Option 1: User, Computer, or Group

If you click Active Directory > User, Computer, or Group, then the Select Users, Computers, or Groups window will appear as shown below.

After you have selected the user, computer, or group, click OK.

  1. Option 2: Container or OU

If you click Active Directory > Container or OU, then the Select Containers window will open as shown below.

After you have selected the container or organizational unit, click OK.

  1. Option 3: IP Address Range

If you click IP Address Range, then the Add IP Address Range window will appear as shown below.

Click Search to type the IP address range, or click Browse to navigate the tree view to find the IP address range you would like to add. When you have selected the IP address range, click Add.

  1. Option 4: Hostname

If you click Hostname, then the corresponding window (respectively) will appear as shown below.

Type the hostname and then click Add.

  1. Option 4: MAC Address

If you click MAC Address, then the corresponding window (respectively) will appear as shown below.

Type the MAC address and then click Add.

  1. Click Save.

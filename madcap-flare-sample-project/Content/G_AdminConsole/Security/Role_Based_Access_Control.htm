Role-based Access Control

Secure Release Printing

Role-based Access Control (RBAC) enables you to assign both pre-packaged (standard) and user-defined (custom) roles to other Admin Console users. These roles both provide and prohibit access to features and functions of the Admin Console. For example, the administrator could provide an end-user with access to the tree view but not to the options in the Tools menu. Any prohibited features will not display in the user's Admin Console interface. In other words, the user will not be able to see features that have been disabled.

Role-based Access Control is available from the Tools > Users menu in the Admin Console. With the combined features of Users and Roles, you can create or edit user accounts and set up, apply, or change role assignments. The image below shows the combined Users and Roles sections in the Admin Console.

The reason these options are placed together is that when you create a user account, you can immediately select the features and functionality that the user can access or even see. In the Roles section, you will notice the columns for Role Name, Description, and Type. The default roles display Standard in the Type column. Because these role types are preferred, you can view but not edit them. Any roles that you create will show Custom in the Type column. You can both view and edit these roles. When you create a role, the Printer Installer gives it the name Untitled Role until you rename it.

Assigning roles to users is not done in the Roles section. Rather, it is done in the Users section, which shows Username, Role Type, and Folder in the columns. The role listed in the Role Type column identifies the role that is assigned to that user. Root Account is the role given to the initial administrator. Other users given administrator roles will show Administrator rather than Root Account. Keep in mind that you cannot change the root account.

The Folder column shows the tree view folder or printer on which the role applies. For example, in the Admin Console, one user can have administrative rights on a specific tree view folder while having non-administrative rights on a different folder or printer. For any overlapping permissions between roles, Printer Installer will automatically select the highest permission for that tree view folder.

Step 1: Set up an LDAP Domain

  1. In the Admin Console, click Tools > Settings > General.
  1. Navigate to the LDAP Authentication Settings section.
  2. Click Add.
  1. When the LDAP settings appear, fill in the required fields.

For core features such as driver management, printer deployment, Role-based Access Control, and reporting, fill in the Domain Name and Primary LDAP server fields. The NETBIOS domain name and Base DN fields should fill in automatically. For both advanced modules (pull printing and mobile printing), fill in the entire top section down to the LDAP port field. Also fill in the Domain Alias, Bind User, and Bind Password fields. Make sure the LDAP Email Attributes are separated by commas. If you think you might need to use the SSO fields, please contact your PrinterLogic Customer Support Manager.

  1. When you have finished entering the LDAP settings, click Test Settings to ensure the settings are correct.
  2. Click Apply.
  3. (Optional) In the Default domain when logging in field, select the LDAP domain you have created.
  4. Click Enable Advanced LDAP administrative authentication.

When you click this option, the LDAP Group/User option becomes available in the Users section of the Admin Console as shown below.

  1. Click Save.

Step 2: Add a New User

Active Directory Admin User

  1. In the Admin Console, click Tools > Users.
  1. In the Users section, click Add User.
  1. Select AD Group/User.

NOTE: The LDAP Group/User option does not appear unless you have set up an LDAP domain in the Admin Console.

  1. Navigate to the Active Directory user account.
  1. Enter a full or even a partial Active Directory account name.
  2. Click Check Names.

If only one Active Directory account with the name you have provided exists, then that name will be selected. If multiple names exist, then the Multiple Names Found window will appear, so you can select the name you want to use.

  1. If multiple names are found, select the name you want to use.

You can select multiple names by holding the <CTRL> button on your keyboard.

  1. Click OK.
  2. When the Active Directory user account appears, click OK.
  3. When the Configure User's Role Assignments window appears, click Administrator.
  1. Click OK.

The new user account will appear in the Users section as shown below.

  1. (Optional) Follow these steps again for any other users you would like to add.

LDAP Admin User

  1. In the Admin Console, click Tools > Users.
  1. In the Users section, click Add User.
  1. Select LDAP Group/User.

NOTE: The AD Group/User option does not appear unless you have set up an LDAP domain in the Admin Console.

  1. In the Location field, select either one domain or all domains in which to search for a user or a group.
  1. Enter a partial or whole name of a user or group in the designated domain(s).
  2. Click Search.

By default, all of the users and groups found will be selected, but you can deselect any user or group from the list.

  1. Click Select.
  2. The users and groups will be added to the Users section of the Admin Console.

Printer Installer Admin User

  1. In the Admin Console, click Tools > Users.
  1. In the Users section, click Add User.
  1. Select Printer Installer User.
  1. In the Add User window, enter the required information for the user account.
  1. Click Add.
  2. When the Configure User's Role Assignments window appears, click Administrator.
  1. Click OK.

The new user account will appear in the Users section as shown below.

  1. (Optional) Follow these steps again for any other users you would like to add.

Step 3: Add a New Role

  1. From the list of available users, select the user to which you would like to assign a role.
  2. Click Add Role.
  1. In the Role Name field, type a name to replace the default name, Untitled Role.
  1. (Optional) In the Role Description field, type a summary to define the purpose of the new role you are adding.
  2. Click the check box to the left of the features you would like the user to be able to access.

You may need to drill down to find specific features because selecting a parent level folder provides access to all the features under it. Additionally, selecting an option in the Permissions column displays a description in the column to the right.

  1. Click Save.

When you are finished creating the role, you will see the role appear in the list of existing roles as a Custom role type as shown below.

Step 4: Assign a Role to a User

  1. In the Users section of the Admin Console, select the user for which you would like to make the role assignment changes.
  1. Click Change Role Assignments.
  1. Select Administrator or Non-Administrator.
  2. If you selected Administrator, click OK. If you selected Non-Administrator, click Add.
  1. Select the tree view folder or printer object to which you are applying the role.
  2. Click Add.

The selected folder will appear as a line item with an option to apply a role to it.

  1. In the Role field, select the role you want to apply to the selected folder.
  1. Follow Steps 6 and 7 for every folder to which you want to add a role.
  2. Click OK.

Conclusion

The end-user who logs in to Printer Installer using the credentials of the user account you created will have administrative access to all features or non-administrative access to specific features depending on the permissions you have set for the account.

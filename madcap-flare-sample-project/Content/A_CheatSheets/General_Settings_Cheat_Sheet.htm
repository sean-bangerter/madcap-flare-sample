General Settings Cheat Sheet

General Settings appear in the Admin Console when you click Tools > Settings > General. General Settings are used to engage or configure some of the features of Printer Installer. For example, the Operating Systems Supported section displays the operating systems that the Printer Installer client will support. If you deselect one of the operating systems, then that operating system will not appear in the Drivers/Profiles tab or Driver/Profile Repository.

The settings of the General section can have a significant impact across all parts of Printer Installer. For example, the LDAP settings enable Printer Installer to work with Active Directory for such activities as printer deployment to workstations, access to the Printer Installer portal, badge authentication, email printing, etc. Read More...

Certain settings are used by printer objects that are configured to use default settings. For example, let's say you select a printer object in the tree view and then you click the Status tab. If the Use Global option is selected, then the settings in the SNMP Status Monitoring section of the General Settings apply to the printer object. Similarly, the SNMP Alerts section is used for printer objects that are configured to use default SNMP Alerts settings (in the Alerts tab).

Furthermore, the General Settings contain options that are more simple, such as Custom Printer Fields, which adds custom fields to the General tab. These custom fields can be used for a variety of reasons. Make sure you save any changes before leaving the General Settings. The fields of the General Settings are described in detail below.

Cache

This section enables you to configure the server you will be using as your Cache Manager. For more information, see Cache Drivers.

Enable end-user redirections to cache

Click this check box to stop using the Cache Locations without actually disabling the Cache Manager synchronization process. This option is generally used when you are just getting started with Printer Installer. You might have several GB of data that needs to be pushed out to the Cache Locations, you can deselect this option to set up everything before you are ready to push to the Cache Locations. Similarly, if you are having an issue with the Cache Manager, you can turn off the Cache Locations until you resolve the issue.

Cache Manager

The Cache Manager is a computer on your network that feeds drivers to the various Cache Locations. For Printer Installer, it is recommended that the Cache Manager is the same server on which Printer Installer is set up.

Protocol

Select the protocol your end-users will enter to access the portal (including the client). Options include Only HTTP, Only HTTPS, or Any. HTTPS will only appear as an option if you are currently logged into the Admin Console using HTTPS to ensure that HTTPS is already set up and working on the server.

Server Mode

Select Enable Epic print server mode if you are using Printer Installer to manage Epic print servers (no desktop management). You can globally enable the epic print server mode on all clients without needing to perform the registry key override.

Tree Management

Selecting this option prevents duplicate names of printer object in the tree view, even if they are in separate folders. Additionally, if you already have two printer objects with the same name in the tree view, then you will not be able to select Require Unique TCP/IP Printer Names.

OM Plus

NOTE: This section appears only if your Printer Installer license contains support for OM Plus. For more information, see your PrinterLogic sales representative.

Any OM Plus servers added to this section, coupled with the credentials you add will be applied to OM Plus printer objects that are configured to use the default OM Plus settings in the Port tab as shown below.

The credentials you enter in the Username and Password fields must have permissions to create print queues on all of the OM Plus servers you add to this section. For more information, see OM Plus Printing.

LDAP Authentication Settings

This section enables you to set up LDAP domains, which Printer Installer uses for authenticating users, deploying printers, and for various types of release and mobile printing actions that require it such as Pull Printing, Direct Email Printing, and Direct Email Guest Printing.

Add/Modify/Remove

The Modify and Remove options will become available only after you select an existing LDAP domain from those available.

When you click Add or Modify, you will see the following window that will enable you to set up a new LDAP domain.

When you are done inputting your settings, click Test Settings. A successful test will show all items in green as shown.

Default domain when logging in

The Default Domain is the domain that will be used if an end-user attempts to log in with only a username and does not specify a domain. For example, if the default domain in abccorp.local, and you log in as john.smith, then PrinterCloud will attempt to log you in as abccorp.local\john.smith.

However, if a second domain is configured, for example xyzcorp, then the end-user would still have to type in that non-default domain followed by the username. For example, xyzcorp\john.smith.

Enable advanced LDAP administrative authentication

When you click this option, the Add User button (Tools > Users) will display LDAP Group/User as an additional option.

This option enables you to add users or groups without the Active Directory Windows interface; i.e., Mac, Linux, or Windows PCs not joined to the domain.

Control Panel Application

These options apply only if you select Use default printer administration credentials in the Printing tab.

Username

The Username field is used to log in to the printer's web interface so you can install the Embedded Control Panel application on the printer associated to the designated printer object only.

Password

The Password field is used to access the printer's web interface to install the Embedded Control Panel application on the printer associated to the designated printer object only.

Active Directory user object attribute containing the badge ID for Console Release Badge Authentication

Use this option to enter an attribute in this field if you are using Active Directory (or LDAP) to store your badge information. If this field is left empty, then the badge information is stored in the Printer Installer database.

Default Single Sign On settings

This option is used only when you select Use default Single Sign On settings in the Printing tab to enable or disable Single Sign-on for the printer associated with the selected printer object in the tree view.

Use HTTPS for all Control Panel Applications

This setting is used for protection against a potential external threat who gains access to your network.

Upload Certificate

This option enables you to upload the CA certificate file to be used as a self-signed certificate for an HP printer.

HP Control Panel Technology

This field enables you to manually select the Simple Badge Release device that will be used with Printer Installer. This setting applies only when you are using an HP printer with an HP badge reader.

Simple Badge Release Settings

The RFID's will respond with the decimal and hex formats if they are available. Some readers are programmed to only do one or the other. If set to default, the system will check to see if there is a decimal format returned. If there is, it will use that. If there isn't, then it'll use the hex format

Settings for RFIDeas Ethernet 241 devices

RFIDeas 241 devices return badge information in both decimal and hexadecimal formats, so you can use this setting to select the format that Printer Installer will use.

Settings for Elatec TCPConv2 devices

This option is automatically enabled when you install and configure an Elatec Simple Badge Release device. However, you can disable this option in cases where you would like to reserve resources, for example, debugging.

Custom Printer Fields

This setting adds a custom field to the General tab as shown below.

The field will appear on the General tab only when you select a printer object in the tree view. It does not apear when you select a folder or IP address range. The field can be text, numeric, or yes/no (true/false).

Enabled

This field must be selected for the custom field to appear in the General tab. If you deselect this option, then the field will not appear.

Name

Unique name you will give to the field. This name will appear next to the custom field in the General tab.

Type

Select the type of value for which the field will be used. Options include text, numeric, or yes/no.

Enable SNMP Status Monitoring

This option enables SNMP status monitoring so that Printer Installer can connect with the printer and request information that is displayed in the Status tab. If you do not have SNMP Status Monitoring enabled, then you will see the following

Under normal conditions

Under normal conditions, the number in this field is the minutes between each attempt to connect to the printer and to receive a status update.

Under error conditions

When an error occurs, the number in this field is the seconds between each attempt to connect to the printer and to receive a status update.

SNMP v1/v2
Community Public (Get) Name

Access name provided to the printer to retrieve SNMP information from the printer. This field is applicable only if you select Global from the Settings section of the Status tab.

Community Private (Set) Name

Access name provided to the printer to write SNMP settings to the printer. This field is applicable only if you select Global from the Settings section of the Status tab.

SNMP v3
Use SNMP v3 When Supported

If a printer supports SNMP v3, click this check box to use SNMP v3, which includes both encryption and authentication security options, so you can set SNMP to require both encryption and authentication, no encryption but with authentication, or neither option.

Username

If authentication is enabled in the Security Level field, then the system will ask for a username, and you would enter the value in this field.

Security Level

Setting that determines the type of security used in the SNMP connection. Options include both authentication and privacy, authentication but no privacy, and no authentication or privacy.

Authentication Protocol

The Authentication Protocol refers to the hashing algorithm used to store your authentication key. Options include MD5 and SHA. If you select a security to include both authentication and privacy or authentication but no privacy, then you will need to add an authentication protocol.

Authentication Key

If you select a security level to include both authentication and privacy or authentication but no privacy, then you will need to add an authentication key. It is similar to a password for the authentication protocol.

Privacy Protocol

If you select a security level to include both authentication and privacy, then you will need to add a privacy protocol. Options include DES and AES, and is an encryption algorithm used to secure communication between the added device and the printer installer server.

Privacy Key

If you select a security level to include both authentication and privacy, then you will need to add a privacy key. It is similar to a password for the privacy protocol.

Context Name

An SNMP context name or "context" in short, is a collection of management information accessible by an SNMP entity. An item of management information may exist in more than one context. An SNMP entity potentially has access to many contexts. In other words, if a management information has been defined under certain context by an SNMPv3 entity, then any management application can access that information by giving that context name. The "context name" is an octet string, which has at least one management information.

Context Engine ID

Within an administrative domain, contextID uniquely identifies an SNMP entity that may recognize an instance of a context with a particular context name. Most of the SNMPv3 entities have the snmpEngineID as their contextID.

SNMP Alerts

These options apply to printer objects that are configured to use the default settings in the Alerts tab as shown below.

Enable Email Alerts

This option enables alerts to be sent to specific email addresses. Additionally, make sure to click Enable SNMP Status Monitoring or you will see an error message when you click Enable Email Alerts.

By default, SNMP alerts for each printer is:

Selecting this option configures the default setting for SNMP monitoring. What this means is that the Alerts tab contains a Settings section with a Use Global option. If you select Use Global, then Printer Installer will use the option you have selected in the By default, SNMP alerts for each printer is: setting.

When a potential problem is detected:

This setting determines the number of times a problem is identified before Printer Installer sends an email notification identifying the issue.

Email (SMTP) Settings

Printer Installer uses the SMTP settings in the General tab for features that use email notifications such as alerts and scheduled reports.

NOTE: The SMTP settings in this section do not apply to email printing. For more information on those settings, see Printing Settings Cheat Sheet under Mobile Printing: Email Processing.

"From" Address

The email address used to send the alert or scheduled report; it is the address that appears in the From: field of the email.

Server Host

The SMTP server's hostname.

Server Port

The ports that needs to be opened through a firewall (if you have one). Port is 25.

Username

The username for the service email account.

Password

The email password for the service email account.

Security

The type of security used on the email server. Options include SSL, TLS, and None.

Test STMP Settings

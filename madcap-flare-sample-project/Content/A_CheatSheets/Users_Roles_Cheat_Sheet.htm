Users/Roles Cheat Sheet

The Users settings appear in the Admin Console when you click Tools > Users. The image to the left shows the Users settings as they appear in Printer Installer.

The Users settings are divided into two primary sections, namely Users and Roles. The Users section enables you to set up new user accounts or to change the user credentials for an existing account. These credentials are the same ones a user would enter to log in to the Admin Console; they do not apply to the Self-service Portal or the Release Portal . The Roles section comes with standard (out-of-the-box) roles that cannot be modified or deleted. Additionally, the Roles section gives you the ability to create custom roles, which give permission to access specific features of the Admin Console. Features not selected will not be available to the user assigned to the role. Read More...

Standard or custom roles are assigned to user accounts in the Users section. Moreover, these roles can be applied to specific folders or printer objects in the tree view, which enables the administrator to assign roles at a high level or to specific folders or printer objects. The fields of the Users settings are described in detail below.

Add User

This section enables you to set up a variety of user accounts using either Active Directory, the Printer Installer database, or even LDAP for PCs not joined to the domain.

AD Group/User

This option enables you to add an Active Directory user to Printer Installer.

NOTE: Before you can add an Active Directory user, you must set up an LDAP domain in Printer Installer. For more information, see General Settings Cheat Sheet.

When you set up this type of user, nothing is added to the Printer Installer database. Rather, on login, Printer Installer checks the user credentials against Active Directory credentials.

Printer Installer User

This option enables you to set up a new Printer Installer user without Active Directory. This type of user is stored in the Printer Installer database. On login, Printer Installer checks the database for matching credentials to provide access.

LDAP Group/User

This option appears in the drop-down only if you have selected Enable advanced LDAP administrative authentication in Tools > Settings > General as shown below.

This option enables you to add users or groups with the Active Directory Windows interface; i.e., Mac, Linux, or Windows PCs not joined to the domain.

Change Role Assignments

This option enables you to assign either a standard or a custom role (as listed in the Roles section) to a user account. When you click Change Role Assignments, the Configure User's Role Assignments window will appear as shown below.

You have Administrator and Non-Administrator options. If you select Administrator, then you are giving full Admin Console access rights to the selected user account. The user will be able to access and change anything in the Admin Console. The Administrator option does not carry any additional settings. A user is either an administrator or not an administrator.

The Non-Administrator option displays additional features, giving you more control over what the user can see and do in the Admin Console.

NOTE: If you are assigning standard roles to a user, then you do not need to set up any roles before configuring role assignments. However, if the standard roles don't have what you need, then you will need to set up a custom role before you can assign that role to a user.

If you select Non-Administrator as the type of role for your user, then you will need to click Add to select a folder and assign your custom role to the user. You can also assign different standard roles to different folders or printer objects in the tree. For more information, see Create a Role & Apply it to a User Account.

Change Name/Password

This option applies only to local user accounts, which are stored in the Printer Installer database. It does not apply to Active Directory or LDAP users because Printer Installer does not write to Active Directory. When you select a user and then click Change Name/Password, the Update User window will appear, enabling you to make changes to a specific user account. When you have made your changes, click Update.

Delete

From Tools > Users, you will see two Delete buttons. One is for removing a user account, and the other is for removing a role. Administrators can remove any role in the list of users with the exception of the root account user. Additionally, administrators can remove any custom role, but not standard roles as they are available as defaults.

Add Role

This option is available to enable you to create a custom role, which is used to assign specific permissions to one or more user accounts. The Roles section displays both standard roles and custom roles. Standard roles are default and come out-of-the-box, which means you cannot edit them in any way. Custom roles are the ones you create because the standard roles do not have a role that fits your needs.

When you click Add Role, the following window appears, enabling you to create a new role that is not available by default.

Creating a role is relatively simple. Just provide a name for the role and then click the check boxes to the left of the features that you want to make available to the user accounts that are assigned the new role. Keep in mind that if you select a permission at the root level, then permissions for all sub-items under that root level will be provided. For example, if you click Tools Menu, then all permission for all items under the Tools Menu will be granted for users assigned the role. When you are done, click Save. Custom roles appear in the Roles section with Custom appearing in the Type column.

View

The View button appears only if you select a standard role and is used to open the settings for the role that is currently selected in the list of roles. If you select a standard role, then the settings will appear as read-only. If you select a custom role, then the settings can be modified as needed, but the View button is replaced with the Edit button.

Edit

The Edit button appears only if you select a custom role because custom roles can be modified. If you select a standard role, which are read-only, then the Edit button is replaced with the View button.

Copy

Click the Copy button to copy the settings from the role currently selected. This feature is generally used when you want to create a custom role with settings very similar (but different) to a role that currently exists. Rather than create a role from scratch, you can use the settings from an existing role.

When you click Copy, the Create Role window will appear as shown below.

The copied role will have the same name as the original with the word (copy) after it to ensure the role has a different name than the original. You can modify the name as needed as long as it does not use the same name as an existing role.

The permissions from the original role will automatically be selected as well, which is the primary reason for copying a role in the first place. You can deselect permissions currently selected, or you can select new ones as needed. When you are done, click Save.

Role Name

The default name given to a default role or the custom name (in the Role Name field) you provided when you clicked Add Role.

Description

The default descriptions given to default roles, or the custom description (from the Role Description field) when you clicked Add Role.

Type

This column will display either Standard or Custom. Standard refers to default roles provided out-of-the-box, and Custom refers to roles you create when you click Add Role. Standard roles are read-only, which means they cannot be edited or deleted. Custom roles can be edited or deleted as needed.

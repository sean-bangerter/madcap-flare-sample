Email Printing Tab Cheat Sheet

The Email Printing tab appears when you select a service host object in the tree view as shown in the image to the left. In versions prior to 18.1, Printer Installer does not have a Service Host. However, as of 18.1 it has been added for future features and functionality.

The Email Printing tab enables an administrator to configure settings for Email Printing, Direct Email Printing, and Direct Email Guest Printing. For more information, see Email Printing. The settings in the Email Printing tab are unavailable until you select Enable Email Printing. Furthermore, specific settings are used for configuring email printing while others are used for enabling email printing. Read More...

After you click Enable Email Printing, the settings become available. The first step is to identify the email address that end-users will enter into the To: field of their email application. For example, if you use print@abccorp.com, then the end-user would add print@abccorp.com to the To: field of their email to send the print job to the printer.

The next step is to select a virtual pull print queue to process the email print jobs and then configure the SMTP settings that PrinterCloud uses for outgoing emails,and IMAP settings that PrinterCloud uses for incoming emails. After you configure those email settings, then you can select whether Direct Email Printing and Direct Email Guest Printing are enabled or not. Keep in mind that you must enable Email Printing before you can enable Direct Email Printing, and you must enable Direct Email Printing before you can enable Direct Email Guest Printing.

Enable Email Printing

Select this option to enable general Email Printing. Both Direct Email Printing, and Direct Email Guest Printing also require this field to be enabled. For more information, see Email Printing.

None Selected

Click None Selected to select the pull printer that will be used as the queue for email print jobs. Clicking this option opens the Select Mobile Pull Printer window (as shown below), which enables you to select the pull printer that will be used as the queue.

Keep in mind that only the printer objects that have the Make Pull Printer field selected in the Printing tab will appear in the Select Pull Printer window above. Once you select a printer object to be the queue, then the None Selected button will display the name of the printer object as shown below.

If a printer object has the Make Pull Printer field selected, and that printer object is selected for mobile printing (through the Mobile Pull Printer window), then the printer object will be designated for releasing mobile print jobs. Furthermore, if a printer object is set for pull printing and for mobile printing, then you cannot delete it from the tree view. You will receive a message stating that you cannot delete the node.

You must select a different printer object for mobile printing before you can delete the printer object.

Pull Printing Email Address

The email address end-users enter to send email jobs to a pull printer. For this field to become enabled, you must first select Enable Email Printing.

Incoming Mail (IMAP) Settings

PrinterCloud uses the IMAP settings specifically for Email to Print and Direct Email to Print to connect to the server where the emails are being stored to check for emails to be printed.

Server

Enter the fully qualified domain name or the IP address in the Server field.

Server Port

The port that must be opened through a firewall (if you have one). Non-secure port is 143. Secure port is 993. For Google SSL/TLS Gmail server connections, use ports 465 or 587.

Username

The email account username for the service email account.

Password

The email password for the service email account.

Security
SSL

SSL (Secure Sockets Layer) is the standard security technology for establishing an encrypted link between a web server and a browser. This link ensures that all data passed between the web server and browsers remain private and integral.

TLS

The Transport Layer Security protocol aims primarily to provide privacy and data integrity between two communicating computer applications.

None

Select None if you are using security other than SSL or TLS or you are not using any type of security technology.

Ignore security certificate warnings

Check this option if you use self-signed certificates.

Test IMAP Connection

Click this link after you have configured all the IMAP settings to see if the settings are functioning correctly.

Outgoing Mail (SMTP) Settings

PrinterCloud uses the SMTP settings specifically for Email to Print and Direct Email to Print to send a notification to an email address when an email print job has finished printing.

Server

Enter the fully qualified domain name or the IP address in the Server field.

Server Port

The port that needs to be opened through a firewall (if you have one). Port is 25.

Username

The email username for the service email account.

Password

The email username for the service email account.

Security

The type of security used on the email server.

SSL

SSL (Secure Sockets Layer) is the standard security technology for establishing an encrypted link between a web server and a browser. This link ensures that all data passed between the web server and browsers remain private and integral.

TLS

The Transport Layer Security protocol aims primarily to provide privacy and data integrity between two communicating computer applications.

None

Select None if you are using security other than SSL or TLS or you are not using any type of security technology.

Test SMTP Connection

Click this link after you have configured all the SMTP settings to see if the settings are functioning correctly.

Enable Office-type File Attachment Printing

Enables an email attachment to be converted to PDF and printed. LibreOffice must be installed for this feature to work.

Path to LibreOffice Executable

Enter the path to the LibreOffice executable. This field is required only if you have the Enable Office-type File Attachment Printing check box enabled.

Enable Direct Email Printing

Click this option to enable Direct Email Printing in the Admin Console so that an end-user can print an email from a mobile device. For more information, see Email Printing.

Enable Direct Email Guest Printing

Click this option to enable Direct Email Guest Printing in the Admin Console so that en end-user who is not authenticated in Active Directory can still send an email to a printer. Additionally, this option is available only if you select Enabled Direct Email Printing first. For more information, see Email Printing.

Limits how much each guest is permitted to print to X pages per day total

The number you place in this field determines the total number of pages that a guest user can print per day. A guest user is a person who cannot be identified through Active Directory to release print jobs to a printer. This field is available only if you select Enable Direct Email Guest Printing and Enable Direct Email Printing.

Force guest print jobs to be black and white

Click this option and any guest user who is not authenticated in Active Directory can still send an email to the printer, which will release in black and white only. This field is available only if you select Enable Direct Email Guest Printing and Enable Direct Email Printing.

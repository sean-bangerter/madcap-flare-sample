Driver/Profile Repository Cheat Sheet

The Driver/Profile Repository appears in the Admin Console when you click Tools > Driver/Profile Repository. The image to the left shows the Repository as it appears in Printer Installer.

The Driver/Profile Repository serves several purposes. First, when you upload a new driver (and profile) through the Drivers/Profiles tab, then the driver appears as a line item in the Repository, so you can have a high-level view of all the drivers in Printer Installer. Additionally, the Repository displays the profiles that have been applied to the drivers that have been uploaded to the system. These profiles appear as child items underneath the driver to which it has been assigned. Read More...

The Driver/Profile Repository also provides an option to upload a new driver. This upload option is no different than the one you would use in the Driver/Profile tab to upload a driver. Additionally, after you upload a driver, you can also apply a profile to the driver. The profile, by default, is provided by the manufacturer with the driver itself. Also take notice of the Replace option, which enables you to overwrite an existing driver with either a new driver or a driver that already exists in the Repository.

Finally, the operating system check boxes configure a driver to be available or unavailable in the Drivers/Profiles tab. For example, let's say your driver is for 64-bit Windows only, and you select Win 8/10 check box, but you deselect the Win 7 check box, then when you click the Driver/Profile tab, then that driver will be available for selection in the Windows 8/10 drop-down, but it won't be available in the Windows 7 drop-down. The fields of the Driver/Profile Repository are described in detail below.

Upload New Driver

This button is no different than any of the buttons with the same name in the Drivers/Profiles tab. When you click this button, you will select the .inf file for the driver you are uploading. Any new driver you upload to Printer Installer will appear in the Drivers/Profiles Repository.

Replace

Select a driver in the Repository, click Replace, and select either a new driver or an existing driver already in the Printer Installer database. The Replace option is grayed-out until you select an existing driver in the Repository. Furthermore, the Replace option is never available for the Citrix Universal Printer line item in the Repository because that line item is basically a placeholder for the system to automatically search for a Citrix driver.

Delete

Select a driver in the Repository and click Delete to remove a driver from Printer Installer. The Delete option is grayed-out until you select an existing driver in the Repository. Furthermore, the Delete option is never available for the Citrix Universal Printer line item in the Repository because that line item is basically a placeholder for the system to automatically search for a Citrix driver.

Filters

These options enable you to narrow the drivers that appear in the Repository.

All Drivers/Profiles

Select this option, and all drivers and profiles in the Repository will appear as line items in the Driver/Profile Repository.

Unused Drivers

Select this option and any drivers not currently in use in the Self-service Portal will appear

Add Profile

This option appears only when you select a line item from the Driver/Profile Repository. It enables you to apply a profile to an existing driver in the Repository.

Import Profile

This option appears only when you select a line item from the Driver/Profile Repository. It enables you to import an existing profile into Printer Installer and apply it to an existing driver in the Repository.

Driver Model Name

The name given to the driver when it was added to Printer Installer. Each printer manufacturer provides default names for the drivers, but you can change the name. However, no two driver names can be the same, which is why Printer Installer provides the annotation string in parentheses at the end of each driver model name. When you set up a new driver, even if the driver has the same name as an existing driver in the repository, if you give each driver a unique annotation string, then both drivers can have the same name. For more information, see Add a Driver/Profile to a Printer Object.

Driver Version

The version of the driver you uploaded. If you would like to change the version of a driver in the Driver/Profile Repository, use the Replace button.

Driver Date

The Driver Date field shows the date the driver was created - not the date the driver was uploaded.

x86/x64/Citrix

These check boxes show the operating systems that a printer driver supports. x86 is for 32-bit Windows operating systems, and x64 is for 64-bit Windows operating systems. The Citrix Universal Printer cannot be altered and does not have any check boxes because it is used as a placeholder to automatically detect Citrix drivers.

When you upload a driver to Printer Installer, the printer manufacturer automatically sets which operating systems are supported for the driver as shown below.

However, you can deselect the check box for an operating system if you no longer want to support the OS for that driver. If an operating system is no longer supported for the driver, then you can click the Drivers/Profiles tab, and the drop-down for the unsupported operating system will still appear, but you will not be able to select the driver for that operating system.

Deploy Tab Cheat Sheet

The Deploy tab appears in the Admin Console when you select a printer object or an IP address range in the tree view. The Deploy tab does not appear when you select a folder.

The image to the left shows an IP address range selected in the tree view and the selected Deploy tab. The purpose of the Deploy tab is to enable Printer Installer to automatically install print queues and drivers on end-users' workstations so you don't have to rely on your end-users to do it themselves.

When you select a printer object in the tree view, you will also notice the Drivers/Profiles tab appears. Because you can upload the drivers to the Drivers/Profiles tab directly, those drivers apply to the selected printer object. As such, when you click the Deploy tab, PrinterCloud already knows which driver and profile to install on the workstation(s). What it doesn't know is which workstation(s) to install it on. That is what the Deploy tab is for. So, the button is aptly named Add because you are adding the Active Directory object or other item that PrinterCloud will use to determine the workstations that will receive the printer deployment. The interface features that appear in the Deploy tab are entirely dependent on the item you select in the tree view. In other words, the Deploy tab displays certain fields for printer objects and other fields for the IP address range. The Deploy tab for both printer objects and IP address ranges displays basic buttons such as Delete, Cancel, and Save used for managing changes to or removing your deployments. However, you will notice interface features that are specific to the Deploy tab when you select a printer object and those that are specific to the Deploy tab when you select an IP address range. For example, selecting a printer object and then clicking the Deploy tab displays the Add button, while selecting an IP address range and then clicking the Deploy tab displays the Add Printer button.

Example: Mary deploys a new printer driver to a group

Mary is the print administrator for her company, and she is using Printer Installer. Mary adds a new printer object named Printer A to her tree view, and in the Drivers/Profiles tab she uploads a new driver called Printer A Driver. She then clicks the Deploy tab and selects Add Printer, which provides her with a number of options, some of which are related to Active Directory while others are not. Because Mary set up an Active Directory group (outside of Printer Installer) called Group 1 consisting of Ben, Michelle and Abel, she selects Active Directory > Users, Computers, or Groups, and then she selects Group 1 and clicks OK. After Mary saves her changes to the Deploy tab, then the next time the workstations belonging to Ben, Michelle, and Abel check in with Printer Installer, the driver from the Drivers/Profiles tab will be automatically installed on all three users' workstations.

The fields of the Deploy Tab are described in detail below.

Add Printer

The Add Printer option is available only if you select an IP address range object in the tree view. Click this option to browse to or search for an existing printer object in the tree view. The printer object you select will be deployed to the workstations that fall within the IP address range set in the General tab.

NOTE: In the Deploy tab, associations between printer objects and IP address range objects run both ways. In other words, if you select a printer object in the tree view, an associated IP address range will appear in the Deploy tab. Similarly, if you select the IP address range object, then the associated printer object will appear in the Deploy tab.

Add

This option is available only if you select a printer object in the tree view. It enables you to deploy a printer to an Active Directory object such as a user, computer, group, etc., or to a (missing or bad snippet) object such as an IP address range object, advanced group, etc.

User, Computer, or Group

If you click Active Directory > User, Computer, or Group, then the Select Users, Computers, or Groups window will appear as shown below.

After you have selected the user, computer, or group, click OK.

Container or OU

If you click Active Directory > Container or OU, then the Select Containers window will open as shown below.

After you have selected the container or organizational unit, click OK.

IP Address Range

If you click IP Address Range, then the Add IP Address Range window will appear as shown below.

Click Search to type the IP address range, or click Browse to navigate the tree view to find the IP address range you would like to add. When you have selected the IP address range, click Add.

Hostname

If you click Hostname, then the corresponding window (respectively) will appear as shown below.

Type the hostname and then click Add.

MAC Address

If you click MAC Address, then the corresponding window (respectively) will appear as shown below.

Type the MAC address and then click Add.

Advanced Group

If you click Advanced Group, then the Add Advanced Group window will appear as shown below.

Select an Advanced Group from the list of those available and then click Add.

Automatic Deployments

The Automatic Deployments drop-down enables you to automatically deploy the printers in the same folder (or subfolder) that the IP address range object resides in. This feature is mostly useful in scenarios in which the administrator is using an Epic print server.

None

Select this option, and only printers that have been manually added (using the Add Printer button) will be deployed to the workstations with IP addresses that fall within the IP address range.

All printers in this folder

Select this option to deploy only printers that reside in the same folder as the IP address range object to workstations with IP addresses that fall within the IP address range.

For example, let's say you have a parent folder named "Dallas" that has an IP address range object and four printers . If you select All printers in this folder, then all four printers will automatically be deployed to workstations with IP addresses that fall within the range set in the IP address range object.

All printers in this folder and sub-folders

Select this option to deploy printers that reside in the same folder as the IP address range object or any subfolders to workstations with IP addresses that fall within the IP address range.

For example, let's say you have a parent folder named "Chicago" that has two subfolders called "Arlington Heights" and "Deerfield" respectively. The Chicago folder has an IP address range object, and the Arlington Heights and Deerfield folders contain three printers each. If you select All printers in this folder and subfolder, then all the printer objects in the Arlington Heights and Deerfield folders will be automatically deployed to workstations with IP addresses that fall within the range set in the IP address range object.

Printer Name

The printer object that will be deployed when the workstation checks in with Printer Installer. This field appears only when you have selected an IP address range object in the tree view.

Name

This name of the Active Directory object, IP address range, hostname, etc. to which the selected printer object in the tree view will be deployed. This field appears only when you have selected a printer object in the tree view.

Share

This option enables you to set a specific printer to be shared by multiple end-users. For example, you can set a workstation to release print jobs, so you need to share that workstation to the end-users who will be using it. The Share button only becomes available when you add a Computer object (Active Directory) to the Deploy tab.

Users

The Users column enables you to select whether or not you will install the printer on the workstations of users who are part of an Active Directory group or organizational unit. This check box appears only for Active Directory group or organizational unit (OU) objects.

Computers

The Computers column enables you to select whether or not you will install the printer on the workstations or users whose computers are part of an Organizational Unit. This check box appears only for Active Directory group or organizational unit (OU) objects.

Set Default

This option enables you to remotely set a printer object as the default printer for end-users who have the printer installed on their workstation.

No

Select No if you do not want the selected printer object to be set as the default printer for the given object; i.e., user, group, computer, etc.

Once

Select Once to set the printer object as the default printer on workstations for the selected object in the Deploy tab; i.e., user, group, computer. The end-user can change the default printer on their workstation at any time, and when the client on their workstation checks in, the selected default printer will not change. The Once option is useful when the administrator would like to identify (to the end-user) the preferred default printer for their workstation while still enabling the end-user to select their own default printer.

Always

Select Always to set the printer object as the default printer on workstations for the selected object in the Deploy tab; i.e., user, group, computer. The end-user can still change the default printer on their workstation at any time, but when the client on their workstation checks in, the selected default printer will revert back to the default printer as defined by the administrator.

Printing Settings Cheat Sheet

Printing Settings appear in the Admin Console when you click Tools > Settings > Printing. Printing Settings are used to configure many of the printing types in PIPC, including pull printing, secure release printing, mobile printing, and email printing.

Many of the settings apply to printer objects that are configured to use default settings. For example, let's say you select a printer object in the tree view and then click the Printing tab. If the Pull Printing section is configured to Use default Secure Release settings or Use default Secure Release settings, then that printer object will use the default settings in the Pull Printing section of the Printing Settings. The same applies to printer objects that are configured to use default settings for mobile printing and email printing. Read More...

Some of the options are a little more simple, such as the Purging section, which enables the administrator to set the amount of time that print jobs stay in the Queue tab or the Release tab before they are deleted. Make sure you save any changes before leaving the Printing Settings. The Printing Settings fields are described in detail below.

Print Jobs

The options in this section enable you to configure Printer Installer to record print job records and to track print queues.

Enable Print Job Auditing

Click this option so that Printer Installer records print job records in the database. If this option is not selected, then print job records are not recorded in the Printer Installer database.

Include local or shared TCP/IP printers

Click this option to log print jobs for any Printer Installer-managed printer.

Include local USB printers

Click this option to log print jobs for USB-attached printers.

Delete old print job records

Click this option to remove print job records more than two years old from the Printer Installer database.

Include print job title in reports

Deselect this option if print job titles contain sensitive information that you do not want printed in reports.

Enable queue management

Click this option to enable the Queue Management feature in the Queue tab. When deselected, the Queue Management tab will display the following message.

Pull Printing

This section contains the default settings for both Pull Printing and Secure Release Printing.

Any printer objects in the tree view that are configured to use the default settings in the Printing tab, as shown below, will use these default Pull Printing and Secure Release Printing settings in the Tools > Settings > Printing. For more information, see Pull Printing or Secure Release Printing.

Enable Workstation Pull Printing/Secure Printing

This option applies to any printer objects, whether they are configured to use default settings or printer-specific settings. If this option is selected, then printer objects that are configured to allow pull print and secure print jobs will be able to release those jobs to the designated printers.

For example, let's say you select Enable Workstation Pull Printing/Secure Printing in the Printing Settings. Additionally, in the Printing tab you select Use printer-specific Pull Print settings and then you click Allow Pull Print jobs to be released to this printer.

In this case, Pull Printing jobs can be released to the selected printer object in the tree view.

Allow Pull Print Jobs to be released to this printer

This field applies only if you have selected Use default Pull Printing settings in the Printing tab. By selecting this option, you are enabling pull print jobs to be released to the printer associated with the printer object selected in the tree view. For more information, see Pull Printing.

Allow secure release jobs to be sent to this printer

This field applies only if you have selected Use default Secure Release settings in the Printing tab.

By selecting this option, you are enabling Secure Release print jobs to be released to the printer associated with the selected printer object in the tree view. For more information, see Secure Release Printing.

Allow End-Users to Override

Regardless of the options selected in the Default per-printer Pull Print setting field and the Default per-printer Secure Release settings field, if you have Allow End-Users to Override selected, then a user can decide whether to print a pull print job or a secure print job to the designated printer.

Mobile Printing

This section applies only if you have selected Use default Mobile Printing setting in the Printing tab as shown below.

The options in this section enable you to configure the printer associated with the selected printer object in the tree view to release print jobs from mobile devices. For more information, see Mobile Printing.

None Selected

Click None Selected to select the pull printer that will be used as the virtual queue for mobile print jobs. None Selected is the default text that displays on the button. After you select a virtual pull printer, then the name of that pull printer will appear on the button.

NOTE: When you select a printer object as the virtual pull print queue, you will not be able to delete it until you select an alternate printer object as the queue. If you attempt to delete a printer that is set as a queue, then you will see the following error.

Enable Mobile Printer

This option displays the Select Mobile Printer option in the Self-service Portal (as shown below), which enables the end-user to select the printer they want to use to release mobile print jobs.

When the end-user selects Menu > Mobile Printing > Select Mobile Printer in the Self-service Portal, the Set Mobile Printer window will appear, enabling the end-user to select from any of the available printers for releasing mobile print jobs.

The print job will be initially directed to the pull print queue set up for mobile printing and then redirected to the printer that the end-user has selected to release the mobile print job.

A user may want to configure a mobile printer in cases where they are using mobile printing but always releasing to the same printer. Specifying the mobile printer saves time and hassle of going to release the print job every time.

Default per-printer Mobile Printing setting

This option applies only to printer objects that are configured to use the mobile printing default setting in the Printing tab. If the option is selected, then the printers associated with the printer objects using the default setting will be able to release mobile print jobs.

Default page size

This option sets a default paper size to be used by those printers associated with printer objects using the mobile printing default setting in the Printing tab.

Mobile Printing: Google Cloud Printing

This section enables you to configure the settings for Google Cloud Print in Printer Installer. For more information, see Mobile Printing.

Enable Google Cloud Printing

Clicking this option enables Google Cloud Printing in your PrinterCloud environment. If Google Cloud Printing is enabled, then PrinterCloud will use the Pull Print Queue selected in the Google Cloud Printing tab.

Google Cloud Print Account
Admin Email

In this field, enter the administrator-level email address for the Google Cloud Print account to which the documents will be published for printing.

Admin Password

In this field, enter the password for the Google Cloud Print account to which the documents will be published for printing.

Use Proxy Server

Use these options if you have an environment that does not allow direct access to the Internet, and your computers must communicate via a proxy server, which is a "go-between" that may allow access to sites that otherwise could not be accessed.

For example, when the Printer Installer server attempts to access real-world web sites associated with Google Cloud Printing, the proxy server is the mechanism by which the computers can access these sites.

Server URL/IP Address

The web address used to access a proxy server through a web browser.

Port

The port number that is used by the proxy server.

Proxy Username (optional)

The username credential that Printer Installer will use to access the proxy server.

Proxy Password (optional)

The password associated with the username Printer Installer will be using to access the proxy server.

Google Cloud Printer Registration

This button registers the selected pull printer with Google Cloud Print.

NOTE: Before you click Register Printer, you must log in to Google Cloud Print using the same credentials you entered in the Admin Email and Google Password fields.

Mobile Printing: iOS AirPrint

This section enables you to configure the settings for iOS printing in Printer Installer.For more information, see Mobile Printing.

Enable Virtual Air Print Printer for iOS printing

Clicking this option enables printing from iOS devices in your Printer Installer environment. If iOS printing is enabled, then Printer Installer will use the virtual Pull Print Queue selected in the Mobile Printing section of the Printing settings, as shown below.

User Authentication
Require login for each print job

Select this option if you want Printer Installer to require the end-user to input their Active Directory credentials prior to releasing any print job from an iOS device.

Allow passwords to be saved on the device

Select this option if you want Printer Installer to save the end-user's Active Directory credentials. However, the end-user must enter their credentials the first time releasing a print job from an iOS device, after which those credentials will be saved on the device being used to release the print job. If the end-user releases a print job from a second device, then the credentials will need to be entered again on that device and saved.

Mobile Printing: Email Printing

These setting enable you to configure basic Email Printing, which uses Active Directory to validate the end-user and then sends the email to a print queue to be held until the end-user releases the print job. For more information, see Email Printing.

Enable Email Printing

Select this option to enable general Email Printing. Both Direct Email Printing, and Direct Email Guest Printing also require this field to be enabled. For more information, see Email Printing.

Pull Printing Email Address

The email address end-users will enter into the To: field of their email application for release to a printer.

Mobile Printing: Direct Email Printing

These settings enable you to configure Direct Email Printing, which uses Active Directory to validate the user's email address and then send the email immediately to a release printerIf. For more information, see Email Printing. If the user cannot be validated, then the user will be considered a guest and will be required to use the Direct Email Guest Printing settings. For more information, see Email Printing.

Enable Direct Email Printing

Click this option to enable Direct Email Printing in the Admin Console so that an end-user can print an email from a mobile device.

Direct Email Address Domain

This is the domain that Printer Installer uses for Direct Email Printing. The domain entered here will appear after the direct email address in the Mobile Printing: Direct Email Printing section.

Enable Direct Email Guest Printing

Click this option to enable end-users who cannot be validated through Active Directory credentials to be able to send an email to a printer.

Limit how much each guest is permitted to print to X pages per day total

If you enable Direct Email Guest Printing and Direct Email Guest Printing, select this option to limit the number of pages the guest can print per day. Make sure you also add the number of pages in the associated field.

Force guest print jobs to be black and white

If you enable Direct Email Guest Printing and Direct Email Guest Printing, then click this object to force guests to print only in black and white.

Default per-printer "Direct Email Printing" setting

This option configures the default setting for Direct Email Printing and applies only if you select Use default Direct Email Printing setting, as shown below.

Default per-printer "Direct Email Guest Printing" setting

This option configures the default setting for Direct Email Guest Printing and applies only if you select Use default Direct Email Guest Printing setting as shown below.

Mobile Printing: Email Processing

The settings in this section enable you to configure Printer Installer both to check the email inbox for jobs to print and to send a notification when an email print job has finished printing.

Incoming Mail (IMAP) Settings

Printer Installer uses the IMAP settings specifically for Email to Print (all types) to connect to the server where the emails are being stored to check for emails to be printed.

Server Host

The IMAP server's hostname.

Server Port

The port that must be opened through a firewall (if you have one). Non-secure port is 143. Secure port is 993. For Google SSL/TLS Gmail server connections, use ports 465 or 587.

Username

The username for the service email account.

Password

The password for the service email account.

Security

The type of security used on the mail server. Options include:

SSL

SSL (Secure Sockets Layer) is the standard security technology for establishing an encrypted link between a web server and a browser. This link ensures that all data passed between the web server and browsers remain private and integral.

TLS

The Transport Layer Security protocol aims primarily to provide privacy and data integrity between two communicating computer applications.

None

Select None if you are using security other than SSL or TLS or you are not using any type of security technology.

Ignore security certificate warnings

Check this option if you use self-signed certificates.

Test IMAP Connection

Click this link after you have configured all the IMAP settings to see if the settings are functioning correctly.

Outgoing Mail (SMTP) Settings

Printer Installer uses the SMTP settings specifically for Email to Print and Direct Email to Print to send a notification to an email address when an email print job has finished printing.

Server Host

The SMTP server's hostname.

Server Port

The port that needs to be opened through a firewall (if you have one). Port is 25.

Username

The username for the service email account.

Password

The password for the service email account.

Security

The type of security used on the email server

SSL

SSL (Secure Sockets Layer) is the standard security technology for establishing an encrypted link between a web server and a browser. This link ensures that all data passed between the web server and browsers remain private and integral.

TLS

The Transport Layer Security protocol aims primarily to provide privacy and data integrity between two communicating computer applications.

None

Select None if you are using security other than SSL or TLS or you are not using any type of security technology.

Test SMTP Connection

Click this link after you have configured all the SMTP settings for email to see if the settings are functioning correctly.

Enable Office-type File Attachment Printing

Enables an email attachment to be converted to PDF and printed. LibreOffice must be installed for this feature to work.

Path to LibreOffice Executable

Enter the path to the LibreOffice executable. This field is required only if you have the Enable Office-type File Attachment Printing check box enabled.

Test Executable

Click this check box if you have Enable Office-type File Attachment Printing enabled, and you have the Path to LibreOffice Executable set. This field enables you to determine if your LibreOffice executable is working correctly.

Purging

These settings configure Printer Installer to hold print jobs in queue for a specific period of time. If the print jobs have not been released by the designated time period, they will be deleted from the queue.

Purge active print jobs after X hours

Click this option to remove print jobs from queue after a specified period of time. The default time is 12 hours. For this setting to work, you must have Queue Management enabled, as shown below.

Purge held jobs after X hours

Click this option to remove pull print, secure print, and mobile print jobs from a print queue. For this setting to work, you must enable Workstation Pull Printing/Secure Printing, as shown below.

Port Tab Cheat Sheet

The Port tab appears when you select a printer object in the tree view as shown to the left. The purpose of the information stored in the Port tab is to configure the settings to enable the end-user machine to communicate with a printer.

When you first set up a printer object in the Admin Console, you are required to enter an IP address or hostname. The value that you use will appear in the Port tab Port Name field, but you can change that value to another IP address or hostname at any time. The hostname is generally used in instances when you are using the DHCP protocol.Read More...

In the Port tab, the Port Name and Protocol sections are required for printing functionality, in particular to connect Printer Installer to the printer. Select Raw when you are sending a stream of raw data to the printer. Raw is the most common protocol and has the fewest settings. Just make sure you use a format the printer will understand. Select LPR if you are sending the data to a print server. LPR is sometimes used to create a "direct TCP/IP" printer that is printing to a windows (or non-Windows) print server. Some printer devices act like a print server for LPR. For example, you can get a network device that is just a little box you plug multiple LPT or USB printers into. That device acts like an LPR server. The port info would be the same IP address for all the devices plugged into that device, but would have a different queue name for each device plugged into it.

The SNMP settings in the Port tab are configured for the printer on the end-user's machine to control "extra" information the end-user might see about the status of the printer. These settings are not required for printing functionality, so in a corporate environment, they would be left blank because the end-users would see all sorts of warnings and annoyances. The recommended best practice is to set the SNMP settings in Tools > Settings > General because setting up SNMP in the Port tab configures printers on the end-user's machine but has no effect on Printer Installer's ability to get status information about the printer. The fields of the Port Tab are described in detail below.

Port Name

By default, the Port Name is the printer's URL. When you add a new printer to the tree view, you will be asked to provide an IP address or hostname. The text you use in this field will appear as both the web interface and the port name.

Protocol

This section refers to the communication protocol the print queue uses to connect to the print device. Raw is used on most printers to send print jobs directly to a printer. LPR is used on a slim number of printers under very specific circumstances.

Raw

Raw is most commonly used for most printers to communicate on a port. If you select Raw, then type 9100 in the Port Number field.

LPR
Queue Name

The Queue Name is a generic name given to a printer attached to a network device (LPR server).

LPR Byte Counting Enabled

Click the LPR Byte Counting Enabled option to process print jobs twice - once for get the page count information and once to send the print job. This is generally used because LPR is a bare-bones protocol that does not identify the number of pages in a print job until you have printed a document.

Port Number

Enter the port number that will be used to send print jobs to the printer. If you are using Raw, the port is 9100.

SNMP Status

Select this option if you want the print queue on the desktop to use SNMP to communicate with the print device to find the printer status. In short, if you click this option, then the SNMP Status Enabled check box in Windows will be enabled, as described below.

  1. In Windows, click Start and then type Printers & Scanners.
  2. Click Printers & Scanners.
  3. Click the name of the printer.
  4. Click Manage and then click Printer Properties.
  5. When the Printer Properties window appears, click the Ports tab.
  6. From the list of ports, select a port and then click Configure Port.

The SNMP Status Enabled check box is affected based on the selection in the Admin Console SNMP Status field.

Community Name

SNMP requires a community name to communicate with the printer and is similar to a password. A printer generally has two community names - one for read-only access and the other for administrative use. The read-only community name defaults to "public" and is the one that you should use to connect Printer Installer to a printer. The administrative name is printer-dependent and is disabled by default. An incorrect community name will prevent communication with the printer via SNMP.

SNMP Dev Index

SNMP Device Index. This field is generally used for special network printing devices (multi-printer devices) but can normally just be set to 1 or any number you have set as the default.

OM Plus

Use these settings if you are integrating your OM Plus environment with Printer Installer.

Route all print jobs to OM Plus Delivery Manager

Click this option to enable the OM Plus integration with Printer Installer. If you do not any OM Plus servers set up, then you will receive a notification to that effect.

Queue Name

The Queue Name field is the name of the printer on the OM Plus server in the OM Plus system. If you select Route all release print jobs to OM Plus Delivery Manager, then the Queue Name field will become available.

If you select an existing printer in the tree view and then select Route all release print jobs to OM Plus Delivery Manager, then you can add the Queue Name of an existing destination in the OM Plus system. However, if you set up an OM Plus Managed Printer through the tree view (New > OM Plus Managed Printer), then the Queue Name field will be auto-populated with the name you give to the OM Plus Managed Printer (this field cannot be modified).

NOTE: The Queue Name must match an existing destination in the OM Plus system.

Use default servers

Select Use default servers if you would like an OM Plus Managed Printer to use the default server settings in Tools > Settings > General under the OM Plus section.

Use printer specific servers

Select Use printer specific servers if you would like an OM Plus Managed Printer to use specific servers rather than the ones allocated for default use. This field is available only if you have selected Route all print jobs to OM Plus Delivery Manager.

Add

Click Add to enter the hostname of the OM Plus servers to which print jobs will be sent.

Remove

Click Remove to eliminate the hostname of the OM Plus servers used for print jobs.

Printing Tab Cheat Sheet

The Printing tab appears in the Admin Console when you select a printer object in the tree view. The Printing tab generally displays many of the settings a printer object uses in the process of executing a printing job.

The most basic settings in the Printing tab enable you to set a printer object to be a virtual pull print queue, which disables any other settings in the tab. However, you can select a different printer object in the tree view and then set that printer object to release both pull print jobs and secure release print jobs. Take note of the default and printer-specific settings, which configure the printer object to use the global pull printing settings in (Tools > Settings > Printing) or to use the settings in the Printing tab, which will apply only to the selected printer object.

For example, after you designate a printer object as a pull printer, and then you configure the pull printer settings (either default or printer-specific) to release both secure release print jobs and pull printing jobs. Read More...

The Printing tab also offers Control Panel Application settings, which are used to release print jobs via badge swipe. For more information, see For more information, see Install the Control Panel Application. Additionally, the Printing tab provides settings for configuring the selected printer object to release mobile print jobs (For more information, see Mobile Printing.) or Direct Email Print jobs (including Direct Email Guest Print jobs). For more information, see Email Printing.The fields of the Printing Tab are described in detail below.

Pull Printer

This section appears in the Admin Console Printing tab only if you select a printer object or an OM Plus printer object from the tree view. Click Make Pull Printer to configure the selected object as a pull print queue, which is used to hold pull print jobs until the end-user releases them via the Web Release Portal, Control Panel Application, or Simple Badge Release.

NOTE: PrinterLogic recommends that you click Make Pull Printer only when creating a new printer object rather than applying the setting to an existing printer object.

For more information, see Pull Printing.

Pull Printing

This section enables you to configure pull printing and secure release printing for a pull print-enabled printer object. When a printer object is configured for pull printing, the device associated with the printer object will be able to print a pull print job when the end-user is ready to release it. For more information, see Pull Printing.

Use default Secure Release Settings

This option configures the selected printer object in the tree view to use the default Secure Release settings configured in the Pull Printing section in Tools > Settings > Printing as shown below.

For more information, see Secure Release Printing.

Use printer-specific Secure Release settings

This option configures the selected printer object in the tree view to use the printer-specific Secure Release settings.

For more information, see Secure Release Printing.

Allow secure release jobs to be sent to this printer

This option is available only when you select Use printer-specific Secure Release settings. This option configures the printer object selected in the tree view to support Secure Release printing.

No

Selecting No means that secure printing will not be available from a specific workstation for a specific printer.

Yes (Always)

Selecting Yes (Always) means that secure printing is always available from a workstation for a specific printer.

Yes (Prompt)

Selecting Yes (Prompt) means that the client will prompt the user to select whether secure printing will be available on a workstation for a specific printer.

Allow Users to Override

This option enables the end-user to select a secure release setting other than the default from the Printer Installer client.

In other words, if you select Allow Users to Override in the Admin Console, then the end-user can right-click the client icon in the task tray on their workstation, select Pull Printing > Secure Print Settings, and in the Secure Print Enabled column change the secure print setting for the printer as shown below.

Use default Pull Print setting

This option configures the selected printer object in the tree view to use the default Pull Printing settings in the Pull Printing section in Tools > Settings > Printing as shown below.

For more information, see Pull Printing.

Use printer-specific Pull Print setting

This option configures the selected printer object in the tree view to use the printer-specific Pull Print settings.

For more information, see Pull Printing.

Allow Pull Print jobs to be released to this printer

This option is available only when you select Use printer-specific Pull Print setting in the Printing tab. If you click this option, then the selected printer object in the tree view will be able to print pull print jobs. When a user sends a pull print job to the queue, the printer appears in the list of available printers to which the job can be sent. Printers that have the Allow Pull Print jobs to be released to this printer check box selected will appear in that list.

Pull Printing: Control Panel Application

This section enables you to configure the settings prior to installing the Control Panel Application.

Use default printer administration credentials

This option applies the printer's default administration username and password (used to log in to the printer's web interface) settings configured in Tools > Settings > General as shown below.

These login credentials are necessary for making changes to the printer itself by installing the Control Panel Application on the printer.

Use printer-specific administration credentials

This option applies the printer-specific admin username and password (used to log in to the printer's web interface) settings configured in the Printing tab as shown below.

These login credentials are necessary for making changes to the printer itself by installing the Control Panel Application on the printer.

Username/Password

This option is available only when you select Use printer-specific administration credentials.

The Username and Password fields enable you to specify the admin username and password required to access the printer's web interface, which is required to install the Control Panel Application on the printer.

Install Application on Printer

Click this link to install the Control Panel Application on the printer object selected in the tree view.

The installation will use either the default or the printer-specific administration credentials to log in to the printer's web interface, which will enable the installation to proceed.

Enable Badge Authentication

Click Enable badge authentication to add the badge swipe release feature to the Control Panel Application.

After you select this option, you can install the Control Panel Application and see the Login Option #2 as shown below.

If you install the Control Panel Application without the Badge Authentication option enabled, then you must uninstall the Control Panel Application, enable Badge Authentication, and then reinstall the Control Panel Application.

Use default Single Sign On settings

This option configures the selected printer object in the tree view to use the default Single Sign-on settings.

The default Single Sign-on settings are configured in Tools > Settings > General under the Control Panel Application section as shown below.

For more information, see Set Up Single Sign-on.

Use printer-specific Single Sign-on settings

This option configures the selected printer object in the tree view to use the printer-specific Single Sign-on settings

For more information, see Set Up Single Sign-on.

Disable single sign on

This option becomes available if you select Use Printer-specific Single Sign-on settings to enable or disable Single Sign-on for only the printer object selected in the tree view.

For more information, see Set Up Single Sign-on.

Pull Printing: Simple Badge Release Device

The settings in this section enable you to configure settings for a Simple Badge Reading device to enable IP printers that do not have an LCD control panel to be able to release pull print jobs with a badge swipe. For more information, see Simple Badge Release.

Enable the use of a simple badge release device with this printer

Select this option to enable Simple Badge Release on the printer associated with the printer object selected in the tree view.

Before you enable Simple Badge Release, you must Enable Pull Printing on a Workstation as described below.

  1. In the Admin Console click Tools > Settings > Printing.
  2. In the Pull Printing section, click Enable Workstation Pull Printing/Secure Printing.
  1. Click Save.
Elatec

Click this link if the Simple Badge Release device you are using is from Elatec. If you select this option, then the device settings will change to enable you to configure the Elatec device.

RFIDeas

Click this link if the Simple Badge Release device you are using is from RFIDeas. If you select this option, then the device settings will change to enable you to configure the RFIDeas device.

IP Address

Enter the IP address of the device so you can install the device settings from the Printing tab onto the actual device. When you click Configure RFIDeas Ethernet 241 or Configure Elatec TCPConv 2 (depending on your device), then the data in the fields under Device Settings will be applied to the Simple Badge Release device.

Username

This option appears in the Printing tab only if you have selected Elatec as the device you will be using for Simple Badge Release as shown below.

The username you enter here is the credential you would use to access the Elatec device.

Password

This option appears in the Printing tab only if you have selected RFIDeas as the device you will be using for Simple Badge Release as shown below.

The password you enter here is the credential you would use to access the Elatec device.

TCP Client Configurations

This option appears in the Printing tab only if you have selected Elatec as the device you will be using for Simple Badge Release as shown below.

Device Port

The Device Port is a read-only field set to 7777.

Printer Port

The Printer Port is a read-only field set to 7777.

Disconnect timeout

A Disconnect timeout is provided to service card readers or Keypads with no predefined "End" character. In this case the converter will close the connection 10 seconds after a character is received. The timer would reset on each character int he data stream and only 10 seconds after the last character would it actually close the connection. The converter is default configured to disconnect 10 seconds after the last character but the user can increase or decrease this time to fit their environment.

Initialization Settings

This option appears in the Printing tab only if you have selected RFIDeas as the device you will be using for Simple Badge Release as shown below.

The Initialization Settings are used the first time Printer Installer connects to the Simple Badge Release devices for multiple connection failures or to determine whether a connection is successful.

Retry Count:The total number of times the system will attempt to connect to the device.

Retry Sleep:The number of seconds between retry attempts.

Long Beep:The number of beeps you will hear (from the device) for a successful connection.

Short Beep:The number of beeps you will hear if the connection fails.

Data Settings

Data Settings are similar to the Initialization Settings, except that the Initialization Settings are used during the initial connection, while Data Settings are used in subsequent connections.

Retry Count:The number of attempts to print after a badge swipe.

Retry Sleep:The number of seconds between retry attempts.

Long Beep:The number of beeps you will hear (from the device) for a successful connection.

Short Beep:The number of beeps you will hear if the connection fails.

Which Print Jobs to release when a user's badge is swiped

Click this option to configure what the printer will release when the end-user swipes their badge over the Simple Badge Release device.

Last Print Job in the user's queue

If you select this option, then when the end-user swipes their badge over the Simple Badge Release device, the last print job that user sent to the printer will be printed.

All print jobs in the user's queue

If you select this option, then when the end-user swipes their badge over the Simple Badge Release device, all print jobs waiting in queue for that user to that printer will be released.

Configure Elatec TCPConv 2

This button appears only when you select Elatec as the device you are using as your Simple Badge Release device. Moreover, Printer Installer uses the Initialization Settings and Data Settings when connecting to the device.

Configure RFIDeas Ethernet 241

This button appears only when you select RFIDeas as the device you are using as your Simple Badge Release device. Moreover, Printer Installer uses the Initialization Settings and Data Settings when connecting to the device.

Mobile Printing

This section enables you to configure the select printer object in the tree view to use default or printer-specific settings for mobile printing. In other words, you can enable or disable mobile print jobs to be released on the printer object selected in the tree view. For more information, see Mobile Printing.

Use default Mobile Printing settings

If you select this option, then Printer Installer uses the default mobile settings in Tools > Settings > Printing under the Mobile Printing section as shown below.

Use printer-specific Mobile Printing setting

This option applies the Allow Mobile Print jobs to be released to this printer setting to the printer object selected in the tree view. If the setting is enabled, then the printer object can release mobile print jobs.

Allow Mobile Print jobs to be released to this printer. This printer can be configured as the user's Mobile Printer.

This option applies to the selected printer object in the tree view if Use printer-specific Mobile Printing setting is selected. If this option is selected, then the selected printer object in the tree view can release print jobs from mobile devices.

Mobile Printing: Direct Email Printing

This section enables you to configure settings for both Direct Email Printing and Direct Email Guest Printing. For more information, see Email Printing.

Direct Email Address

The email address added to this field appears is the designated email address used to send an email to a printer.

When adding the email address to this field, do not manually enter the domain; i.e. @yourcompany.com. Instead, click Tools > Settings > Printing, and in the Direct Email Address Domain field, type the domain (without the @) as shown below.

Use default Direct Email Printing setting

This option configures the selected printer object in the tree view to use the default Direct Email Printing settings configured in Tools > Settings > Printing as shown below.

For more information, see Email Printing.

Use printer-specific Direct Email Printing setting

If you click this option, then the selected printer object in the tree view will be configured to use the Allow print jobs to be emailed directly to this printer as shown below.

For more information, see Email Printing.

Direct Email Printing is disabled...

This message appears if Direct Email Printing has not been enabled.

To disable this message, click Tools > Settings > Printing and click both Enable Email Printing and Enable Direct Email Printing as shown below.

Allow print jobs to be emailed directly to this printer

If you click Use printer-specific Direct Email Printing setting, then this option applies to the selected printer object in the tree view. Click this option to enable or disable Direct Email Printing for the printer object.

For more information, see Email Printing.

Use default Direct Email Guest Printing setting

This option configures the selected printer object in the tree view to use the default Direct Email Guest Printing settings configured in Tools > Settings > Printing as shown below.

For more information, see Email Printing.

Use printer-specific Direct Email Guest Printing setting

If you click this option, then the selected printer object in the tree view will be configured to use the All print jobs to be emailed directly to this printer from guests.

For more information, see Email Printing.

Direct Email Guest Printing is disabled...

This message appears if Direct Email Guest Printing has not been enabled.

To disable this message, click Tools > Settings > Printing and click Enable Direct Email Printing, Enable Direct Email Printing, and Enable Direct Email Guest Printing as shown below.

Allow print jobs to be emailed directly to this printer from guests

This option applies to the selected printer object in the tree view if Use printer-specific Direct Email Guest Printing setting is selected.

If this option is selected, then the guest users will be able to send print jobs from an email to the selected printer in the tree view.

Security Tab Cheat Sheet

Use the Security tab in the Admin Console to set user access permissions to the Self-service Portal and to the Release Portal. The Security tab itself appears when you click a either a folder or a printer object in the tree view, so two options appear - Portal Security and Administrator Security. However, clicking an IP address range object in the tree view only provides a link to the Administrator Security options. For more information on setting Portal security, see Set Access Permissions for the Self-service Portal. Read More...

Access permissions refer to the ability for end-users to see (or not see) specific items you set in the Admin Console tree view. Essentially, you set up folders and printer objects in the Admin Console tree view, and then you use items such as Active Directory users, groups, computers, or IP address ranges, hostnames, and others to determine whether users can view the corresponding folders and printer objects in the Self-service Portal.

Consider the following example using the image shown here. In this case, the administrator has selected the Floor 34 folder in the tree view of the Admin Console, and then adds Active Directory computers and an IP address range to the folder. Furthermore, the administrator has also selected Include Inherited Security, which uses the parent folder's (Empire State Building) permissions for the Floor 34 folder. In this case, the Empire State Building folder has been assigned the Everyone permission setting, so the Floor 34 folder will also use that setting. Because these settings are defined in the Security tab, they will be used in the Floor 33 folder in the Self-service Portal, which means that all users (Everyone) will have access to this folder. The Portal Security options are available only if you select a folder or a printer object in the tree view.

The Administrative Security settings are available (along with the Portal Security settings) if you click a folder or a printer object in the tree view. If you select an IP address range in the tree view, the Administrator settings are available in a standalone tab. The fields of the Security Tab are described in detail below.

Portal Security

 

Include Inherited Security

This option applies permissions from a parent folder to a subfolder or printer object. For example, let's say you have three offices in San Francisco, so your tree view has a San Francisco parent folder and a subfolder for each office. Rather than configuring the settings on each subfolder individually, configure the settings on the San Francisco folder and then click Include Inherited Security for each subfolder so they all use the settings from the San Francisco folder.

NOTE: Inherited objects can affect the entire tree view. For example, if you set the Everyone object at the root level, then every folder and printer object in the tree view that has Include Inherited selected in the Security tab will include the Everyone object.

Object Name

The object to which you are providing permissions for a given folder or printer object in the tree view. The term "object" applies to both Active Directory objects such as users, groups, and containers, and non-Active Directory objects such as IP address range or hostname.

Assignment Type

This column shows the type of permission applied to each object (user, group, IP address range, hostname, etc) assigned to a selected folder or printer object in the tree view. This column shows

Inherited

If an object appears with Inherited in the Assignment Type column, then that object has been added to a parent folder and the Include Inherited check box is selected. In other words, the object is inherited from the parent folder.

NOTE: Inherited objects can affect the entire tree view. For example, if you set the Everyone object at the root level, then every folder and printer object in the tree view that has Include Inherited selected in the Security tab will include the Everyone object.

Explicit

If an object appears with Explicit in the Assignment Type column, then that object is not inherited from a parent folder. Rather, the object has been added to the Security tab (Portal Security) on the item selected in the tree view.

Add

Click Add to select the object (user, group, IP address range, etc.) that will be able to install the selected printer object in the tree view from the Self-service Portal.

For example, in the Admin Console, let's say you select a printer object in the tree view, and you want to give a specific user rights to install this printer from the Self-service Portal. You would click the Security tab, select Portal Security, and click Add to select the user's Active Directory account. The user will be added as an object to Security tab. The next time that user logs in to the Self-service Portal, they can click the same printer object and install the drivers for it.

Administrative Security

User Name

This field shows the Printer Installer user accounts (set up in Tools > Users) that are given a designated permission for the selected folder, printer object, or IP address range object in the tree view.

Assignment Type

This column identifies whether the user in the Administrative Security field is inherited from a parent folder higher in the tree view or if the user is set at on a specific object in the tree view.

Inherited

If a user appears with Inherited in the Assignment Type column, then that user has been added to the Security tab on a folder higher in the tree view. In other words, the user is inherited from a parent folder.

Explicit

If a user appears with Explicit in the Assignment Type column, then that user is not inherited from a parent folder. Rather the user has been added to the Security tab (Administrative Security) on the item selected in the tree view.

Role

In the Role field, you can select the roles, which are created in Tools > Users to apply to a user account for the selected printer object in the tree view.

When you apply a role to a user in the Administrative Security tab, you can click Tools > Users and see the role applied to the user account and the tree view folder for which the role is applied, as shown below.

Add

Click Add so that you can apply a role to a user or group for the object currently selected in the tree view.

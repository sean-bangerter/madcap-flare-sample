iOS Printing Tab Cheat Sheet

The iOS Printing Tab appears when you select a service host object in the tree view as shown in the image to the left. PrinterCloud uses the service host object rather than a printer object because PrinterCloud cannot directly connect to a printer that is behind a company's firewall.

Click Enable iOS printing first and then click None Selected to select a printer object as the virtual pull print queue to process mobile print jobs. After you select a printer object, the None Selected text will change to show the name of the selected printer object. For more information, see Mobile Printing.

The folder you select in the tree view is the folder to which the map will be associated. If you select another folder, a different map (or no map at all) will appear in the Map tab. For this reason, the tree view is recommended to be set up by location. For example, if your office is on the fourth floor of your building, then the tree view folder would be named Fourth Floor and a map that shows the layout of the fourth floor is added to the Map tab (making sure you have the Fourth Floor folder selected in the tree view. Furthermore, the printer objects in the Fourth Floor Folder will be available as printer icons in the Map tab as well, the purpose of which is to drag-and-drop those icons into the location on the map that represents their actual location on the Fourth Floor.

Both the folder (assuming it has not been set to be hidden in the General tab) and the map will appear in the Self-service Portal. The end-user can click the icon and easily install the printer. The fields of the Map Tab are described in detail below.

Enable iOS Printing

Clicking the option enables iOS mobile printing in your PrinterCloud environment. If iOS mobile printing is enabled, then PrinterCloud will use the Pull Print Queue selected in the iOS Printing tab.

None Selected

Click None Selected to select the pull printer that will be used as the queue for mobile print jobs. None Selected is the default text that displays on the button. After you select a virtual pull printer, the name of that pull printer will appear on the button.

User Authentication

The two options here provide basic security options for releasing print jobs. Require login for each print job requires the end-user to authenticate before the print job will release. The other option, Allow passwords to be saved on the device, does precisely what the name entails.

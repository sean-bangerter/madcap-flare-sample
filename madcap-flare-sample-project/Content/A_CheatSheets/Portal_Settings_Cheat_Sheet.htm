Portal Settings Cheat Sheet

The Portal Settings are used to set behavior in the Self-service Portal, which end-users take advantage of to install printer drivers on their workstation.

The Portal settings contain a link, which opens the Self-service Portal in a new browser tab. Moreover, other settings configure the way in which the tree view will appear when the Self-service Portal opens, depending on the end-user's subnet or IP address. Another setting installs the client on the end-user's workstation if it is not installed when the end-user opens the Self-service Portal for the first time. The Header section is used to customize the image that appears in the Self-service Portal header. The Portal Settings fields are described in detail below.

Printer Installer Portal URL

The URL that appears here is the same URL that you can enter into your browser's address bar to access the Self-service Portal. Additionally, the URL that appears here is a link that you can click to open the Self-service Portal, as shown below.

When you are adding your own logo to the Self-service Portal, this link will enable you to quickly see whether the logo you have added appears in the portal as you would like.

Options

Auto open to folder with printers in the same subnet as the end user

This setting will cause the Self-service Portal to automatically open the common parent of all printers in the same subnet as the end-user. The client knows the local IP address of the user and does a search in the database to find the printers in the same subnet. The client then opens the folder containing the most printers in the same subnet. If printer objects from multiple folders are in the same subnet as the ends user, the parent folder common to all of those folders will be the one opened.

NOTE: The printer port values must be IP addresses for this feature to function correctly.

Auto open to folder containing the IP Address Range object the end user is a member of

Opens the Self-service Portal tree view to the location of a specific IP Address Range object.

Enable Advanced Portal Security: Do not show folders and items that a user does not have rights to view / install

If you select this option, then the view/install permissions for a folder or printer (set in the Portal Security tab) will determine whether a folder or printer will appear in the portal for a specific user.

Enable "Driver Model" column

Shows or hides the Driver column in the Self-service Portal as shown below.

Password Protect

By enabling this option, end-users will be required to enter the specified password to gain access to the portal.

Header

This section enables you to change the header in the Self-service Portal. Many administrators add their organization's logo to the header. For more information, see Customize the Self-service Portal.

Header URL/File Path

Web address or file path of a page to show at the top of the portal. For example portal/header.html (default) or http://wwwyourdomain.com.

Header Height

Number of pixels used to display your logo in the Self-service Portal and the Release Portal.

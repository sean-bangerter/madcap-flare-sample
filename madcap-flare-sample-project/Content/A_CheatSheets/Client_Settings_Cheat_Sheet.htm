Client Settings Cheat Sheet

The Client Settings are available in the Admin Console by clicking Tools > Settings > Client. The image to the left shows an example of these settings.

The Client Settings are fairly general, and are used for such things as configuring the client to "check in" with the Printer Installer server to check Active Directory for new users or group membership, synchronizing deleted printers from the Admin Console to also be deleted on end-user's workstations, synchronizing driver and profile updates, and even downloading the latest client files to deploy to end-user workstations. The Client Settings provide so many options for keeping the settings on client machines in sync with the settings in the Admin Console. The fields of the Client Settings are described in detail below.

Deployable Client Packages

Click the button corresponding to the appropriate operating system to download the client installer for that operating system. The downloaded file can be used to manually install the client on a workstation, or the file can be used with software management tools to automatically push out the client to workstations.

Client Check-in

Enable client check-in at startup and login

This option enables the client to "check in" with the Printer Installer server at a specific frequency. This option is always enabled because you cannot turn off check-in at startup and still log in.

Enable client check-in every 'x' minutes

This option enables you to change the frequency that the client checks in with the Web Service. The default check-in frequency is four hours (240 minutes). The lowest recommended frequency for a client check-in 60 minutes because the lower you set the frequency, the higher your environment traffic will be. If you do not select Enable client check-in every 240 minutes, then the client check-in will occur every 240 minutes.

TCP/IP & Windows Shared Printer Settings

Sync Delete

If you delete a printer object from the tree view, then the printer associated to the printer object will be deleted from end-user workstations that have the printer installed. The deletion will occur the next time the end-user logs into the Self-service Portal or the Release Portal.

For example, you have the Printer A printer object listed in the tree view in the Admin Console. Moreover, John, Mary, and Chad all have the drivers and profiles assigned to the Printer A printer object installed on their workstation. One day, you decide you want to remove the Printer A printer object from the tree view. When you delete the printer object, Printer Installer will automatically remove the drivers and profiles assigned to the Printer A printer object from all three employee's workstations the next time each of them logs into or refreshes the Self-service Portal or the Release Portal. For more information, see Sync Deleted Printers.

Sync Deployment

If you select a printer object in the tree view, click the Deploy tab, and delete a user, group, computer, IP address, hostname, or other type, then the end-users corresponding to the item you deleted in the Deploy tab will have the printer associated to the printer object removed from that end-user's printer.

For example, you select the Printer A printer object in the tree view, and then you click the Drivers/Profiles tab to see which drivers and profiles are assigned to the Printer A printer object. Furthermore, you then click the Deploy tab, and you see a line item for John Smith. With this information, you know that John Smith had a driver and profile automatically assigned to his workstation. By clicking Sync Deployment and then deleting John Smith from the Deploy tab, then the next time John logs in to the Self-service Portal or the Release Portal, or he refreshes either portal in the browser, then Printer Installer will automatically remove the driver and profile from the Printer A printer object on his workstation. For more information, see Sync Printer Deployments.

Keep Existing Printers

This option enables any end-users who have a driver or profile installed on their printer prior to the end-user being added to the Deploy tab to retain the driver and profile even if their name, group, computer, etc has been removed from the Deploy tab.

For example, you (as the administrator ) have the Printer A printer object in the tree view, with drivers and profiles set in the Drivers/Profiles tab. You had John install the driver a month ago, and two weeks ago you added Chad to the Deploy tab for the Printer A printer object. Today, you remove the printer deployment in the Deploy tab, so the next time Chad logs into the Release Portal or the Self-service Portal, Printer A is removed from his workstation. However, because John had the printer installed before the printer was deployed, his workstation is not impact when the printer deployment is deleted.

TCP/IP Printer Settings

Automatically manage all end-user TCP/IP printers

Click this option if you have implemented Printer Installer and an end-user already has one or more printers installed on their workstation, but you want Printer Installer to start managing the printer on that workstation.

Enable auto-management item-level targeting

Click this check box if you would like to restrict client updates to a subset of the computers with the client installed.

Show Filter

Click Show Filter to add the object types (users, computers, groups, organizational units, IP address ranges, etc) you would like to convert.

Create connections to shared TCP/IP printers

Click this option to install printers from a print server that is being managed by Printer Installer.

Only apply in terminal services sessions

If you select this option, only printers that are installed in Citrix or a terminal services session will be installed. Printers installed on workstations will be TCP/IP.

Allow direct IP printers to be installed from the portal

If you select Create connections to shared TCP/IP printers in the Client Settings, but Printer Installer cannot find a print shared server deployment for the printer you are attempting to install, then Printer Installer will instead create a TCP/IP printer.

NOTE: Shared server deployments can be found in the Deploy tab with the Shared check box selected.

If this check box is not selected, then Printer Installer will generate an error message rather than creating a TCP/IP printer.

Sync General/Port

If you make changes in the Admin Console General tab or Port tab for a printer object, then by clicking this check box, any workstations to which the printer object have been installed can be synchronized to make the same changes the next time the end-user logs in to the client or refreshes their browser.

Sync Driver

Click this check box if you want drivers updated in the Admin Console to automatically be updated on workstations on which they are installed the next time the end-user logs in to the client or refreshes their browser.

Sync Profiles

These settings apply to scenarios in which the end-user made changes to the profile on their workstation, and you want to reapply the profile you set up back to their workstation. However, these settings are global and apply only if the printer object is configured to use the global setting in the Drivers/Profiles tab. Additionally, you must have Sync Profiles enabled for the following two options to be available.

Reapply profiles every login/client check-in or refresh

Click this check box to apply the default profile to a workstation the next time the end-user logs in to the client or refreshes their browser. Note that this option will be available only if you select the Sync Profiles check box. For more information, see Reapply a Profile.

Reapply profiles after each print job

Click this check box to apply the default profiles to a workstation the next time the end-user sends a print job to a printer. Note that this option will be available only if you select the Sync Profiles check box. For more information, see Reapply a Profile.

Allow non-administrators to modify/delete TCP/IP printers

Click this check box if you want end-users to be able to modify the settings on TCP/IP printers installed through the Admin Console.

Allow end-users to capture printer ports for legacy applications

Click this check box if you are using printer ports such as LPT1 or LPT2, but you want to be able to output to a TCP/IP printer.

Windows Shared Printer Settings

Create/Delete Global Printers (Per Workstation)

This option applies only if you are using Windows Print Server Links to install printers from a print server. Click this option if you want any printer installed from the portal to be automatically available to all users who log into the workstation (not recommended).

Create/Delete Global Printers (Per User)

This option applies only if you are using Windows Print Server Links to install printers from a print server. Click this option to maintain the printers installed from the portal to be available only to the users who installed them (default).

Remove end-user printers connections from roaming profiles

Select this option to remove all shared printers from your roaming profiles. This feature can be useful when users log in to workstations at different locations.

Remove all end-users printers connections when a user logs in to a terminal server

Select this option to remove all shared printers when a user logs in to a Citrix session or Terminal Services session.

Client Update

These settings enable you to configure Printer Installer to update the client installed on workstations.For more information, see Update the Client.

Enable automatic update during the next client check in or refresh

Click this check box to enable the client to automatically update the next time the client checks in with the server for any reason, namely the frequency set in the Client Check-in, a client check-in or refresh, a manual update, etc.

Enable auto-deploy item level targeting

Clicking this check box enables you to deploy a client update to a targeted group of users or computers.

Show Filter

Click this option to set the filter that will be used for targeting the client update. Only computers that are in the filter list will automatically update the client.

Update Client (Windows, Mac OS, and Linux

Click the button corresponding to your operating system to download the client update. By downloading the client update, you can push the update out to workstations on which the client is deployed. For more information, see Upgrade the Client, more specifically the Auto Update Client (options 1 and 2).

Home URL Security

The Home URL Security section enables you to switch between Printer Installer servers without needing to manually change the HomeURL and reinstall the client.

If the HomeURL on your machine is not pointing to a supported Home URL, then you will see the following image (or some variation thereof).

For more information, see Create a Whitelist to Prevent HomeURL Changes.

Prevent web pages from automatically changing the end-user's Home URL

Click this option to enable the end-user to switch between servers without having to manually change the HomeURL and reinstall the client.

Add

Click Add to enter the URL of a new server that you want to add to the Home URL list. When the URL is added to the Whitelist, the end-user would be able to switch to that server without having to reinstall the client on their workstation.

Remove

Click Remove to eliminate the URL of a server from the Whitelist. If the URL is removed from the Whitelist, then the end-user would need to uninstall the client from their workstation, change the HomeURL in the config file, and then reinstall the client.

Also allow the end-user's Home URL to be changed automatically to any licensed URL:

This option can be used to add the servers to a new license file. When you replace the old license file with the new one, the HomeURLs added to the license file will appear in the Whitelist (beneath the Also allow the end-user's Home URL to be changed automatically to any licensed URL check box).

Novell NDPS/iPrint Printer Conversions

Enable Printer Installer Client to convert end-user's Novell printers that have been copied into Printer Installer as TCP/IP printers or copied to Windows print servers

After you have imported NDPS, iPrint, or legacy queue printers into the Administrator using the Printer Import, use the Printer Installer Client to convert your end user’s NDPS, iPrint, or legacy queue printers to Printer Installer managed TCP/IP printers. Use the options on this page to enable the desired Novell printer conversions. Once the printer conversions are enabled the Client will convert the end user printers during the next login.

Enable Novell conversion item-level targeting

Enable this option if you imported printers from Novell and want to convert the existing end user Novell printers to the newly created printer. When this option is enabled the Client will convert NDPS/iPrint printers that have been imported from Novell to Printer Installer managed TCP/IP printers. Only NDPS printers that were actually imported with the Printer Import will be converted. For example if NDPS_Printer1 and NDPS_Printer2 are installed on an end user computer, but only NDPS_Printer1 was imported, then only NDPS_Printer1 will be converted to a TCP/IP printer managed by Printer Installer. Note that this setting applies only to end-user computers only. It does not affect the Novell server, NDPS printer objects in eDirectory, or Netware in any way.

Show Filter

Click this option to set the filter that you will be used to restrict the computer that will convert their Novell printers.

Convert end-user Novell NDPS printers

Same as the above Enable Printer Installer Client to convert Novell printers option, but specifies that NDPS printers can be converted on end user computers. Note that this setting applies only to end user computers only. It does not affect the Novell server, iPrint printer objects in eDirectory, or Netware in any way.

Convert end-user Novell iPrint printers

Same as Enable Printer Installer Client to convert Novell printers option (above), but enables iPrint printers to be converted on end user computers. Note that this setting applies only to end user computers only. It does not affect the Novell server, iPrint printer objects in eDirectory, or Netware in any way.

Change default printer after converting existing default printer

If when converting NDPS/iPrint printers the NDPS/iPrint printer was marked as the default printer, mark the new Printer Installer managed TCP/IP printer as the default printer.

Delete converted printers on end-user workstations

Enabling this option will instruct the Printer Installer Client to delete the NDPS or iPrint printer once it has successfully created the new Printer Installer managed TCP/IP printer. Only NDPS or iPrint printers that were actually imported with Printer Migrator, will be deleted. For example if NDPS_Printer1 and NDPS_Printer2 are installed on an end user computer, but only NDPS_Printer1 was imported with Printer Migrator, then only NDPS_Printer1 will be deleted after the new Printer Installer managed TCP/IP printer is installed. Note that this setting applies only to end user computers only. It does not affect the Novell server, NDPS/iPrint printer objects in eDirectory, or Netware in any way.

Delete all remaining Novell printers on the end-user computer

Be careful with this option. This option instructs the Printer Installer Client to delete all remaining NDPS/iPrint printers on the end user's computer after all printers that have been imported using Printer Migrator, have been converted. This option is very useful for removing all the remaining NDPS/iPrint printers that were not imported with Printer Migrator. This option can be enabled during the initial printer conversion, or days later to clean up any remaining NDPS/iPrint printers.

Disable NDPS Remote Printer Management (RPM) automated NDPS printer deployments

Click this option to disable the Novell tool that deploys printers to end-users following the printer migration from Novell to Direct TCP/IP.

Use existing driver and migrate existing profile (only applies to printers imported as TCP/IP printers

This option instructs the Printer Installer Client to use the existing driver and profile (device settings and preferences; duplex option, paper trays, separator page, paper size, etc…) from the NDPS/iPrint printer when creating the Printer Installer managed TCP/IP printer. If later on you update the Printer Installer managed TCP/IP printer object it will update the end per-user printers if you have it set to do so. This setting makes the conversion of end user NDPS or iPrint printers to TCP/IP printers as seamless as possible.

This feature is not supported with iPrint per-user printers; only with iPrinter per workstation printers. The print processor is missing for iPrint user printers and requires that the drive be installed completely. Additionally, if this feature is enabled, the Printer Installer driver and profile will not be downloaded and applied from the server.

Microsoft Printer Conversions

Enable Printer Installer Client to convert end-user's Novell printers that have been copied into Printer Installer as TCP/IP printers or copied to Windows print servers

After you have imported the printer from a Microsoft printer server into the Administrator using the Printer Import, use the client to convert your end user’s Microsoft shared printers to Printer Installer managed TCP/IP printers. Use the options on this page to enable the desired printer conversions. Once the printer conversions are enabled the client will convert the end user printers during the next login.

Enable Microsoft Conversion item-level targeting

When this option is enabled, only computers that are listed in the filter (either directly by AD computer object or indirectly by AD group/OU or IP Address Range) will perform automatic conversions. This is useful if you want to deploy the client to all of your workstations, but want to slowly roll out the printer conversions.

Show Filter

Click this option to set the filter that you will be used to restrict the computer that will convert their Microsoft printers.

Change default printer after converting existing default printer

When converting printers, if the printer was marked as the default printer, mark the new Printer Installer managed TCP/IP printer as the default printer.

Delete user's original Microsoft printer after successful conversion

Enabling this option will instruct the client to delete the original printer once it has successfully created the new Printer Installer managed TCP/IP printer.

Write Filter Settings

Click Disable clients installed with write filter support if the Printer Installer client on the write-filtered computers is causing an infinite reboot loop. This option will stop the client from rebooting the computer as part of its check-in (when attempting to disable/enable the write filter to install printers).

General Tab Cheat Sheet

The General tab appears in the Admin Console after you select a folder, printer object, or IP address range object in the tree view. The options that appear vary depending on the selection in the tree view. For example, when you select a folder in the tree view, the General tab displays only the Folder Name field and the Hide from Portal field.

The image to the left shows the General tab as it appears when you select a folder in the tree view. The Folder Name field appears only when you select a folder and is not available when you select a printer object or an IP address object. Still, the Hide From Portal option is available when you select a folder or a printer object. Read More...

The image to the right shows the General tab when you select a printer object in the Admin Console tree view. Consider the added fields that appear when you select a printer object as opposed to when you select a folder. These fields take care of a variety of needs for the administrator and the end-user. For example, the information you enter in the Location and Comment fields will appear in the Self-service Portal for the end-user to see additional information you want to provide to them for installing those specific printers on their workstation.

The Web Interface and Install URL links are added to the General tab just for simplicity's sake, to provide quick access to the printer's Web Interface settings and to the same link that enables end-users to install the printer from the Self-service Portal. Some of the other options in the General tab enable you to automatically (via SNMP) or manually set the printer object to support color or dual-sided printing. Keep in mind that you can set the printer object to support black and white printing only for a printer even if the printer supports color printing. Note that you are not actually setting the printer to print color or black and white. You are just identifying the printer as a color or a black and white printer.

A couple of other useful options in the General tab that appear when you select a printer object in the tree view include the Print Job Costing field, which enables you to select the template you want to determine the costs you want to apply to specific paper sizes, so Printer Installer can report correctly on your printing costs. Additionally, the Hide from Portal option enables you to hide a printer object from appearing in the Self-service Portal. Let's say you have a printer that you do not want to make available for general availability in your office, then you can click Hide from Portal. For example, if you have a printer that generates checks, then you most likely want that printer to be available only to your employees in Accounting. You can hide the printer from the Self-service Portal but deploy the printer to specific users through Active Directory.

The fields that appear when you select an IP address range in the tree view are specific to setting up an IP address range for the purposes of caching. The image to the left shows the settings that appear in the General tab when you select an IP address object in the tree view. The name and range itself are pretty straightforward. The Cache Location is used to set the computer located at the remote location and is used to stored the drivers and profiles, and the Update cache files now link pushes out updates to the computer being used for storing cached files. For more information, see Cache Drivers.

In addition to the many fields you would see when you select a folder, printer object, or IP address object in the tree view, the other tabs that appear vary as well. Moreover, if a printer object is selected as a print queue, then some of the fields in the General tab are replaced by the following text: This is a pull printer. This setting does not apply.

In addition to the fields described above, the General tab displays custom fields that you can create manually. These fields are generated in the Admin Console via Tools > Settings > General under the Custom Printer Fields section. These fields can be given any name and type (text, numeric, or Yes/No) and appear only when you select printer objects in the tree view. The fields of the General Tab are described in detail below.

Folder Name

The Folder Name field appears only if you have the root folder or another folder selected in the tree view. This field contains the name of the folder that you have actively selected in the tree view.

Hide From Portal

The Hide from Portal field appears when you select either a folder or a printer object in the tree view. This field enables you to hide a printer object from the Self-service Portal, the result of which is to prevent end-users from being able to install a printer.

Printer Name

The Printer Name field appears only if you have selected a printer object in the tree view. This field contains the name of the printer object that you have actively selected in the tree view.

Location

The information entered into the Location field in the Admin Console appears in the Location field of the Self-service Portal as shown below.

Comment

The information entered into the Comment field in the Admin Console appears in the Comment field of the Self-service Portal.

Web Interface

The Web Interface field displays a link that takes you to a printer-hosted, manufacturer-defined web page that enables you to manage a printer's settings from the web. Consider this web interface example for an HP printer.

Install URL

The hyperlink that appears in this field opens the Self-service Portal immediately and displays the installation wizard for the printer associated with the printer object you have selected as shown below.

Supports Color

This option enables the printer object to support or prohibit color printing. You can even set a printer to support only black and white printing even if the printer, by default, does support color printing. This drop-down menu provides three options, described below.

Unspecified

Click this option to set up a new printer object in the tree view. Additionally, you can right-click the root folder or any subfolder to see an option to add a new TCP/IP printer.

Yes

If you are not using SNMP to regularly check a printer for status updates, then you can manually set a printer to support color printing by selecting Yes in the Supports Color field. Alternatively, if you are using SNMP to check a printer for status updates, then you can set the Supports Color field to Unspecified, and if the printer does support color, then the Admin Console will automatically change the Supports Color field from Unspecified to Yes.

No

If you are not using SNMP to regularly check a printer for status updates, then you can manually set a printer to support black and white printing only by selecting No in the Supports Color field. Alternatively, if you are using SNMP to check a printer for status updates, then you can set the Supports Color field to Unspecified, and if the printer does not support color, then the Admin Console will automatically change the Supports Color field from Unspecified to No.

Supports Duplex

The Supports Duplex settings appear in the Admin Console General tab only when you select a printer object in the tree view and are used to determine whether the printer associated with the printer object supports duplex printing (double-sided printing). Keep in mind that even if you set the printer object to support duplex printing, and the printer itself does not support duplex printing, then you will not be able to print duplex printing.

The Supports Duplex drop-down contains three possible selections. One option enables Printer Installer to use the SNMP value returned from the printer. You can see the Duplex value in the Status tab if you have SNMP set up correctly. The other options enable you to manually set the value to Yes or No. The following sections provide greater insight into each of these available options.

Unspecified

If you select Unspecified in the Supports Duplex field, then the Admin Console auto-populates the field to Yes or No depending on the Duplex value the printer returns (using SNMP Status Monitoring).

Yes

If you are not using SNMP to regularly check a printer for status updates, then you can manually set a printer to support duplex printing by selecting Yes in the Supports Duplex field. Alternatively, if you are using SNMP to check a printer for status updates, then you can set the Supports Duplex field to Unspecfied, and if the printer does support duplex printing, then the Admin Console will automatically change the Supports Duplex field from Unspecified to Yes.

No

If you are not using SNMP to regularly check a printer for status updates, then you can manually set a printer to not support duplex printing by selecting No in the Supports Duplex field. Alternatively, if you are using SNMP to check a printer for status updates, then you can set the Supports Color field to Unspecified, and if the printer does not support duplex printing, then the Admin Console will automatically change the Supports Color field from Unspecified to No.

Print Job Costing

The Print Job Costing field enables you to select the template containing the paper cost values you would like to apply to the printer associated with the printer object you have selected in the Admin Console tree view. The template you select has specific values applied to explicit paper types (letter, legal, executive, A0-A10, and several others). Each cost you apply is for a single side page only. It does not apply to duplex printing.

IP Address Range Name

The IP Address Range Name field appears when you select an IP address object in the tree view. This field displays the name given to the IP address object selected in the tree view.

IP Address Range

The IP Address Range field shows the two IP addresses in which the IP address range is set. This range is a grouping of consecutive IP addresses, the purpose of which is for setting up caching or deploying printers. If you set up the IP address range for caching, it is recommended that you also provide information in the Cache field.

Cache

The fields in the Cache section enable you to select the computer at the remote location where cached drivers, profiles, and other objects are stored, while the Update cache files now link sends the cached files to that computer.

Cache Location

The cache location is the computer at the remote location on which the drivers, profiles, and other objects will be stored.

Update Cache Files Now

Click this option if you want to manually update the drivers and profiles currently stored at the cache location.

Redirect clients in this IP address range to obtain resources from the cache location instead of the Printer Installer server

Click this option to force the workstations with IP addresses in the IP Address Range field to get drivers and profiles from the cache location.

Synchronize data files to the specified cache location

Click this option to automatically update the drivers and profiles in the cache location. If this option is not selected, then you will need to click Update cache files now to update drivers and profiles at the cache location.

Service Host Name

The Service Host Name is a unique name given to the Service Host, which is a server or workstation PrinterCloud uses to install mobile printing and email printing services on a workstation that has the PrinterCloud client installed.

IP Address or Hostname

The IP address or hostname of the computer that is being used as the Service Host for mobile printing and email printing for PrinterCloud.

Service Host OAuth

This button is used as a fail-safe in the unlikely event that the Service Host is not communicating with the authorized clients on the workstations.

NOTE: Clicking this option will completely invalidate the existing tokens on the workstations, so please be careful before using the Service Host OAuth feature.

Status Tab Cheat Sheet

The Status tab appears in the Admin Console when you select a printer object in the tree view as shown in the image to the left. The data that appears in the Status tab is a result of the default or printer-specific configuration settings you apply in the Settings section of the Status tab. If you opt to use the default settings, then Printer Installer applies the settings from Tools > Settings > General under the SNMP Status Monitoring section. Printer Installer, by default, uses SNMP v1/v2unless v3 is available on the printer.

The read-only information in the Status tab comes from status data obtained from the printer itself. You will notice such information as the date and time of the last status check, the printer's serial number and model number, any errors that have occurred, color and duplex printing support, the printer's IP address, the total number of pages the printer has released, and many other information types. Read More...

All the information in the Status tab is a result of communication between the printer and Printer Installer. The printer is able to communicate with the Printer Installer server directly using SNMP settings. If the communication is successful, the Status tab will be populated with data from the printer. If the communication is not successful, the monitoring frequency changes depending on the settings you configure in Tools > Settings > General as shown in the image to the right.

To get up-to-date printer information, click the Refresh button at any time. The fields of the Status Tab are described in detail below.

ICMP

NOTE: To improve accuracy in determining if a printer is accessible, Printer Installer no longer uses the ICMP protocol even though this section is named ICMP. Instead, Printer Installer connects to a printer over TCP 9100, which is not the same as a ping. As a result, you may have different results with a ping than the result you see on the Status tab. Additionally, you can change the TCP port from the default 9100 in the Port tab.

Ping Status

This field identifies whether Printer Installer can communicate with (and release print jobs to) a printer. The printer will return Success or Failure, which will appear in the Ping Status field.

Last Attempted

The last date and time the system attempted to connect to the printer.

SNMP

When Printer Installer is able to connect to a printer, then the status information returned from the printer appears in this section. If Printer Installer cannot connect to the printer, then the Ping Status will return as Failure, and the SNMP settings will return as Not Available.

Last Updated

The last date and time that the SNMP settings were modified.

Serial Number

The serial number belonging to the connected printer.

Console Message

Message that displays on the printer's console the last time it was checked.

Operational State

The options available in this field vary by printer manufacturer, and some printers may not even provide all of these options. See your printer manufacturer's documentation for definitions of the available operational states.

Available operation states include:
  • Unknown
  • Running
  • Warning
  • Testing
  • Down
Current Status

The options in this field vary by printer manufacturer, and some printers may not even provide all of these options. See your printer manufacturer's documentation for definitions on the available current statuses.

Available current status states include:
  • Unset
  • Other
  • Unknown
  • Idle
  • Printing
  • Warmup
Error Condition

The options in this field vary by printer manufacturer, and some printers may not even provide all of these options. See your printer manufacturer's documentation for definitions on the available error conditions.

Available error condition states include:
  • ICMP Host Unavailable

If you receive the ICMP Host Unavailable error condition, then either the printer is not on or is not connected to the network, or there is a problem with the firewall (blocking port 9100 from the Printer Installer server.

  • Low Paper
  • No Paper
  • No Toner
  • Door Open
  • Jammed
  • Offline
  • Service Requested
  • Input Tray Missing
  • Output Tray Missing
  • Marker Supply Missing
  • Output Nearly Full
  • Output Full
  • Input Tray Empty
  • Overdue Preventive Maintenance
  • Other
Model

The model number of the printer connecting to Printer Installer via SNMP.

Color

The ability of the printer to print in color or in black and white only.

Duplex

The ability of the printer to print on single side or on both sides of the paper.

Uptime

The amount of time since the last time the printer was down.

Hostname

Network name the IT Department has given to the device.

IP Address

IP address of the printer.

MAC Address

Network address of the printer

Lifetime Page Count

The number of pages printed in the life of the printer. Printer manufacturers vary in whether a printer includes multi-function options such as copy or fax in the lifetime page count. See your printer manufacturer's documentation for more information.

Use Global

This section enables you to configure the selected printer object in the tree view to use the default Status settings in Tools > Settings > General as shown below.

For more information, see Configure a Printer Object for Status Monitoring.

Use Printer-specific

This option configures the selected printer object in the tree view to use the printer-specific SNMP settings.

For more information, see Configure a Printer Object for Status Monitoring.

Enable SNMP Status Monitoring

This option configures the selected printer object in the tree view to use the printer-specific Status Monitoring setting. For this field to be available, select Use Printer-specific in the Settings section of the Status tab.

For more information, see Configure a Printer Object for Status Monitoring.

SNMP v1/v2

Community Public (Get) Name

Access name provided to the printer to retrieve SNMP information from the printer. This field is applicable only if you select Use Printer-Specific in the Settings section.

Community Private (Set) Name

Access name provided to the printer to write SNMP settings to the printer. This field is applicable only if you select Use Printer-Specific in the Settings section.

SNMP v3

Use SNMP v3 When Supported

Permits the system to use SNMP v3 when it is supported by the printer to which it is connecting. If SNMP v3 is enabled, the system will use v3 first. If it cannot connect to the printer, it will use v2/v1. This check box will be available if you select Use Printer-Specific in the Settings section.

Username

If authentication is enabled in the Security Level field, then the system will ask for a username, and you would enter the value entered in this field. This field is applicable only if you select Use Printer-Specific in the Settings section.

Security Level

Setting that determines the type of security used in the SNMP connection. Options include both authentication and privacy, authentication but no privacy, and no authentication or privacy. This field is applicable only if you select Use Printer-Specific in the Settings section.

Authentication Protocol

The Authentication Protocol refers to the hashing algorithm used to store your authentication key. Options include MD5 and SHA. If you select a security to include both authentication and privacy or authentication but no privacy, then you will need to add an authentication protocol. This field is applicable only if you select Use Printer-Specific in the Settings section.

Authentication Key

If you select a security level to include both authentication and privacy or authentication but no privacy, then you will need to add an authentication key. It is similar to a password for the authentication protocol. This field is applicable only if you select Use Printer-Specific in the Settings section.

Privacy Protocol

If you select a security level to include both authentication and privacy, then you will need to add a privacy protocol. Options include DES and AES, and is an encryption algorithm used to secure communication between the added device and the printer installer server. This field is applicable only if you select Use Printer-Specific in the Settings section.

Privacy Key

If you select a security level to include both authentication and privacy, then you will need to add a privacy key. It is similar to a password for the privacy protocol. This field is applicable only if you select Use Printer-Specific in the Settings section.

Context Name

An SNMP context name or "context" in short, is a collection of management information accessible by an SNMP entity. An item of management information may exist in more than one context. An SNMP entity potentially has access to many contexts. In other words, if a management information has been defined under certain context by an SNMPv3 entity, then any management application can access that information by giving that context name. The "context name" is an octet string, which has at least one management information. This field is applicable only if you select Use Printer-Specific in the Settings section.

Context Engine ID

Within an administrative domain, contextID uniquely identifies an SNMP entity that may recognize an instance of a context with a particular context name. Most of the SNMPv3 entities have the snmpEngineID as their contextID. This field is applicable only if you select Use Printer-Specific in the Settings section.

Release Tab Cheat Sheet

The Release tab appears in the Admin Console when you select a printer object in the tree view. This tab displays the print jobs that end-users have sent to the selected printer. The administrator can also take administrative action and release or delete print jobs at will, regardless of the person who submitted the print job or the print job's place in the list of print jobs to be released.

The Release tab for a standard printer object displays any Secure print jobs that have been sent to that specific printer. The Release tab for a pull print queue selected in the tree view displays all pull print, mobile print, email print (not including Direct Email Guest to Print) waiting to be released by the end-user. Read More...

Deleting print jobs from the Release tab is useful for an administrator when an end-user has submitted a big print job that is using a high number of resources. Not only is the print job using all your ink and paper, but it is also backing up the queue. For example, an end-user submits a 600-page PDF to the printer. Another instance in which the Delete feature in the Release tab is useful is when an end-user has submitted a print job that is not company related, so you do not want the print job using valuable company resources.

Print jobs in the Release tab appear in the order in which they have been released by the end-user. The administrator can select a specific print job and then click Release to release the print job. If the printer is currently printing a print job, then that print job will finish printing, and then the print job released by the administrator will start printing. When the print job finishes printing, then the first print job in the Release tab will begin printing.

Print jobs remain in the Release tab for 12 hours, by default. However, you can change the amount of time a print job stays in the Release tab by clicking Tools > Settings > Printing, and in the Purging section, change the value in the Purge held jobs after 'X' hours option. The fields of the Release Tab are described in detail below.

Release

Job Type

This field indicates whether the print job is sent directly to a printer (Secure Release) or to a pull print queue via a pull print, Google Cloud Print, iOS print, or email to print job.

Computer

The workstation used to release the print job.

User

The end-user who released the print job.

Title

The file name of the document being printed.

Status

The current state of the print job, for example, selecting a print job and clicking Release displays the status as Release Initiated.

Pages

The number of printed pages that the printer will output for a given print job

Size

Size of the file being printed.

Submitted

The date and time the print job was submitted for release.

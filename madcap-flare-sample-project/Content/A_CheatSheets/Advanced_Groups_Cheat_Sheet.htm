Advanced Groups Cheat Sheet

The Advanced Groups settings appear in the Admin Console when you click Tools > Advanced Groups. The image to the left shows the Advanced Groups settings as they appear in Printer Installer.

Advanced Groups are used specifically for printer deployments to workstations. Click Add to set up a new Advanced Group, which requires both a name and a description. After you add an Advanced Group, a line item will appear in the Advanced Groups settings. Read More...

For each Advanced Group, set as many "rules" as needed. To set up a rule, click the plus sign (+) and set the operation as Contained In or Excluding and then select the value for that rule. The rules you add to the Advanced Group will narrow the number of workstations to which a printer will be deployed. For example, you have a rule that has Contained In as the operation and the value set to the Marketing Group, and you have a rule that has Excluding as the operation and John Jones as the value.

When you have finished setting up your Advanced Group, you can select a printer object in the tree view and apply the Advanced Group to the Deploy tab. The printer will be deployed to the Marketing Group but not to John Jones' workstation even though he is part of that group. The fields of the Advanced Group settings are described in detail below.

Add

Click Add to set up a new Advanced Group, which enables you to create rules that combine such things as Active Directory objects, IP address ranges, hostname, etc. for printer deployment. When you click Add, just provide a name and description for the Advanced Group and then click Save. For more information, see Deploy a Printer.

Delete

Click the check box to the left of one or more Advanced Groups and then click Delete. Your Advanced Group will be removed from Printer Installer. If you delete an Advanced Group that is currently assigned to a printer deployment, then the Advanced Group will automatically be removed from the list in the Deploy tab.

Operation

The Operation field enables you to select whether to include or exclude a specific value for the deployment. For example, let's say you select Contained In for the Operation, and then you select John Smith, an Active Directory user for the value. In this case, the printer deployment will include John Smith.

Contained In

If you select Contained In, then the printer will be deployed to the item selected in the Value field. For example, let's say you select Contained In as the operation, and in the Value field you select Active Directory User, Computer, or Group. The Active Directory user you select is John Smith. In this case, when the Advanced Group is applied to a printer deployment, then the printer will be installed on John Smith's workstation.

Excluding

If you select Excluding, then the printer will be prevented from being deployed on the selected item in the Value field. For example, let's say you set two operations. The first one is Contained In, and you set the value to Marketing, which is an Active Directory group. The second operation is set to Excluding, and you set the value to John Smith. In this case, when the Advanced Group is applied to a printer deployment, then the printer will be installed on workstations for users in the Marketing group with the exception of John Smith.

Value

The value you select here identifies the item that will be included in or excluded from (depending on the operation you select) the printer deployment. For example, let's say you set two operations. The first one is set to Contained In, and the value is set to IP Address Range. You would select an IP address range object from the tree view. The second operation is set to Excluding, and the value is set to Active Directory User, Computer, or Group. You would select John Smith as the user.

Plus Sign (+)

The Plus Sign (+) button is used to create additional rules for an advanced group. Keep in mind, however, that the more rules you add to an Advanced Group, the more narrow the printer deployment will be.

Failover

Secure Release Printing

Failover ensures Printer Installer availability in the event of the primary server's failure. For Printer Installer, there are two failover options available.

Warm Standby failover uses both a primary and a secondary server, both of which have Printer Installer, a database, and a Printer Installer license. This type of failover is relatively simple to set up and manage.

Put simply, the primary server pushes a backup to the secondary server once per day. If the primary server ever fails, then the administrator can point the DNS to the backup server's IP address to get back up and running. There are third-party applications that will automate this process.

The issue, however, with warm standby failover is one of timing. If you are not using the third-party application to switch to the backup server automatically, then depending on when the failure occurs and when the backup is supposed to run, you can be left with several hours of lost data. For example, let's say the failure occurs at 4 a.m., but the backup does not run until 5 p.m., then you have over almost 12 hours of data. The benefit of Warm Standby failover is that it is simple to set up to ensure you have failover, but if you want to ensure that data will never be lost, then you might want to consider High Availability failover.

The High Availability failover eliminates the potential for lost data by using multiple nodes to automate the failover. The process for setting up High Availability is more complicated than that of warm standby failover because you are following steps on several servers, but the end-result is worth it.

Step 1: Prerequisites

To get started, make sure you have five clean Windows 2016 or 2012r2 64-bit servers.

  • Two servers are for Master/Master MySQL DB replication (GTID)
  • At least two servers for IIS Web servers
  • One server for IIS Farm/Microsoft ARR (application request routing)
  • All servers need to be added to a domain as domain member servers
  • A DFS Replication Group in Active Directory with adding all IIS servers to that group
  • Microsoft .NET framework 3.5 must be installed on all IIS servers

The server requirements are as follows.

  • Two 2.0 GHz processors or one 2.0 dual-core processor for up to 15,000 users (add a core for every additional 15,000 users)
  • 4GB RAM for up to 15,000 users (add 4GB for every additional 15,000 users)
  • 20GB free hard disk space (add 4GB for every 100 printers)

Continuous ReplicationReplication takes place 24/7. The amount of bandwidth that replication consumes can be set to Full or one of a few other selections.

  2. In the Connections pane, expand the server node, and select Server Farms.
  3. In the Actions pane, click Create Server Farm.

The Create Server Farm wizard opens to the Specify Server Farm Name page.

  1. In the Server farm name box, type a name for your server farm and click Next.

NOTE: This Server Farm Name will become the URL the users will enter for accessing the Printer Installer Portal. For more information, see Step 8: Licensing.

  1. The Add Server page is displayed.
  2. In the Server address box, type the IP address or FQDN of the first server and then click Add.
  3. Continue typing IP addresses and clicking Add until all web servers are entered.
  4. Click Finish.
  5. When the Rewrite Rules window appears, click Yes.

7.B Configure Load Balancing with ARR

This procedure configures ARR load balancing with an algorithm that distributes incoming request evenly among the web servers. It then sets up the server-farm health test feature.

  1. Open IIS Manager.
  2. In the Connections pane, expend the server.
  3. Under the server node, expand Server Farms, and then select the server farm that you created.
  4. In the Server Farm pane, double-click Load Balance.
  5. On the Load Balance page, select Weighted round robin from the Load balance algorithm list, and then click Apply.
  6. In the Connections pane, select the server farm that you created.
  7. In the Server Farm pane, double-click Health Test.
  8. On the Health Test page, in the URL box, enter a URL that you want ARR to test by sending a GET request to determine the health of the servers. Use http://farmwebserver/tools/tests/tests.php.

NOTE: "farmwebserver" should be replaced with the hostname.domain.com of the ARR server.

  1. Click Apply.
  2. To verify the health of your farm, click Verify URL Test.

7.C Change Application Pool Settings

Because all HTTP requests and responses go through ARR, delays or errors occur if the application pool times out or recycles unexpectedly. Set the idle time-out and application pool recycle settings to zero to avoid problems.

  1. On one of the web servers in your farm, open IIS Manager.
  2. In the Connections pane, select Application Pools.
  3. In the Application Pools pane, select the application pool for your website, and then click Advanced Settings in the Actions pane.
  4. In the Advanced Settings dialog box, expand Process Model, and change the Idle Time-out (minutes) value to 0.
  5. Expand Recycling, and change the Regular time intervals (in minutes) value to 0.
  6. Click OK.

7.D Configure Client Affinity with ARR

  1. Open IIS Manager.
  2. In the Connections pane, expend the server.
  3. Under the server node, expand Server Farms, and then select the server farm that you created.
  4. In the Server Farm pane, double-click Server Affinity.
  5. Select the Client Affinity check box in the Server Affinity page.
  6. Click Apply.

7.E Configure Proxy Timeout with ARR

  1. Open IIS Manager on ARR server.
  2. In the Connections pane, expend the server.
  3. Under the server node, expand Server Farms, and then select the server farm that you created.
  4. In the Server Farm pane, double-click Proxy.
  5. Increase Time-out (seconds): to 60.
  6. Click Apply.

Conclusion

Printer Installer is now set up as an HA active/active solution. You can access the admin console and continue to configure by loading http://farmwebserver.domain.com/admin. You will need to install your clients using the same URL or hitting the portal at http://farmwebserver.domain.com.

You will also have request in Printer Installer license that will need to be put on all Printer Installer Web Servers in the wwwroot directory. Please be sure and request a High Availability license as it will need to include the hostname of the webfarm and all PI IIS web servers.

NOTE: "farmwebserver" should be replaced with the hostname.domain.com of the ARR server.

Step 8: Licensing

Request an HA license file and put it to each of your IIS servers where PrinterInstaller is installed. The licensed URLs need to be, for example:

farmwebserver;PIserver1.domain.com;PIserver2.domain.com;PIServerN.domain.com

Step 9: Using SSL Certificates for HTTPS

To use SSL Certificates, you will need to upload the SSL Certificate to the IIS Farm/ARR server and each of the Printer Installer IIS web servers. You will then need to bind HTTPS port 443 to the SSL Certificate.

Step 10: Upload a Certificate to the IIS Farm/ARR Server

  1. Open IIS Manager, select your server name, and double-click Server Certificates.
  2. On Actions menu, select Import.
  3. In the Import Certificate dialog select the Certificate file, enter the Password and select Personal for the Certificate Store.
  4. Click OK.
  5. In Connections, select Default Web Site.
  6. In the Actions menu, click Bindings.
  7. In the Site Binding dialog, Click Add.
  8. In the Add Site Binding dialog, select HTTPS as the Type, select the SSL Certificate imported above as the SSL Certificate, and click OK.
  9. Click Close.
  10. In the Connections, select your webfarm and double-click Routing Rules.
  11. Deselect Enable SSL offloading and then click Apply in the Actions menu.

The ISS Farm/AAR server is now configuring to use HTTPS, but it wont work until you import and bind HTTPS port 443 on each Printer Installer IIS Web server. Follow these steps on each server:

  1. Open IIS Manager, select your server name, and double-click Server Certificates.
  2. On Actions menu, select Import.
  3. In the Import Certificate dialog select the Certificate file, enter the Password and select Personal for the Certificate Store.
  4. Click OK.
  5. In Connections, select Default Web Site and click Bindings. In the Actions menu.
  6. In the Site Binding dialog, Click Add.
  7. In the Add Site Binding dialog, select HTTPS as the Type, select the SSL Certificate imported above as the SSL Certificate, and click OK.
  8. Click Close.
  9. Repeat these steps on each Printer Installer IIS Web Server.

To verify you should be able to access Printer Installer Admin from https://farmwebserver.domain.com/admin.

NOTE: Master/Master replication is now configured and running. It is still good practice to do regular backups of the MySQL DB. It is recommending that these backups happen on the secondary server. To determine which MySQL DB that Printer Installer is using you can look in the connection.txt file located in the \inetpub\wwwroot\storage\app directory.

Step 11: Cache Location

If driver caching is going to be used in an HA environment, it is important to know that none of the replicated PI (IIS) server must be set as a cache manager. Use any other computer with the PI Client installed as cache manager.

Step 12: Upgrade Printer Installer in an HA Active/Active Environment

HA active/active setup is a very involved setup and will require a little more work to upgrade. Follow these steps when upgrading.

  1. Be sure that you have a recent backup of the MySQL database.
  2. Following good practices, schedule a maintenance period when the Printer Installer servers can be down for a brief time.
  3. Take Printer Installer Web IIS servers offline
  1. Open IIS Manager on the IIS Farm/ARR and select servers under your farm name.
  2. In the Servers window, right-click each server and select Take Server Offline.
  1. If you are upgrading to from 17.1 to 18.1, disable all DFS connections and memberships, run the 18.1 upgrade, and then enable the DFS connections and memberships.
  2. On each of the Printer Installer severs complete the following:
  1. Open IIS Manager, select your server name, and double-click Shared Configuration.
  2. Deselect Enable shared configuration and click Apply in the Actions menu.
  3. Click Yes on the Shared Configuration dialog.
  4. Click OK to confirm changes.
  5. Download PrinterInstallerUpdate-<build number>.exe and launch.
  6. In the Printer Installer install, select Advanced install and then click Next.
  7. Select Printer Installer is hosted on the following IIS Website and click Next.
  8. Deselect Backup website files and Backup database for Backup Options.
  9. Deselect Enable automatic upgrade.. for Client Upgrade. You can configure this in the Admin Console after upgrade.
  10. Click Install.
  11. When the install completes, click Close.
  12. Verify that the browser comes up and you can log into the Admin Console
  13. Open IIS Manager, select your server name, and double-click Shared Configuration.
  14. Select Enable Shared Configuration and then select c:\IIS-Shared-Config (or the directory that contains the replication shared configuration) in the Physical path field.
  15. Click Apply in the Actions menu.
  16. Enter the password in the Encryption Keys Password dialog and click OK.
  17. Click OK to confirm changes
  18. Click OK on object reference warning dialogs
  19. Click OK to confirm changes
  20. Repeat steps 4, a-r on all PrinterInstaller IIS Web servers.
  1. Bring Printer Installer Web IIS servers online.
  1. Open IIS Manager on the IIS Farm/ARR and select servers under your farm name.
  2. In the Servers window, right-click each server and select Bring Server Online.
  1. To verify you should be able to access Printer Installer Admin from https://farmwebserver.domain.com/admin.

NOTE: "farmwebserver" should be replaced with the hostname.domain.com of the ARR server.

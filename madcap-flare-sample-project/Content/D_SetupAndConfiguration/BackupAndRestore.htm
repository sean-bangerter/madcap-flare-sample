Backup and Restore

Back up the Printer Installer Database

Use any MySQL database backup software to back up the MySQL database. The following steps describe the process for backing up the Printer Installer database with the built-in 'mysqldump" command line.

  1. From the server running MySQL, click Start and type CMD to open a command prompt.
  2. Right-click Command Prompt and select Run as Administrator.
  3. Type cd c:\program files\mysql\MySQL Server 5.5\bin to change directories.
  1. Type the following on a single line.

mysqldump --user=[DBUser] --password=[DBPassword] --single-transaction --add-drop-database --databases [DBName] > C:\backup.sql

NOTE: The command shown above required the values for the following items: DBName, DBUser, and DBPassword. These values can be found at c:\inetpub\wwwroot\config\settings.php as shown in the following example. Keep in mind that you must remove the brackets surrounding DBName, DBUser, and DBPassword.

The backup file will be saved to the location designated in the script above. In this case, the backup file is saved as c:\backup\sql and will be used for restoring the database on a new server, if necessary.

Restore Printer Installer on a New Server

STEP 1: Install Printer Installer on a New Server

NOTE: If you have already performed the installation on the new server, skip Step 1 and move to the next step.

Prior to the installation, you should have received an email with a link to the installation file you will use to download the installation files. If you have not received this email or link, please see your PrinterLogic sales representative. The following steps describe the process for installing Printer Installer on your Windows server.

  1. Open the installation email you received from your PrinterLogic sales representative.
  2. Copy the link, which should appear as shown below.

NOTE: Do not copy and paste the link below. It is used here only as an example.

http://downloads.printerlogic.com/PrinterInstallerSetup.exe.

  1. Log in to your Windows server.
  2. Open a browser and paste the link you copied into the Address bar.
  3. Press <ENTER> on your keyboard.
  4. When the following window appears, click Save to download the files to a folder on the server.
  1. When the system finishes downloading the installation files, click Run.
  2. In the User Account Control window, click Yes when the system asks you if you want to make changes to the computer.
  3. Click Standard.
  1. Click Next.

The following option will appear only if you have more than one website set up in the IIS Manager on the server you are using for Printer Installer.

The following image shows the IIS Manager with more than one website.

You can either delete one of the websites in the IIS Manager and then try the Printer Installer installation wizard again, or you can select one of the websites in the Printer Installer installation wizard and then click Next.

The installation wizard will run automatically and add the following items to your server.

  • IIS
  • PHP
  • Printer Installer website files in the default website location
  • MySQL

NOTE: The Printer Installer database will not be created during installation. Rather, it is set up the first time you log in to the Admin Console.

The installer will notify you when the installation is complete.

  1. Click Close.

STEP 2: Apply the License File to the New Server

NOTE: If you have already applied your license to the installation on the new server, skip Step 2 and move to the next step.

You will not need a license file only if you created an A Record or a CName for the Printer Installer server website, which is recommended.

  1. Copy the .dat file from server you are already using for Printer Installer.
  1. Paste the .dat file into the webroot folder on your new server.

STEP 3: Import the Database File from the Backup File

  1. On the server where Printer Installer is installed, copy the backup file at c:\backup.sql.
  1. Paste the backup file to the root folder (c:) on the new Printer Installer server.
  2. Open Windows Explorer and navigate to c:\inetpub\wwwroot\config.
  3. Open settings.php in Notepad or your preferred text editor.
  4. Copy the values for DBNAME, DBUSER, and DBPASSWORD.
  1. Paste these values into a temporary location such as Notepad.
  2. Click the Start menu and type CMD.
  3. Right-click Command Prompt and select Run as Administrator.
  4. In the command prompt, type cd c:\program files\mysql\mysql server 5.5\bin and then press <RETURN> on your keyboard.
  5. Type mysql --user=[DBUser} --password=[DBPassword] [DBName] < c:\backup.sql and then press <RETURN> on your keyboard.

The restored files do not include the portal header file. As a result, the header will display the default logo.

STEP 4: Point the Client to the New Server

Option 1: Client HomeURL is a DNS Alias

This option is the preferred method. Change the client's HOMEURL to point to the new server. Update the Client's HomeURL.

Option 2: Client HomeURL is a Server Name

Open DNS and change the DNS entry to point to the new server.

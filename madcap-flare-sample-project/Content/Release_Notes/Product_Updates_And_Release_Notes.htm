Product Updates and Release Notes

Secure Release Printing

NOTE: You cannot upgrade directly from 15.1 (or older) to 18.1. You must upgrade to 16.1 or 17.1 before upgrading to 18.1.

Release Date: March 19, 2018

PHP Version: 7.0.28

If you are interested in upgrading to 18.1.1.91, please contact your PrinterLogic Support Engineer.

New Features

  • Printer Installer client now supports Redhat.
  • Printer Installer client supports Ubuntu 15 and 16.
  • Printer Installer now uses service hosts for mobile and email printing.
  • We now meet color contrast requirements found within the Section 508 guidelines.
  • We now support the keyboard navigation found within the Section 508 guidelines.
  • You now have the option to set up Non Single Sign-on for Toshiba printers.
    • For Non Single Sign-on, Toshiba customers must have the latest firmware and a patch available from Toshiba. Please contact Toshiba for more information.

Fixed Issues

  • Xerox Workcenter 775 printers are no longer having issues with Printer Installer.
  • SMTP settings now work correctly for Gmail.
  • You can register printers with Google Cloud Print without any issues.
  • Multiple files open on Mac no longer causes the Admin Page to hang.
  • Scheduled Reports no longer shows blank results (unless no results are found).
  • Socket Server setting in the General Settings no longer automatically unchecks.

Known Issues

  • Printer Installer 18.1 is experiencing high CPU usage, which can cause performance issues. A fix is currently in progress.
  • Badge authentication may not work correctly with various Toshiba printers.
  • Client installation prompt appears even if the client has already been installed on Safari for computers using Mac Sierra.
  • Sorting the Print Job Records report by Color Duplex or Pages results in a 500 error (Internal Server Error).
  • Release Portal application fails to launch on Chromebook. Note that you can still browse to the Release Portal directly from a web browser.
  • Email file attachments are not printing correctly in the Mobile Printing module.
  • If you rename a group in Active Directory and then change the group name in Advanced Groups, Printer Installer does not retain the new Advanced Group name.
17.1.1.165 Maintenance Release

Release Date: December 19, 2017

PHP Version: 5.6.32

Download Printer Installer 17.1.1.165

Fixed Issues

  • Printer installation no longer fails when Portal Security is set with Active Directory objects.
  • Maps are now centered in the browser window.
  • Resolved an issue with Ricoh's proprietary software, Streamline, which prevented the user from logging out of the Control Panel Application without rebooting the printer.
  • In the Self-service Portal, users who have rights to a child folder but not the root folder can now access the child folder.
  • Print jobs are not automatically released from a closed laptop when that laptop comes out of sleep mode.
  • Control Panel Application no longer installs with the password field blank.
  • The first Secure Release Print job sent to the printer shows as DOMAIN\USER as designed.
  • Print spooler no longer crashes.
  • Removed a possible whitelist security vulnerability that, under very specific circumstances, could allow malicious code to be run on an end-user Max/Linux computer with elevated permissions.

Known Issues

  • Client installation prompt appears even if the client has already been installed on Safari for computers using Mac Sierra.
  • Printer status is not available for printer objects configured to use printer-specific SNMP v3 settings.
  • Sorting the Print Job Records report by Color Duplex or Pages results in a 500 error (Internal Server Error).
  • Release Portal application fails to launch on Chromebook. Note that you can still browse to the Release Portal directly from a web browser.
  • Email file attachments are not printing correctly.
  • If you rename a group in Active Directory and then change the group name in Advanced Groups, Printer Installer does not retain the new Advanced Group name.
  • In certain circumstances, scheduled reports may appear without data.
17.1.1.123 Maintenance Release

Release Date: September 28, 2017

PHP Version: 5.6.31

Fixed Issues

  • The Chrome Extension installs with a fresh 17.1 client install.
  • Jobs sent to an OM Plus queue created by Printer Installer now display the document title and user information.
  • Scanned badges on Toshiba printers work as intended rather than beeping without any activity on the printer.
  • Simple Badge Release jobs on RFIDeas devices release as intended.
  • The user interface on Lexmark printers no longer freezes when deleting print jobs.
  • The Import Tool on 32-bit 2003 servers no longer displays "Error 102: The source structure was not large enough."
  • The Total Pages by Week report graphs reflect the numbers in the reports as designed.
  • User can search the portal page for printers without access to parent folder as intended.
  • Printers that have a name with special characters can be imported as designed.
  • Printers will be deployed to a computer even if the computer's hostname is longer than 15 characters.
  • Custom fields with a number value type are now searchable.
  • Single Sign-on in HP printers does not give an error before displaying the print queue.
  • After registering a badge at a printer, the badge can be unregistered as designed.
  • When you copy a printer object with CPA installed, the CPA installation is no longer copied as well.
  • On Lexmark printers, manual login no longer fails with badge authentication enabled.
  • Email Print jobs and Direct Email Print jobs are no longer identified as guest print jobs.
  • Direct Email Printing jobs with different attachment types no longer print only the last file type.
  • Email Print and Direct Email Print jobs are no longer duplicated on iOS 10 and 11 devices.

Known Issues

  • If a user swipes their badge on a Dell Lexmark printer (before the Control Panel Application finishes loading) the Username field will populate with an incorrect username.
  • When changing printer-specific Secure Release settings, pre-existing Secure Release jobs in queue are no longer available to release.
  • If the global override option is not selected, then Yes (Always) and Yes (Prompt) Secure Release settings do not function correctly.
  • On iOS devices, Email Print Jobs and Direct Email Print jobs with attachments print only text beneath the attachment rather than all body text and attachments.
  • On iOS devices, Email Print jobs and Direct Email Print jobs without attachments print the subject and the body in two separate pages.

Release Date: August 23, 2017

Fixed Issues

  • Certain badge and card reader combinations are no longer outputting two rows of numbers.
  • CPA and card readers now working properly following a Printer Installer upgrade from version 16.x to version 17.1.

17.1 Major Release

Release Date:June 16, 2017

PHP Version: 5.6.30

New Features

  • The PrinterLogic client now supports Firefox web extensions.
  • Printer Installer now integrates closely with OM Plus to enable you to set up and manage queue names in the Port tab, which will be mirrored in the OM Plus Delivery Manager.
  • You can now select the currency and format that you want to use in your reports.
  • The Control Panel Application, client, and the portals now include Italian and Turkish in its list of supported languages.
  • PrinterLogic now offers High Availability failover, which automatically kicks off a failover server in the event of your primary server's failure.

Fixed Issues

  • In the Self-service Portal, any folder name entered into the search box automatically scrolls to that folder's location in the tree. For example, if you type Folder X into the search box, then the portal will automatically scroll down to the Folder X location in the tree.
  • Client uninstall on Mac Sierra now functions correctly.
  • Corrected an issue that would sometimes cause print jobs to be printed out of order.
  • In a virtual environment, the client icon that was not appearing in the System Tray the first time a user logs in is now fixed. The icon now appears as it should whenever a user logs in to Printer Installer.
  • On Ricoh printers, a user who badge swipes into the Control Panel Application no longer sees the previous user's print jobs.
  • Printers that an end-user installed from Citrix Xendesktop were being removed automatically, but now those printers remain and the profiles are removed (as designed).
  • Xerox Fiery driver settings are now being saved in profiles.
  • Authorization Code prompt in the Printer Installer client no longer appears.
  • Resolved slow tree load times.
  • Resolved 414 error that occurred when user/computer with several groups checks-in with the server, causing the URL to be too long.
  • Japanese characters now appear correctly on the Printer Installer confirmation prompt after server update.
  • Print Job report page counts now provide accurate data rather than returning high numbers.
  • Monthly scheduled reports that never arrived are now being received as designed.

Known Issues

  • Multi-page print jobs that are printed to Google Cloud Print appear as one-page documents in the Release Portal. For example, a five-page document will print five pages, but it appears in the Release Portal as a one-page document.
  • TCP/IP printers cannot be renamed from the Data Manager if you are using a PrinterLogic OM Plus license.
  • Direct Email Printing prints only the first file type if the email being printed contains different file types. For example, if the email contains .jpg, .bmp, and .png attachments, Direct Email Printing will print three copies of the .jpg file type.
  • Badges registered with the Elatec TCPConv2 simple badge release device will continue to release print jobs even if the global setting for Elatec TCPConv2 is disabled.
  • An error occurs when attempting to copy a printer in the Admin Console tree view.
16.1.5.94 Maintenance Release

Release Date: March 30, 2017

PHP Version: 5.6.30

Fixed Issues

  • Xerox printers using the FX series firmware (for example, C60 printers) now support badge scanning.
  • Control Panel Application now supports the HP X3D03A badge reader.
  • Resolved the 414 error received when the workstation checked-in with the server but the URL was too long.
  • For all printer types, fixed the badge registration issue where trying to specify a domain other than the default domain would fail to register a badge.
  • On the Xerox FX Series printers, resolved an issue where the uninstall feature locks up if no other applications besides PrinterLogic reside on the printer.
  • Resolved the Communication Error that appeared on Xerox printers when the end-user logs in to the Control Panel Application a second time, immediately after registering a badge.
  • On Xerox printers, Log In buttons now activate correctly following a JavaScript fix on the back-end.
  • Scheduled reports being created no longer time out and are sent as designed.
  • Mac mobile printing can now find printers on iOS devices so that printing from iOS devices works as designed.
  • Print Job Costs error that temporarily appears in Firefox no longer shows in the interface.

Known Issues

  • On Firefox v.52.0, access to the portal is blocked because the prompt to install Printer Installer continues to appear even after the client has been installed (both Windows and Mac).
  • The workaround for this issue is to use a different browser or to use an older version of Firefox.
  • On iOS devices, Email to Print and Direct Email to Print do not print emails with one or two lines of body text (attachments still print). Emails with more than two lines of body text print as expected.
  • On Mac OSX (Mavericks, Yosemite, El Capitan, and Sierra), changing the Secure Print Settings while there are secure print jobs held in the Release Portal will cause those print jobs to disappear or become unreleasable. Any secure print jobs printed after the change should behave normally.
  • Mac OSX Sierra does not support the client uninstaller and displays an error.
  • The workaround for this issue is to uninstall the client from the command line.
  • Printer Installer records Mac print jobs as black and white for printers that have the Job Cost Template set to Default Costs.
  • In Tools > Settings > Printing under Mobile Printing: Google Cloud Printing, if you have a proxy server enabled and set up, and you disable the Use Proxy Server field but retain the settings, then when you save the changes and click Register Printer, the GCP Printer Activation window will not appear.
16.1.5.74 Maintenance Release

Release Date: March 10, 2017

PHP Version: 5.6.30

New Features

The Whitelist feature enables an end-user to easily switch between servers for the purpose of installing printers without having to manually change the HomeURL and reinstall the client. Servers can be added to the Whitelist either through the Admin Console or through additions to the license file. The image to the left shows the Whitelist settings available in Printer Installer through Tools > Settings > Client.

Fixed Issues

  • Self-installed printers (not using Printer Installer deployments) during a VDI session are now visible after logging out and back in.
  • Custom RBAC permissions previously being "removed" during an upgrade now persist as intended.
  • A custom portal header previously disappearing following an upgrade now persists as intended.
  • The HP universal print driver that was not saving the Output Bin Options is now saving the setting as expected.
  • Badge swipe to log in to the Printer Installer Control Panel Application on an HP Laserjet 700 MFP M775 Model CC524A now opens the application rather than hanging indefinitely.
  • Customers unable to automatically upgrade the client must use absolute URLs rather than relative paths in the location headers.
  • This issue occurs only when upgrading Printer Installer server from version 16.1.3.533 or earlier to version 16.1.5.35 or later.
  • Improved memory and CPU usage for Mac.
  • Printer Installer now sends the Printer report when scheduled monthly as intended.
  • Deleting an Advanced Group that is assigned in the Deploy tab no longer deactivates that printer's Deploy tab.

Known Issues

  • On Firefox v.52.0, access to the portal is blocked because the prompt to install Printer Installer continues to appear even after the client has been installed (both Windows and Mac).

  • The workaround for this issue is to use a different browser or to use an older version of Firefox.

  • On iOS devices, Email to Print and Direct Email to Print do not print emails with one or two lines of body text (attachments still print). Emails with more than two lines of body text print as expected.

  • On Mac OSX (Mavericks, Yosemite, El Capitan, and Sierra), changing the Secure Print Settings while there are secure print jobs held in the Release Portal will cause those print jobs to disappear or become unreleasable. Any secure print jobs printed after the change should behave normally.

  • Mac OSX Sierra does not support the client uninstaller and displays an error.

  • The workaround for this issue is to uninstall the client from the command line.

  • In Chromebook only, print jobs are not appearing in the Portal page (browser) or in the Release Portal opened in the PrinterLogic application.

  • Printer Installer records Mac print jobs as black and white for printers that have the Job Cost Template set to Default Costs.

16.1 Major Release

Release Date: May 10, 2016

New Features

  • Print Job Costs assigns monetary values to printing scenarios to deliver Print Job Records report data for printers that release the print job scenarios.
  • Role-based Access Control (RBAC) assigns both standard and custom roles to Printer Installer users to provide or prohibit access to Printer Installer features and functionality.
  • Single Sign-on provides access to multifunction printer capabilities such as fax or copy when an end-user is simultaneously logged into the Control Panel Application.
  • Scheduled Reports sets any of the Print Job Costs reports to be sent to one or more users via email at a future date and time.
  • Simple Badge Release provides badge swipe access for pull print and secure print jobs to printers that cannot support the Embedded Control Panel application.

Enhancements

  • Printer settings have now been moved from the General tab to the Printing tab.
  • Printer Installer now supports both pull printing and secure printing for Mac and Linux.
  • Network Scanner (scan to server) has been added to the Tools menu.
  • Improved LDAP 4-stage Verification through the General Tab settings.
  • Caching is now available for Mac clients.
  • Unused drivers can now be removed from the Driver/Profile Repository.
  • SNMP Host Unreachable has been added as one of the alert triggers in Printer Installer.
  • SNMP Serial Number has been added to the Status tab.
  • Printer settings have now been moved from the General tab to the Printing tab.

No Longer Supported

  • Windows XP (32-bit and 64-bit)
  • 2003 Server (32-bit and 64-bit)
  • Windows Vista (32-bit and 64-bit)
  • 2008 R1 Server (32-bit and 64-bit)
  • Max OSX 10.7

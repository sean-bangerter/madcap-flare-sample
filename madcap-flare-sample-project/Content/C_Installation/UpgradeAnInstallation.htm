Upgrade an Installation

Upgrades occur on a regular basis with both maintenance and full releases. The updates can be downloaded here.

NOTE: You cannot upgrade directly from 15.1 (or older) to 18.1. You must upgrade to 16.1 or 17.1 before upgrading to 18.1.

A Note on OAuth for 18.1

OAuth is used to secure communication between the Printer Installer clients installed on workstations and the Printer Installer server. Printer Installer uses tokens to authenticate the client. These tokens are created in the Admin Console.

When you upgrade to Printer Installer 18.1, you will see a new section in the upgrade wizard. This section is shown below.

Notice the Require Client Authorization section, which enables you to set the point at which Printer Installer will require a token from the Printer Installer client installed on the workstations. There are three specific options in the Require Client Authorization section.

These options are explained in detail below.

 
Always Enabled

The Always enabled setting requires the clients installed on workstations to use a token to communicate with the Printer Installer server.

WARNING!:

If you select Always enabled during the installation process, then the change is immediate and irreversible. If the clients on your workstations are not updated with the 18.1 client before you proceed with this option, then they will no longer be able to communicate with Printer Installer.

The recommended workaround is to use the Enabled On calendar to select a date in the future by which your clients will be updated. This feature gives you a little leeway to prepare your clients prior to upgrading Printer Installer with the OAuth

Enable On...If you select Enabled On, then on the selected date the 18.1 clients installed on the workstations will be required to use a token to communicate with the Printer Installer server. The difference between Always enabled and Enabled on is that one is immediate and the other is stretched out a bit into the future. However, in the end, both options configure Printer Installer clients to use a token to communicate with Printer Installer.
Always DisabledIf you select Always disabled then nothing changes, and the clients installed on the workstations do not need the token to communicate with the Printer Installer server.

The OAuth Support section enables you to select the point at which OAuth will be installed on your Printer Installer server. While there is no option for Always disabled, if you're not interested in OAuth Support, just set the Enable on date to a date some point far into the future.

Always Enabled

The Always enabled setting immediately enables OAuth on your Printer Installer server

WARNING!:

If you select Always Enabled during the upgrade process, then the change is immediate and irreversible without PrinterLogic Support assistance. If the clients on your workstations are not updated with the 18.1 client before you proceed with this option then the clients will not be able to communicate with Printer Installer.

The recommended workaround is to us the Enabled On calendar to select a date in the future by which Printer Installer will be upgraded to 18.1 (and OAuth).
Enable On... If you select Enable On... and then select a date from the calendar, you are setting the date by which Printer Installer will be upgraded to 18.1 and OAuth. Keep in mind that there is no Always Disabled option for OAuth Support. If you do not want to use it, then you can assign a date far into the future.
Is there any downtime during an upgrade?

There is no downtime for users who have the direct IP printers already installed. However, the Admin Console and portals will temporarily be down during the upgrade. Additionally, all previous settings will remain unchanged following a successful upgrade.

Can I upgrade directly to the latest version?

Yes. Any version of Printer Installer can upgrade to the latest.

Will I still be able to print during an upgrade?

Yes. All direct IP printers that have already been installed will continue to print without problem.

Will I be able to install a printer during the upgrade?

No. You will not be able to reach the server during the upgrade.

Will my older client version work with the newer server version?

Yes. The Printer Installer Client is backwards and forwards compatible with the server version.

Will any of my settings change during an upgrade?

All your settings within Printer Installer will remain unchanged.

How long does the upgrade take?

  • The amount of time the upgrade takes depends on the what version you’re currently on, database size, and if you’re performing any backups.

  • When upgrading from 15.1 or older it will take longer upgrading to 16.1 and newer. There are several database updates that were implemented in the 16.1 version which depending on the size of your database can take some time. Set aside an hour to give you some extra time.

  • Backing up the database and web files can 10 to 20 minutes depending on your database size.

How do I update the client?

For more information, see Update the Client.

Do I need to stop any services?

The upgrade requires both MySQL and IIS running.

Do I need to reboot my server?

No. There is no need to restart the server after or before upgrading.

How can I ensure a successful upgrade?

Make sure you have enough space to run the upgrade. The recommended free space when upgrading your server is at least the size of your database. This is the ibdata1 file located on the server in C:\Program Files\MySQL\MySQL Server 5.5\data.

How do I revert back if something goes wrong during an upgrade?

The best way is by taking a snapshot and reverting back to the snapshot. If you don’t have a snapshot then you’ll need to import the your backup.sql file using the import command.

  • Before you begin, clear up as much space on the server as possible because temp files and log files generated during the upgrade could potentially use up all your drive space, which would prevent the upgrade from finishing.
    • Have at least the size of your database or more free to eliminate possible issues.
    • Refer to the database in c:\program files\MySQL\MySQL server 5.5\data/ibdata1.
    • The log files are located in c:\inetpub\logs\Log Files\W3SVC1.
  • Also, before you start the upgrade, back up your database and web files (just in case). If you can take a snapshot of the server, then that is also recommended because the snapshot is the easiest way to restore the server if anything happens.
    • While the upgrade does not take long, the database backup can take a while depending on the size of your Printer Installer database.
  • Clear out any old backups from your server. These backups are located in c:\ program files \Printer Properties Pro\Printer Installer\Backups\database.
  • When doing an update for a major release (16.1, 17.1, etc.), the recommended best practice is that you perform an advanced installation rather than a standard installation.
  • An upgrade provides the option to auto-upgrade the client at the same time you are upgrading Printer Installer. If you decide to upgrade the client at another time, open the Admin Console, and navigate to Tools > Settings > Client. In the Client Update section, click Enable automatic update during the next client check-in or refresh.
  • Older versions of the client will work with new versions of Printer Installer on the server. However, the recommended best practice is to use the most recent version of the client as well.
